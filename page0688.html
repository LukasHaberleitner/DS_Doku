Commands

This page deals with the Commands options for DPM - Assembly Process Planner and Manufacturing System Definition.
 

  • Assignment Assistant

  • Part Assignment

  • AST Editor

Assignment Assistant

The options listed under Assignment Assistant apply to the Assignment Assistant and Assignment Assistant for AST in DPM - Assembly Process Planner, and to the Assignment Assistant for Resources in Manufacturing System Definition.

 

When the Display Fastener Geometry in 3D Viewer check box is selected, the fastener geometry appears in the 3D view when Assignment Assistant is open, as well as in the Assignment Assistant 3D Preview window. If the check box is not selected, the fastener geometry only appears in the 3D Preview window, which speeds up the product representation updates, and consequently, both opening and closing the Assignment Assistant.

When the Preview Parts in Design Position check box is selected, the parts appear in the 3D Preview window in their design position; otherwise, they appear in their current position in the 3D view.

By default, neither check box is selected.

When the Copy Object Id on Process part relation is selected, copy the old PFPP relation object id to the new PFPP relation object in case the new relation is created and old is deleted. This happens in case of assigning parts from one process to another using Assignment Assistant only.

If the check box is not selected, then the OID value is not copied to new process product relation.

When you move a product assignment from an old process to a new process in Assignment Assistant, Object ID of old relation will be copied to an attribute of new relation if the attribute of old relation is empty. If attribute of old relation has a value, it will be copied to attribute of new relation. While copying old relation OID, $ sign will be replaced by _.

By default, external attribute is used.

In case of multiple moves, the relation ID value will be carried over to the latest new relation.  If the item assignment goes back to the original process, this is also treated as delete and create action. In such a case, the relation ID value will be carried over to the new relation saved to Manufacturing HUB.

Process Assignment Assistant

If you select the Allow Multiple Process Product Relations check box in the Commands tab of the DPM Assembly Process Planner section of Tools > Options, you see a list of four options under the arrows on the Process Assignment Assistant dialog box.

If you select the Allow Multiple Process Product Relations check box, all four process relation check box options will be enabled:

By default PFPP (process First Processes Product) is enabled.

These options refer to process product relations:

  • Process First Processes Product (PFPP)
  • Process Processes Product (PPP)
  • Process Removes Product (PRP)
  • Process Creates Product (PCP)

 

 

Part Assignment
The options listed under Part Assignment apply to the Assignment Assistant and Assignment Assistant for AST in DPM - Assembly Process Planner.

When the Check Part Assignment with Process in the Manufacturing Hub check box is selected, V5 checks parts to see whether they are assigned to processes in the Manufacturing Hub.  If any part is assigned to a process in the Manufacturing Hub, but the process is not loaded in V5, then V5 still recognizes that the part is assigned to a process, even though it does not show the process. As a result, the part is not listed under None in the process list in Assignment Assistant; it is treated as though it is already assigned.

If the check box is cleared, a part assigned to a process in the Manufacturing Hub only appears under the None process list in Assignment Assistant.

Licensing
  If you do not have a Manufacturing Hub license, the option is not available.

The default is that this option is selected.

AST Editor

This option enables the AST editor to show a part name as an instance name. This option is provided to improve the performance of AST Editor.

If the option in unchecked, the name appears as it is shown in the PPR tree.

The default is that this option is selected.
