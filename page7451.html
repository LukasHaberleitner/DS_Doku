Duplication Rules from Existing Annotation Set    

When using the Import Annotation Set command to import an annotation set from another CATPart document, rules are applied.

This section explains these rules. The following topics are discussed:

  • Geometry
  • Functional Tolerancing and Annotation Features
  • Functional Tolerancing and Annotation Features Duplicated but Cannot Be Rerouted
 

Geometry

  Geometries involved in Functional Tolerancing & Annotation features, are rerouted based on the comparison of geometrical characteristics.
  • Equivalent geometries are processed by their order of appearance in the Specification Tree in the target part document.
  • As soon as an equivalent geometry is found, affected Functional Tolerancing & Annotation features are rerouted.
    If no equivalent geometry exists, affected features are not rerouted and will be invalid (views will be not associative).
    • Equivalent geometry of a point is:
      • A point.
      • With the same coordinates in the axis system of the target 3D part as the original point.
    • Equivalent geometry of a line is:
      • A line.
      • With the same curvilinear length.
      • With the same center of gravity coordinates in the axis system of the target 3D part as the original line center of gravity.
      • With the same tangent vector at its center of gravity as the original line at its center of gravity.
    • Equivalent geometry of a face is:
      • A face.
      • With the same surface.
      • With the same center of gravity coordinates in the axis system of the target 3D part as the original face center of gravity.
      • With the same normal vector at its center of gravity as the original face at its center of gravity.
  • There is no geometry rerouting for Functional Tolerancing & Annotation features created on: PartBody/Bodies, Pad/Pocket, Groove/Shaft, Hole, Axis System, User Feature or Technological Result features.
    Affected Functional Tolerancing & Annotation features are not rerouted and will be invalid.
 

Functional Tolerancing and Annotation Features
  All Functional Tolerancing & Annotation features in the reference annotation set are duplicated even if they are not necessarily displayed.
 

Not Displayed Features
 
  • Functional Tolerancing & Annotation features in no show are kept with this status.
  • Functional Tolerancing & Annotation features layer is kept.
  • Annotation Set switch on/switch off status is kept.
 

Naming
 

Names of Functional Tolerancing & Annotation features are kept.

For Geometrical Link features of Functional Tolerancing & Annotation features, names are partially kept.

  • Group Of Surfaces and User Surface feature names are kept.
  • Geometric Component names are not kept and a default name is created.
 

Constructed Geometry
 
  • Constructed Geometry are duplicated, but for imported Constructed Geometry created from Version 5, only those are created.
  • Thread Constructed Geometry feature is rerouted when a Thread Technological feature, a Threaded Hole or a Thread is found, otherwise the Thread Constructed Geometry feature is invalid.
 

Parameters
 
  • All parameters in Functional Tolerancing & Annotation features are duplicated.
  • Formulas in Functional Tolerancing & Annotation features are not duplicated, but their values are kept.
  • Attribute links in Functional Tolerancing & Annotation Text feature are not duplicated, but text content is kept.
  • The links to generative parameter are not kept.
  • Tabulated geometrical tolerances based on length parameters, with their name displayed in the callout, are duplicated, but valid only if a parameter with the same name already exists in the target part.
 

Captures
 
  • Cameras in Functional Tolerancing & Annotation Captures are not duplicated, but Capture point of view is kept.
  • Captures links to Body or Geometrical Set features are not kept.
 

Offset/Aligned Section View/Cut
  The sketch referenced by an Offset or an Aligned Section View or Cut feature is duplicated and isolated if all its component views cannot be rerouted to the lines of the same sketch.
 

Ditto Type Note Object Attribute
  An instance of ditto type NOA is duplicated, but its link to the reference NOA document is removed. It is linked to the drawing document containing the corresponding detail.
 

Drafting Standard
 

The name of the standards associated with the source and target annotation sets must be identical and the target drafting standard level must be equivalent to or higher than that of the source. Otherwise, the drafting standard of the target annotation set is upgraded to the level of the source annotation set.


This is possible only if the drafting standard level of the source annotation set is of equivalent or higher than that of the target. If it is of a lower level, the duplication fails.
   
 

Functional Tolerancing and Annotation Features Duplicated but Cannot Be Rerouted
  This section defines the behavior of Functional Tolerancing & Annotation duplicated features that cannot be rerouted on the geometry of the edited document.
 

Annotation Features
  The annotations features are duplicated but are invalid.
 

Construction Geometry Features
  The construction geometry features are duplicated but are invalid.
 

Restricted Area
  The restricted area features are duplicated but are invalid.
 

Views
  The views are duplicated but the associativity to the geometry is lost.
 

Symmetry
  Functional Tolerancing & Annotation features can be rerouted on geometries created by symmetry, by using the same symmetry plane.
 

Merge of Annotation Sets
  Functional Tolerancing & Annotation features can be merged in an existing annotation set.
All rerouted Functional Tolerancing & Annotation features are duplicated, except for datums where naming is checked with the following rules:
 
  • The original datum is not linked with a reference already in-use by an existing datum:
    • The original datum label is not used, the duplicated datum is not modified.
    • The original datum label is already in-use, the duplicated datum is modified using the first free label.
  • The original datum is linked with a reference already in-use by an existing datum:
    • The target datum is linked with a Datum Reference Frame:
      • The original datum and its Datum Reference Frame are not duplicated.
      • Annotations linked with original datum are rerouted with the target datum.
    • The target datum is not linked with a Datum Reference Frame:
      • The original datum is linked with a Datum Reference Frame, the target datum is replaced by the original datum.
      • The original datum is not linked with a Datum Reference Frame:
        • The original datum is not duplicated.
        • Annotations linked with original datum are rerouted with the target datum.
 

Deviations, Correlated Deviations and Distance Between Two Points Features
 

The Deviations, Correlated Deviations and Distance between two points features are duplicated and rerouted. If there is no geometry in target part that supports the rerouting, these features are invalid.

Invalid Deviations, correlated deviations and distance between two points features are not displayed in the 3D of the target CATPart document. They are displayed only in the specification tree with a ! mask indicating that they are invalid features.
