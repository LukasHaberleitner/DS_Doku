Creating New Versions and Alternatives

New versions and   alternatives can be created at the time of selecting a process to load, when the Select Process dialog is displayed. 

1. When prompted to select a process, select the process to create a new version or alternative from:

 
2. Click the Alternate Version button to view display the Alternative/Version Management dialog box:

  3. The Alternative/Version Management  dialog box displays the current version number and alternative name of the selected process:

To create a new version of the selected process

  1. Enable the Create New Version radio button

  2. To preserve any current links to resources, enable the Preserve links to resource check box.

  3. Click the OK button. A new version of the process is created (Version 3.2 using the example above). In the above example, the new version of the process replaces Version 3.1.

To create a new alternative of the selected process

  1. Click the Create New Alternative radio button and enter a name for the new alternative in the empty text box. 

  2. Use the button next to the text box to view a list of names that have already been used. Names that have already been used can not be used again.

  3. To preserve any current links to resources, enable the Preserve links to resource check box.

  4. Click the OK button to create the new alternative. The new alternative will be created at the same level as the process it was created from in the PPR tree.

Click the Back button to return to the Select Process dialog without any changes.

Click the Cancel button to cancel the Process Selection process without any changes.
