Defining a Constructed Feature Prior to Measurement

In certain planning scenarios, users may wish to define a constructed feature using an actual feature that has not yet been measured.
When users define a constructed feature that relies upon a measurement that has not yet occurred, they see the Construction Warning dialog box:
 When an activity (such as alignments or measure with RMEAS) references this type of constructed feature, the user receives a warning about the fact that the measurements on which the constructed feature is based have not yet occurred.  Users may opt to continue with activity or not. 

Impact on DMIS
Construction statements can only be exported to the DMIS file after all referenced actual features are measured
The DMIS command warns of errors if the required features have not been measured.
