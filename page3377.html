Assembly Variant

 

This task shows you how to use the Assembly Variant toolbar to define, instantiate and replace a variant generic product. The task deals with:
  • Define Variant Generic Product
  • Instantiate Variant Generic Product from File
  • Replace Variant Solved Product

Open an assembly.
 

Define Variant Generic Product

The Define Variant Generic Product command allows to create a new variant and select components as per the required configuration.

  1. Click Define Variant Generic Product in the Assembly Variant toolbar. The Variant Generic Product Definition dialog box is displayed. All the components at the first level nodes are displayed in the Variant Definition field. By default they are all selected. You cannot change their configuration.


  • Variant List field displays the list of variants with their comments.
  • Variant Definition field identifies relations between instances of products and variants.
 

  1. Click New to create a new variant. A new column New Variant1 is added in the Variant Definition field.

  2. Select New Variant1 and click it to change the name of the new variant in the Variant List field. You can also put comments related to it. Type name as 1 Gear and add the comment as This is 1 gear Assembly for pedal in the Comments field.

  3. Clear check boxes for Front_sprocket_2.1 and Front_sprocket_3.1 in the New Variant1 column of the Variant Definition field. 

  4. Again click New to create a new Variant.

  5. Select New Variant2 and click it to change the name of the new variant. Type name as 2 Gear and add the comment as This is 2 gear Assembly for pedal in the Comments field.

  6. Clear check box for Front_sprocket_3.1 in the New Variant2 column of the Variant Definition field.

  7. Click OK. A design table is created (or updated if you are editing the generic) with the first level component.

  8. Save the CATProduct file and close it.
  • The Delete button deletes the selected variant.
  • If you click Define Variant Generic Product and the current CATProduct is already a generic, then it will be edited.
  • When you enter the command, if the DT and two variant parameters already exist, the generic is recognized and the variants are displayed in the dialog box.
  • An error is displayed if the generic is no more valid. You can delete DT and parameters manually, but cannot delete only DT or only parameters otherwise the application raises an error message.
  • In edition mode, a check is done to see if the first level nodes have changed or not. The changes in these nodes make the generic product invalid.
 

Instantiate Variant Generic Product from File

The Instantiate Variant command allows to instantiate variant generic product from file.

  1. Click Instantiate Variant in the Assembly Variant toolbar. The Variant Generic Product Instantiation dialog box is displayed.

  2. Click Select... to select the Variant Generic Product file. Select the SprocketProduct.CATProduct file that was saved in the Define Variant Generic Product section. The possible variants are displayed in the Variant list.

  3. Select the 1 Gear in the Variant List.

  4. Click OK. The configuration set for 1 Gear in the Define Variant Generic Product section is opened.
The Reuse Solved Variant option is disabled.
 

Replace Variant Solved Product

  The Replace Variant command replaces the existing solved by a new variant of another generic product. It is more advanced application used in SmarTeam where Reuse Solved Variant option is not disabled.
