Starting with Schematic Driven Design

Equipment and Systems design is based on work packages. Each work package is created directly under the PRC. During 2D/3D schematic driven routing, the schematic documents are inserted directly underneath the PRC and not underneath the work packages.  
You cannot select a P&ID in the schematic browser while schematic routing. In the schematic mode when you click Route a Run, the Open Schematic Diagram option is unavailable.
1. Create the schematic documents. (e.g. TEST_PID1, TEST_PID2 and so on).
  2. Insert the schematic documents under the root (PRC).
    Product structure is as follows:

PRC (or Root document for file based)
|
|
| ------ >  TEST_PID1 (Inserted directly under the PRC)
|
| ------ >  TEST_PID2 (Inserted directly under the PRC)
|
| ------ >  TEST_PIPIING WORKPACKAGE1
|
| ------ >  TEST_PIPIING WORKPACKAGE2
|
| ------ >  TEST_EQUIPMENT_ WORKPACKAGE1
|
| ------ >  TEST_EQUIPMENT_ WORKPACKAGE2
 
Note: You can drag and drop the schematic documents to insert them into the PRC.
In case of file based scenario, the schematic documents are inserted under the root document.
