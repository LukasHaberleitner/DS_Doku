Computing Quantity According to Instance Activity

 

When an assembly is saved in SmarTeam, it is useful to be able to obtain the quantity of a given component under its parent. This functionality provides SmarTeam with the activity information of each instance of a component under its parent. In this way, the quantity can be computed as the sum of all active instances of this component.

In the following example, the instance of Part1.2 is deactivated. The quantity of Part1 under Product1 is 2.
