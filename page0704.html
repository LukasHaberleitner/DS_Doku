Fasteners Assignment

Assignment Status Color

If this option is selected, the color of the fastener on the 3D geometry will reflect the fastener's assignment status when the Fastening Sync command is run.

By default, the option is selected.

Active Validation Rules

Process Syntax Validation
Process syntax validation validates fasteners according to the process.

Collocation Validation

Collocation validation determines whether the fastener's manufacturing and design positions are collocated.

By default, neither rule is selected.

Fasteners in Viewer

Show the design fastener
The design fastener tags can either be visible or hidden.

You can also use this option to determine the color, line type, and weight of the lines that make up the x's marking the design tags.

By default, the design fastener tags are visible.

Show the line between the design fastener and the manufacturing fastener
The line between design fastener tags and manufacturing fastener tags can either be visible or hidden. The line can only be visible if both the manufacturing and design fasteners are visible.

You can also use this option to determine the color, line type, and weight of the lines.

The line between the design fastener tags (or positions) and manufacturing fastener tags is unaffected by the assignment status. The status color displayed on the manufacturing fastener tags and the lines that make up the manufacturing fastener tags (tag/frame/coordinate system) are of double thickness while displaying status colors. The orthogonal vectors that make up the manufacturing fastener tags are the ones that are of double thickness.

The example below shows the manufacturing tags set as visible, in yellow, and the line between the manufacturing tag and design tag (which is only visible when the two tags are not coincident) set to green.

By default, these lines are hidden.

Show the manufacturing fastener
The manufacturing fastener tags can either be visible or hidden.

You can also use this option to determine the color, line type, and weight of the lines that make up the x's marking the manufacturing tags.

By default, the manufacturing fastener tags are hidden.

CGR from XML

Default path for the XML document
To customize the appearance or visualization of a fastener, users can create an XML file that points to a CGR file.  The location of the XML document is set in this tab.

The default value for the path is shown.

Default File Name for the XML document
The default XML file name used is listed in the tab. An example of a graphics location file, which is used with Edit BIW Feature Appearance  , follows.
<xml>

<AdvancedGraphics>

<Graphic ID="Sphere">
<Path>D:\users\DevelDoc\RECORDINGS\FEATURE_CGR\Sphere.cgr</Path>
</Graphic>

<Graphic ID="Cone">
<Path>D:\users\DevelDoc\RECORDINGS\FEATURE_CGR\Cone.cgr</Path>
</Graphic>

<Graphic ID="Cube">
<Path>D:\users\DevelDoc\RECORDINGS\FEATURE_CGR\Cube.cgr</Path>
</Graphic>

</AdvancedGraphics>

</xml>

By default, the name is WeldText.
