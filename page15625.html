Using a Design Table Saved in ENOVIA V5 VPM in Another Part

This task is designed to show you how to use a design table saved in ENOVIA V5 VPM in a new Part. It is divided into the following steps. Your:
  • Save the Excel file in ENOVIA V5 VPM.
  • Open the Part that will contain the Excel file in CATIA and save it in ENOVIA V5 VPM.
  • Perform a Search in the ENOVIA V5 VPM database.
  • Send the founded Part to CATIA.
  • Save the design table Excel file on a local disk.
  • Create the design table file in CATIA and save the .CATPart file in ENOVIA V5 VPM.
  • Send the file from ENOVIA V5 VPM to CATIA and perform a synchronization.
For more information about the use of design tables in ENOVIA V5 VPM, see Working with Design Tables in ENOVIA V5 VPM.

Save the Excel file in ENOVIA V5 VPM

  1. Open your ENOVIA V5 VPM database.

  1. In ENOVIA V5 VPM, click the ENOVIA Home icon (). Expand the Content Management node and double-click the Documents folder.

  1. The Content Management Startup Selection dialog box is displayed. Click Create a New Document and click OK.

  2. In the Document ID field of the opening window, enter New_Designtable and click Browse. The Choose a file dialog box is displayed.

  3. Select the KwrDesignTable1.xls file and click OK. The design table external file is saved in your ENOVIA V5 VPM database.

  4. In CATIA, open the KwrDesignTable.CATPart file.

  5. Click the Connect to ENOVIA V5 VPM icon () to connect your ENOVIA V5 VPM database.

  6. Click the Save in ENOVIA icon (). The Save in ENOVIA V5 dialog box is displayed. Select the file and check the Publications Exposed storage mode.

  7. Click OK to validate. Close the file in CATIA.

The Excel file and the new .CATPart file are now both saved in ENOVIA V5 VPM.

  1. In ENOVIA V5 VPM, expand the Content Management node (if need be) and double-click the Documents folder.

  2. The Content Management Startup Selection dialog box is displayed. Click Search Documents and click OK to launch the search.

  3. In the opening Search window,

    • select Document Revision in the Search for scrolling list.
    • select the Creator field, enter the Creator's name and click Search. The KwrDesignTable.CATPart file and the KwrDesignTable1.xls are displayed in the Results tab.

  4. Right-click the KwrDesignTable.CATPart file and select Send To>CATIA V5. The file is displayed in CATIA.

  5. In ENOVIA V5 VPM, right-click the KwrDesignTable1.xls file and select Send To>Local Disk.

  6. In the Copy Out dialog box, indicate the directory where the Excel file will be stored as well as the file name (DesignTable1 in this scenario).

  7. Click OK when done. An information dialog box indicates that the file was correctly saved on your disk. Click OK to exit this dialog box.

  8. In CATIA, change the part number of the KwrDesignTable.CATPart file into KwrDesignTable2.

  9. Click the Design Table icon () in the Knowledge tool bar. The Creation of a Design Table window is displayed.

  10. Click the Create a design table from a pre-existing file option and click OK. The File Selection dialog box opens.

  11. Select the KwrDesignTable2.xls file and click Open. Click Yes when prompted to associate the columns of the table with the parameters.

  12. Click OK to apply the default configuration. DesignTable.1 is displayed below the Relations node. Save your file.

  13. Click the Save in ENOVIA icon (). The Save in ENOVIA V5 dialog box is displayed.

  14. Select both files, check the Publications Exposed storage mode.

  15. Close the file in CATIA.

Search for the Saved Design Table in ENOVIA V5 VPM

  1. In the Search tab, click Search to launch the search. Your files display in the Results tab.

  2. Right-click the KwrDesignTable.CATPart file and select Send To>CATIA V5.

  3. Right-click the DesignTable1 file and select Synchronize (ENOVIA). The DesignTable1 is synchronized.
