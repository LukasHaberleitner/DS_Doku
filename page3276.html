Synchronizing the Attributes of an Activity Subtype

Whenever a new subtype is created from an existing activity type, the new subtype automatically inherits the attributes of the existing super type at the time of its creation. However, when attributes are then added or changed in the super type, any existing subtype activities under the parent activity type are not automatically updated to reflect the same attributes. This synchronization of the type must be done manually, and is accomplished via the Synchronize User Attributes command.

  1. Create a new activity type, as well as a subtype of that activity (see Creating a Basic Process Library).

  2. Add an attribute to the parent or super activity type (see Adding Attributes to an Activity).

    Note that the icon for the subtype (Child1) changes to indicate that the subtype is no longer in sync with its parent activity type (Parent):

  3. Click the Synchronize User Attributes with Super Type  .

  4. Click on the activity subtype to synchronize it with its parent or super activity type.

    The activity subtype now has the same attributes as its parent or super type, and its icon returns to the default image. Synchronization does not propagate deleted attributes from a super type to a subtype. The system marks a subtype as requiring synchronization only when an attribute is added to its super type.
