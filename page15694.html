Performing DMU Space Analysis Tasks with VPM Navigator

The following task shows you how to work with ENOVIA V5, using VPM Navigator.

VPM Navigator allows you to access both the product definition and the product data stored in the ENOVIA Hub from a single, simple user interface integrated within CATIA.
The following procedures are covered in this section:
  • Searching for Clashes in VPM Navigator
  • Sending Clashes to VPM Navigator
  • Opening Clashes in CATIA from a VPM Search Result window
  • Deleting Clashes
Make sure you have selected ENOVIAV5 in the View  > Toolbars  menu.
This scenario assumes that you already created interferences and saved them in ENOVIA V5 VPM database.
For more information, see CATIA/ENOVIA Interoperability Guide: User Tasks: Saving in ENOVIA V5 VPM.

Before you begin:

  • In CATIA, click Connect in the ENOVIA V5 VPM Navigator toolbar . The Log On to Enovia V5 dialog box is displayed.
  • Enter your user name, and your password.
  • Select your role in the Role drop-down list, and click OK.
  • Create interferences and save them in ENOVIA V5 VPM by clicking the Save in ENOVIA icon .

Searching for Clashes

  1. Click Search ENOVIA .
    The VPM Search dialog box is displayed.

  2. In the Objects drop-down list, select the VPMClash option.

  3. In the Domains list, select Clash.

  4. Enter your search criteria if necessary.

  5. Click OK.

     
    To customize the VPM search dialog box, see ENOVIA LCA People Organization and Security Guide: Understanding Object Access Control Using Masks: Using Masks.
    The matching clash objects are displayed the VPM Search Results window.

 

Sending Clashes to VPM Navigator

  1. In the VPM Search Results window, right-click the clash you want to send to VPM Navigator

  2. Select the Send to VPM Navigator item from the contextual menu displayed.

    The Part Instances and PRC of the selected clash object are displayed in the VPM Navigator window:

    The lock/unlock capability is not available in the contextual menu.

    For more information about Properties, Open, Transfer Ownership and Delete operations, see the VPM Navigator User Guide. 

Opening Clashes in CATIA from a VPM Search Result window

  1. Right-click the clash object and select the Open command.
    The Open Modes dialog box appears.
    The In Context (Product) option button is selected by default.

  2. Select the option button.


     
    You can open multiple clash objects at a time, for this select multiple clash objects. The related part instances involved as well as the interferences are loaded in the CATIA window.

    Note that if you select clash objects involving different PRCs, i.e.; if there are two clash objects and they belong to two different PRCs, two CATIA windows open.

    If the clash objects involve a single PRC, a single CATIA window opens.

  3. Click OK.
    The CATIA window opens.

Useful Reminder:

In Context (Product)

The tree of the selected object is opened and displayed within the appropriate context i.e. with all its parents as far as the PRC.
For example, if you select a CATPart instantiated on a PRC, the CATPart opens with the PRC and the associated instance. You can see the geometry, all the parent instances and the PRC.

Out of Context

The tree of the selected object appears out of context.
For example, if you select a CATPart instantiated on a PRC, the CATPart opens without the PRC and the associated instance. You only see the geometry but not the position of the instance.

The Out of Context mode allows you to:

  • open objects in a new or existing VPM Session. For more information about this workbench, see VPM Session: Working with Reference Data from VPM Navigator.

  • open objects in a window.

For more detailed information, see VPM Navigator User Guide.

The CATIA window opens.

Deleting Clashes

You can delete clash objects either from the VPM Search Results window or VPM Navigator window using the contextual menu.

  1. Right-click the clash object you want to delete either in the VPM Search Results window or in the VPM Navigator window.

  2. Select Delete in the contextual menu.
    A confirmation message appears.

     Alternatively, you can click Delete in the VPM Edition toolbar.

  3. Click Yes to confirm the delete operation.
    The selected clash object is removed from the database.
    Important:
    • You cannot delete multiple clashes at a time (using multi selection).
    • Preview of multiple clashes is not available (using multi selection).
