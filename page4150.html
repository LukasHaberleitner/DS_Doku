Creating a Tag Group  

This procedure describes how to create a tag group.  A tag group is a collection of tags that are grouped so that they can be manipulated together.  Tags can only be created in a tag group. Once a tag group has been created, you can insert tags in it.
The steps in this procedure are illustrated using the file BodySide_1.CATProcess.

  1. Click New Tag Group .

    The Tag Group dialog box appears.

  2. Enter the name of the tag group or accept the default name.

  3. Do you want the tag group to be associated with a device or product?

    • If no, select the OK button.  An explanation of where to find the tag group on the PPR tree appears below.
    • If yes, go to next step.

  4. Select the device or product, either from the 3D inventory space or from the PPR tree.

    The name of the selected device or product appears in the Owner Name field; the path to the file containing the item's data appears beneath.
    The lock icon in the dialog box indicates that the product data is stored in a read-only file.

  5. Do you want to set the Link Mode to Locally or Modify Reference?

    • If you select Locally , a positional link between the item and the tag group is created.  If the object moves, the tag group will follow.  The data is entirely created within the .CATProcess document and the .CATProduct file containing the device is not affected.
    • If you select Modify Reference, then the .CATProduct file containing the product data will contain the new tag group.  The tag group will be created as a child of the selected object, and you will need to save the .CATProduct file.

  6. Select the link mode option you want and click OK.

    The new tag group appears on the PPR tree.

Notes

  • If a tag or tag list is saved as a CATProduct (via Save as CATProduct from the contextual menu), any attributes that have been defined on the affected tag(s) are not saved in the resulting CATProduct.

  • Tag points and tag groups do not have an Initial State.
    A tag group defined under the ResourcesList acts as a manufacturing position or via points for the robot program. There is no meaning of initial state for robot targets. If a tag or tag group is attached to a product, it will move because the initial state is applied on the product and attachment--not because of the tag having its own initial state.
