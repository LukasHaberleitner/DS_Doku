Viewing the Results of CATDUA V5 Execution

This task shows you how to visualize the report of a Check or Clean process. You can have access to the results of the process when using:
  • CATDUA V5
  • CATDUA V5 through CATUTIL
  • CATDUA V5 through File > Desk....
Before performing this task, you have to be familiar with the use of the CATDUA V5. For this, refer to:
  • Running CATDUA V5 from a V5 Session.
  • Running CATDUA V5 in Batch Mode.

CATDUA V5:

When using this tool out of a V5 session, a message appears at the end of the Check or Clean process.

This message gives you the path directory of the Check or Clean report.

The Check or Clean results are stored in a .htm file (GlobalResults.htm) which can be accessed through a Windows Explorer. The results are identical to those you get with CATUTIL.

 

CATDUA V5 through CATUTIL:

Example 1: In check mode on one or several CATIA Documents

The .htm file gives the results for all the documents selected for the Check execution.
  A htm report file is generated for each of the processed documents (with the name "...filename.htm"). To access each report, click the corresponding OK or KO hyperlink in the Global Status.

This file is an htm version of the .txt file and it contains a hyperlink to the official documentation for each detected errors. You can click a rule, for instance FGM_1, and see the corresponding explanations about this rule in the documentation.

You can also access the .txt report file (....checker_traces.txt) by clicking the corresponding hyperlink in the Report - Return Codes column of the Report for Checked Documents table. This document displays the check results and the status of the return codes (i.e. whether they can be fixed or not).
 

The following information is displayed:

  • Priority Level: corruption level of the file.

  • Return codes can be fixed: the number of return codes that can be solved in Clean mode.

  • Synthesis report providing the list of errors for each priority.
 

 

Example 2: In Clean mode

The .htm file looks like the one of the Check mode. It is a .htm document with a Report for Cleaned Documents table containing hyperlinks.
Clicking a link displays the clean results in .cleaner_traces.txt with the number of clean return codes out of the total number of return codes:
 
 

Example 3: In Check/Clean mode on one or many documents

You can select one or many documents in both the Check and Clean sections. In that case, you have the same .htm file combining both Check and Clean results.
  • When using CATDUA V5 or CATDUA V5 through CATUTIL, several documents can be displayed in the Report / Return Codes box.
    The cleaned documents are automatically saved in the path directory you have previously specified. In this case, the .htm file indicates in the Save box of the Report for Cleaned Documents table that the cleaned documents have been saved .
  • The .txt report file cleaner_traces.txt is available by activating the hyperlink in the .htm report or by navigating in a Windows Explorer (in the same folder as the .htm file). This file displays the Check and Clean results. It informs you about the status of the return codes, the number of cleaned return codes out of the total number of return codes.
In a V5 session, the Check and Clean results are stored in a .htm file. This .htm file is associated to a .txt file which gives more details (the directory is given in the CATDUA V5 Information message or in the CATDUA V5 Information box of the CATDUA V5 dialog box).
After running the Check and Clean process, you obtain exactly the same result reports and you can have more details in the associated .txt file.
Note that you can automatically open the report window and select the .htm output file.

The Global Results contains a synthesis of both files (.htm and .txt), and the synthesis of the detected errors that have been checked and cleaned in the document. This table is identical to the one in the .txt file.
 

 

CATDUA V5 through File > Desk... in a V5 Session:
  The reports are generated in a Global Report file (OutputCATDUA.htm) and in a .txt file (checker_traces.txt or cleaner_traces.txt):
 
The following information is displayed:
  • Priority Level: corruption level of the file,
  • Return codes can be corrected: the number of return codes that can be solved in Clean mode.
  • Synthesis report providing the list of errors for each priority.
The output directory is not automatically emptied by the CATDUA V5 before launching a new Check or Clean execution. If you do not wish to visualize your former results in the .html file, you need to clear the directory. Moreover, the documents (CATParts, CATProducts, CATDrawings, CATAnalysis, CATProcess, CATMaterial...) in the output directory are not write-protected.

The following information is displayed:

  • Name of checked documents

  • Number of detected return codes

  • Number of fixed return codes

  • Number of priority1/priority2/priority3 return codes

  • Hyperlinks pointing to a .txt file which give the details of the results for each CATPart (in the Reports - Return codes column),

  • Information about the save operation.
