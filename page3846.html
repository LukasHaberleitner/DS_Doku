Understanding Warmstart/Automatic Save

When working with documents saved as Publications Exposed in ENOVIA V5 VPM, you have an option to have your data automatically backed up in a temporary directory in the case your session is unexpectedly interrupted.  Upon restart, you will be given the option to retrieve your data from this temporary storage and start your new session where it ended, regardless of whether any of the data has been saved to the database. 

To adjust the settings for Automatic Save, click Tools>Options>General in VPM Navigator.  In the Data Save section under the General tab, you can turn this feature on and off and adjust the time interval in which the save occurs.  For additional information on Data Save, see this topic in the General page under Application Settings.  For more information on saving data as Structure Exposed or Publications Exposed, see Insert New CATProduct in Session section of the ENOVIA V5 VPM section of Customizing Compatibility Settings in CATIA V5. 

Warmstart is not available for documents saved as Structure Exposed.  Automatic Save  (Warmstart) will not occur when a Structure Exposed document is loaded within a CATIA session, and will be deactivated until the Structure Exposed documents have been unloaded.  When the first automatic save is cancelled, a Warning box is displayed advising that Warmstart has been deactivated and also listing the documents that are stored as Structure Exposed.  Further cancellations of automatic saves won't display any warnings.  

Once all Structure Exposed documents have been unloaded, a Warning box is displayed advising that Warmstart has been reactivated and documents stored as Structure Exposed have been unloaded.

Note: During Warm Start, Logon at Start option is deactivated. The login panel which appears is triggered by Warm Start itself.
