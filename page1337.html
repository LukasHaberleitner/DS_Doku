Consuming a Cipher File

This task shows how to open a cipher file then save changes made to this file.
To be able to consume a cipher file, you must have been assigned the Read and Save credentials through the DRM settings.

  1. As user1, Select File > Open....

  2. In the File Selection dialog box, navigate to the cipher file.

    If you are not in the list of authorized users, the access to the file is forbidden and an error message is displayed.

  3. Modify the file as needed.

    In our example, the holes on top and bottom of the object have been enlarged and shifted to the right:

  4. Select File > Save.

    As you have been assigned the Save right, the file is saved with the same name and in the same electronic envelope.

  5. Select File > Save As....

    As you have been assigned neither the SaveAsNewRight, nor the SaveAsKeepRight right, the use of the Save As command is forbidden on this file and an error message is displayed.
  • If Save and SaveAsKeepRight are assigned, you can read the cipher file, modify it, and run the Save or the Save As... command. The security is guaranteed because the new document is saved in the original electronic envelope.
  • If Save and SaveAsNewRight are assigned, you can read the cipher file, modify it and run the Save or the Save As... command. The new document is saved either in a new envelope, or in no envelope at all (depending on the user settings) and in that case, a security breach might occur.
