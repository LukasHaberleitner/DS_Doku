Managing Representations

After a CATIA V5  >  ENOVIA LCA and then an ENOVIA LCA  >  CATIA V5 migration,  this task will show you what are the results in the Managing Representations dialog box: Shapes (CATShape, cgr, model) or CATParts are listed.
 

  1. Create PRC1 in ENOVIA LCA and send it to CATIA V5.

  2. In order to create design in CATIA V5, add:

    • Part1 under PRC1,
    • a Pad under Part1.
    		 You have this assembly in CATIA V5:
    PRC1
    --Part1
    ----Pad    		  

  1. Save this assembly in ENOVIA LCA.

  2. In the Product Editor, under PRC1, select Part1 and insert a CATPart Document (call it Part2) and 2 cgr (cgr1 and cgr2). You obtain:

    There is no link between these new Representations (CATPart and 2 cgr) and CATIA V5.

  3. Select the Part2 Document and send this PRC to CATIA V5. The result is:

  4. Click the Manage Representations icon or right-click Part1 (Part1.1) and select the Representations  >  Manage Representations... contextual command.

    Note that there is only the CATPart document listed in the dialog box, the cgr cannot be seen.

  5. In ENOVIA LCA, select cgr1 and the Send to CATIA V5 command. You obtain:

    Part1 is a cgr.

  6. Select Part1 and open the Manage Representations window, both cgr documents are listed in the dialog box, but not the CATPart. A "Replace" operation has been performed: the CATPart is replaced by the selected cgr.

    If a cgr is sent from ENOVIA LCA to CATIA V5, CATIA V5 looks for all the other Shapes that are attached to the same assembly node.

  7. Select the document Part2.CATPart and send the assembly into CATIA V5; you can see again:

    The cgr has been replaced by the Part and you can visualize Part2 in Manage Representations.

In VPM Navigator, when opening a part instance in CATIA, with open as CGR option, a CATProduct is created on the fly to aggregate the CGR (to be able to display it in CATIA V5). Consequently, through File > Desk... command in CATIA an additional document is seen in the assembly (this document is not a part of the assembly and this document/PV does not exist in DB).
