Electrical  

This tab lets you define the migration mode:

  • for the bundle segments
    see also migration option disabled.
  • for geometrical bundle.
  • for shell migration.
  • for supports.

Bundle Segment Migration Mode

 

Bundle segments

If this option is selected, it means that each V4 BNS is migrated to a V5 bundle segment. Each V5 bundle segment is stored in a CATPart document. They are instantiated in the geometrical bundle, a CATProduct document.

Multi-branchable document

If this option is selected, it means that all the BNSs of a V4 GBN are migrated to one multi-branchable document in a V5 geometrical bundle. All the V5 bundle segments are stored in a single CATPart, the multi-branchable document.
In case there are several V4 GBNs to be migrated, as many V5 geometrical bundles (CATProduct documents) are created, with one multi-branchable document (CATPart) containing all the bundle segments.

It is the default value.

It is recommended to use this option as it reduces the number of CATPart documents generated during the migration.

Migration Options Disabled

The electrical migration options can be disabled if the Interface Name parameter is not filled up with the CATIE3DMigration value in the Migration Batch tab.
Refer to Migration Interface.

Geometrical Bundle Migration Mode

Component

If this option is selected, each GBN is migrated as a component under the root product.

Product

If this option is selected, each GBN is migrated as a product in a separate CATProduct.

By default, this option is selected.
Choosing the right migration mode option is important since it determines how geometrical bundles will be stored in ENOVIA LCA:
  • Migrating them as products lets you store them in ENOVIA LCA using Workpackage mode
  • Migrating them as components lets you store them in ENOVIA LCA using Explode mode.

Shell Migration Mode

Yes, Migrate Technology

If this option is selected, the V4 shells are extended as electrical parts.

By default, this option is selected.

No, Migrate Geometry Only

If this option is selected, only the geometry of the V4 shells is migrated.

The name given to the migrated non-electrical part is the same as for any standard part.

Support Instance Migration Mode

 

Migrate support to common root

Migrates supports and place them under the common root product of the GBNs whose branches or segments are using these supports for routing.

By default, this option is selected. 

Migrate support to first found GBN

Migrates supports and place them under the first GBN found in the root product. It is must that this GBN has the branches and segments that are using these supports for routing.

By default, this option is cleared.
