Setting Up the File Collaboration Server


This topic describes how to setup the File Collaboration Server (FCS).

Before you begin

 

Ensure that the FCS has been installed. See Installing File Collaboration Server.
About Setting Up the SDAM Environment. See Setting Up the SDAM Environment.

Create DB Tables Required for FCS Metadata Server

 

During the ENOVIA V5 installation and setup, the DB and the tables that are created are related to ENOVIA V5. You can use the same DB for FCS tables.

The following tables are required for managing the FCS metadata:
  • FILES
  • INODE
  • LOCATION
  • PHYSICAL
  • SITE
  • STORE
  • USERNAME

You can use the following scripts to create tables:
  • ORACLE: <CATInstallPath>\reffiles\DBMS\ddl\VPMV5_FCS_Tables.sql
  • DB2: <CATInstallPath>\reffiles\DBMS\ddl\VPMV5_FCS_Tables.clp

Note: Before running the DB scripts, modify them with table owner and table space used for ENOVIA V5 DB.

Run the DB Scripts

 

  This section describes how to run the DB scripts.
  1. Connect to DB.
    For oracle, set ORACLE_SID=<DB NAME> on terminal or command prompt
  2. Login to the SQL console as DB Administrator.

    3. For example, conn sys as sysdba
  3. Run the script:
    4. @<CATInstallPath>\reffiles\DBMS\ddl\VPMV5_FCS_Tables.sql

Create Default Store

 

You can create the default store, which points to the URL of FCS metadata server.
 
  1. To create a store, use the following SQL command:

    2. INSERT INTO STORE (NAME,PATH,URL,PROTOCOL) values (‘V5FCSSTORE',<Physical location>,<FCS Server URL>,'file');
  2. In SQL, replace the following values as per the setup:
    V5FCSSTORE is the name of the default store.
    Physical location – Create a directory to store the physical files and provide the value for <Physical location>
    3. .
    For example – ‘C:\DassaultSystemes\R2022x\FCS\FCSVault’
    FCS Server URL – URL of the previously installed FCS server
    For example, http://<hostname><port for FCS>/fcs

Assign Specific FCS Server to the User (Optional)

 

You can assign a specific FCS server to the user.
 

The following SQL queries are used to assign specific FCS server to the user as per user's site configuration. Ensure that the FCS server used in this step is different than the FCS server used in the default store.

  • Create a store:
    INSERT INTO STORE (NAME,PATH,URL,PROTOCOL) values ('SITEASTORE',<Physical location>,<FCS Server URL>,'file');
  • Create site
    INSERT INTO SITE (NAME) values ('Site_A');
  • Add user to the 'Site A' – USER1
    INSERT INTO USERNAME (NAME, SITE) values ('USER1', 'Site_A');
  • Assign FCS Server to site 'Site A'
    INSERT INTO LOCATION (NAME,PATH,URL,PROTOCOL,STORE,SITE) values ('Location_A', <Physical location> ,<FCS Server URL>,'file','SITEASTORE', 'Site_A');

    Where,
    Physical location – Create a directory to store the physical files and provide the value for <Physical location>.
    For example – ‘C:\DassaultSystemes\R2022x\FCS\FCSVault’
    FCS Server URL – URL of the previously installed FCS server.
    For example, http://<hostname><port for FCS>/fcs

Configure FCS Server to Access DB

 

You can configure the FCS server, which provides access to DB.
 
  • Copy DB vendor (Oracle/DB2) specific jdbc driver jar to either of the following paths:
    • <SDAM middleware_Install_DIR>\webapps\V5SDAMClientServ\WEB-INF\lib\
    • <SDAM middleware_Install_DIR>\lib
      Where,
      <SDAM middleware_Install_DIR> - Deployment directory of V5SDAMClientServ.war
  • Configure FCS metadata server for DB connection.
    • Modify dbinfo.txt file located at <WEB-INF\classes> directory of WebServer and add following details:

      For Oracle:
      jdbcurl=jdbc:oracle:thin:@<SDAMServerInstallationMachine>:<ListenPort>:<DBNAME>
      user=<fcs_admin>
      password=<fcs_admin_password>
      DB=oracle

      For DB2:
      jdbcurl=jdbc:db2://<SDAMServerInstallationMachine>:<TCP Port for DB2 service>/<DB Name>
      user=<fcs_admin>
      password=<fcs_admin_password>
      DB=DB2 or db2 or ibm
