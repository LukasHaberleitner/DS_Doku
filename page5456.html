Defining Design Composites Parameters with Power Copies

This task shows you how to quickly reuse and edit materials, rosettes, laminates, ramp definitions or the whole Composites Parameters from another document into the current one.
Available in Composites Engineering Design (CPE), Composites Design for Manufacturing (CPM) and Composites Braiding (CPB).

  1. Go to Knowledgware > Product Knowledge Template.

  2. Create the Power Copy:
    • In the source document, select Create Power Copy from the toolbar.
    • In the Definition tab, select the element of which you want to make a Power Copy.
    • Click OK to validate and exit.
      The Power Copy is created in a dedicated node in the specifications tree of the source document.

  3. Instantiate the Power Copy from a selection.
    • Open the source and the target documents.
    • From the specifications tree of the source document, select the Power Copy to instantiate.
    • In the target document, select Instantiate from Selection from the toolbar.
    • In the dialog box, select the destination and the input, and validate.
    The Power Copy is instantiated in the target document.

  4. Alternatively, instantiate the Power Copy from a document.
    • Make sure the document containing the Power Copy is closed.
    • Open the target document and select Instantiate from Document from the toolbar.
    • Browse the directories and select the source document.
    • From the Reference list in the dialog box, select the Power Copy to instantiate.
    • Select the destination and inputs, and validate.
    The Power Copy is instantiated in the target document.

     

    Power Copy features can be created for a whole set of Composites Parameters or for individual ones (Materials, rosettes, laminates, ramp definitions) but not for Main Stacking Sequences.

    Instantiation is done as follows:

    • Instantiation of the whole set of Composites Parameters: They are created with replacing input parameters as selected by you.
      All the individual parameters are created similar to the source Composites parameters.

      Note: Instantiation of the whole set of Composites Parameters is not possible if a set of Composites Parameters already exists in the document.

    • Instantiation of individual parameters requires that a set of Composites Parameters already exists in the document.
    • For materials:
      • If the name of the instantiated material is not already present in the current document, the material is created with this name.
      • If the name of the instantiated material is already present in the current document (For example Carbon 0.4), the material is created
        with its name changed to <materialname>(Number of Occurrence). In our example, Carbon 0.4(2)
    • For rosettes:
      • You must select the mandatory inputs (Axis System and transfer curve if the source rosette is defined with 0/90 degree guided by the curve).
      • The rosette is instantiated in the existing Composites Parameters, similar to the source rosette but with the replacing mandatory inputs.
    • For ramp definitions:
      • You must select the mandatory inputs (For example geometry selected for clearance geometry field, or start and end curves for Between two curves type of ramp definitions).
      • The ramp definition is instantiated in the existing Composites Parameters, similar to the source ramp definition but with the replacing mandatory inputs.
      • If an automatic name is set on the source ramp definition, the name of the instantiated ramp definition is modified accordingly.
    • For laminates:
      • You must select the mandatory inputs (Materials and orientations).
      • If the name of the instantiated laminate is not already present in the current document, the laminate is created with this name.
      • If the name of the instantiated laminate is already present in the current document (For example TL.1), the laminate is created
        with its name changed to <Laminatename>(Number of Occurrence). In our example, TL.1(2).
      • Power Copy of a source laminate that contains a MSS layer information is created without the MSS Layer information.
        The laminate is instantiated by replacing the material and orientation corresponding to your selection and with its type set as Stacking Sequence (SS).
    • Orientations are not supported, We recommend you create them.
    • For Main Stacking Sequences:
      • They are not supported. We recommend you export the Main Stacking Sequence from the source document and import it in the current one.
      • MSS layer information associated to the laminate in SS-MSS mode are lost during the power copy of laminates.
        Tip: Use the Power Copy of a Composites Parameters feature in a document without existing Composites Parameters.
