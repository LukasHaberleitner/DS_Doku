Exporting Data (as IDF)    

This task shows how to export data into an IDF file.
A CATProduct document is open containing at least a board or a panel and optionally components.

  1. Click Export as IDF File.

    The Export dialog box opens.

  2. Click .

    The Idf File dialog box opens.

  3. Enter Export for example in the File name field.

    You can choose between three extensions for the IDF type:

    • .idf: the general format
    • .brd: the MG specific format
    • .bdf: the Allegro specific format

  4. Click Open.

    You are back in the Export dialog box.

    You can choose between two unit systems:

    • MM: millimeters
    • THOU: 1/1000 of inch

    and the two IDF versions:

    • IDF 2.0
    • IDF 3.0

    Another option is available: Use a CSV file.
    Refer to Using a csv File in Working with Catalogs section.

  5. Select the Decimal Symbol to be used for the export.

       The Decimal Symbol makes the export totally independent of the language of the session.

  6. Click OK to perform the export.
    An Information message is displayed:


    Note: Only features created with Circuit Board Design commands can be exported in an IDF file.
     

To be exported to an IDF file, the document must contain at least a board or a panel. Other objects are also exported:

  • components
  • constraint areas including multi-domain ones
  • holes
  • cutouts
  • patterns of holes or cutouts including patterns of pattern (one level).
  • It is possible to export CBD objects built on a pad based on 3D curves. However, if the board pad is based on a 3D curve, the export is possible only if an axis system has been defined as the origin of the board.
  • You can export patterned pads to an IDF file. However, merged constraint areas are not supported during export.
