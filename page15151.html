Create Replication Package

This task shows you how to create a replication package for the objects in VPM Navigator, that you are going to send to external sites.
In VPM Navigator, the Create Replication Package can only be done from the VPM Navigator window and the Search Results dialog box. 

In this example the VPM Navigator will be used.

  1. Open the object of which you want to create the replication package in VPM Navigator.

  2. Select the object (product or a part instance) in VPM Navigator and click Create Replication Package in the ENOVIA V5 VPM Replication toolbar. The Replication Package Creation dialog box appears.

  3. In the Replication Package Creation dialog box:

    • In the Definition section

      • Type the name of the Replication Package in the Name box.

      • Select the target site(s) from the Target Site list.

    • In Capture Mode section

      • Select the capture mode as per your requirements. For example in this case, the Selected Object Only capture mode is selected.
        Refer to the Capture Modes document in the VPM Product Editor Administration Guide section of the LifeCycle Administration Guide for available capture modes.

    • In the Options section

      • Select how often you want the data to be replicated. Select Replicate at Each Scheduled Replication, if you want the data sent with every extraction or select Replicate at Only Once if you want the data sent during the next extraction only.

      • Optionally, you can select the Transfer Site Ownership, if you want the ownership of the objects to be given to the target site. This will allow the target site to edit the data being sent. The source site will no longer be allowed to modify it.
        Note: Transfer Site Ownership option is available only with Replicate at Only Once option.

  4. Click OK. The replication package is created. You can look for the replication package in Replication Package Manager, or in VPM Lifecycle Navigator. Refer to Multi-Site System Administration Web book for more details.
