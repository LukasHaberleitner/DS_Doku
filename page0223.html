Creating Submodel Properties

A submodel property refers to the output database that contains the response of a global model that will be used to drive the submodel. The driven variables can be displacements or temperatures. You must create a submodel property before you analyze a submodel. See Using Submodeling for an overview of submodeling with Abaqus for CATIA V5. Submodeling is described in detail in About submodeling in the Abaqus Analysis Guide.

This task shows you how to create a submodel property.

  1. Click the Submodel Property icon .

    The Submodel Property dialog box appears, and a submodel property object appears in the specification tree under the Nonlinear and Thermal Properties objects set.

  2. You can change the submodel property identifier by editing the Name field. This name will be used in the specification tree.

  3. In the Global Job field, click the Browse button to choose the output database file that was generated by the analysis of the global model. Abaqus for CATIA V5 drives the boundaries of the submodel with time-dependent variables from this output database file. In addition, a part (.prt) file with the same name must be present in the same directory as the output database file. In most cases the files will be located in the Computation and Results Data directory that you specified under Tools>Options. See Configuring an Abaqus Analysis for more information.

  4. Click OK in the Submodel Property dialog box.
