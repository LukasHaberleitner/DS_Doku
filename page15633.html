Working with Design Tables in ENOVIA V5 VPM Using VPM Navigator

It is possible to save design tables in ENOVIA and to apply design tables saved in ENOVIA to existing parts using VPM Navigator.

Note that:

  • To modify a design table saved in ENOVIA, the sheet must be synchronized with the design table saved in ENOVIA. To do so, right-click the design table in CATIA and select Synchronize (ENOVIA). The design table must also be locked.
  • To share a design table, see Saving a Design Table in ENOVIA V5 VPM Using VPM Navigator.

See also Working with Design Tables in ENOVIA V5 VPM.

When Saving a Design Table in ENOVIA V5 VPM, the design table for a CATPart or CATProduct is automatically attached to the part.

  • The design table is attached to the part reference with a representation link as a supplementary document.
  • Enabling this feature allows you to easily work with the design table in VPM Navigator since it will be displayed in the VPM Navigator tree. Design table documents associated to children of publication exposed assemblies will not be displayed, but the ones attached to the assembly itself will be seen.
  • The attached document will have the same functionality as similar documents in VPM Navigator.
  • Since the association between the part and the document is established during the save from CATIA V5, only design tables created with this feature activated will be automatically associated to the corresponding part. Design tables created prior to this must be associated to their corresponding part manually.
  • In CATIA V5, Edit Links > Isolate (see the CATIA Infrastructure User Guide > Editing Document Links for more information on the Edit Links command) cannot be used to remove the design table's representation link to the part.
  • Design tables will not be revisioned when saving to ENOVIA V5 VPM from CATIA V5 using the Revision All option.
