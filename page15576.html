Managing CATDrawing Documents in ENOVIA LCA

This task shows you how to manage the integration between CATIA V5 Drafting and ENOVIA LCA. You will learn how to:
  • create a CATDrawing document and save it in ENOVIA LCA
  • retrieve the CATDrawing document in ENOVIA LCA
  • view the documents which impact the CATDrawing in ENOVIA LCA
  • open the CATDrawing document in CATIA V5
  • create a CATDrawing document from scenes saved in ENOVIA LCA

This task uses an assembly drawing (i.e. a drawing with views generated from a CATProduct document) as an example, but the same possibilities apply to part drawings as well.
   
For more information about interoperability between CATIA and ENOVIA LCA, refer to the ENOVIA-CATIA Interoperability User's Guide. For more information on using ENOVIA LCA, refer to ENOVIA V5 LifeCycle Applications documentation.
   

Have a CATIA V5 and an ENOVIA LCA sessions running.
     
 

Creating a CATDrawing Document and Saving it in ENOVIA LCA

  1. In ENOVIA LCA Product Class View, select a product and send it to CATIA V5.
    For more information, refer to Sending an ENOVIA LCA Document to CATIA V5 in the ENOVIA-CATIA Interoperability User's Guide.

  2. In CATIA V5, select Start > Mechanical Design > Drafting. The New Drawing Creation dialog box is displayed.

  3. Select the views to be automatically created on your drawing, for example the Front, Top and Left icon.

  1. Click OK. A progress bar appears while the views are being generated from the opened CATProduct.

The views are then displayed in the CATDrawing document.

  1. Click Save Data in ENOVIA LCA Server... to save the drawing in ENOVIA LCA. The Save in ENOVIA LCA dialog box is displayed.
    For more information, refer to Saving an ENOVIA LCA Document from CATIA V5 in the ENOVIA-CATIA Interoperability User's Guide.

  2. Click OK to validate. The drawing as well as the parts and products from which the views were generated are vaulted in ENOVIA. Note that the drawing is saved as a document in ENOVIA, it is not attached to a part at this moment.

Retrieving the CATDrawing Document in ENOVIA LCA

  1. In ENOVIA LCA, click Search ENOVIA Tree in the left-hand pane to perform a query in order to retrieve your CATDrawing document.

  2. Select Content Management > Documents > Document. The Search dialog box is displayed.

  3. In the Search dialog box, enter the name of your CATDrawing document and click OK. The search results are displayed in the bottom pane.

  4. Right-click the CATDrawing document and select Copy.

  5. In ENOVIA LCA Product Editor paste the document under the PRC.
 

Viewing the Documents which Impact the CATDrawing in ENOVIA LCA
The Impact Graph is updated only if the CATDrawings have been saved in CATIA V5.

  1. In ENOVIA LCA Content Management, right-click the CATDrawing document (it can be either a part drawing or an assembly drawing) and select Send To > Impacted By. The Impacted By Tree View is displayed.

  2. Double-click the CATDrawing document (here Drawing1) to expand the whole view of the product.

The drawing dependencies are displayed, letting you view the links between the CATDrawing document and the CATPart and/or CATProduct documents it references. This also lets you know whether the drawing is up-to-date or not.

In the examples below, Drawing1 identifies the drawing, and the object identifies the drawing dependences.
 

Example 1

The thumb relates to the status of the branch: here the thumbs are up and green, which means that Prt01 and property of Prt01 are synchronized with the Drawing1 object.

Since all elements are synchronized, the drawing is up-to-date.
 

 

Example 2

The red thumb down for Prt01 shows that it has been modified: it is not synchronized with the Drawing1 object. 

Since not all elements are synchronized, the drawing is not up-to-date.
 

  1. If the drawing is not up-to-date, you can update it using the following method:

    1. Open the CATDrawing document in CATIA V5 (as described in Opening the CATDrawing document in CATIA V5 below).

    2. Click Update to refresh it.

    3. Click Save Data in ENOVIA LCA Server... to save the document.

    4. Click Refresh to perform a local refresh in ENOVIA LCA.

    If you now go back to the Impacted By Tree View in ENOVIA LCA, the drawing is indicated as being up-to-date.
For more information, refer to Publishing Links in ENOVIA LCA in the ENOVIA-CATIA Interoperability User's Guide.

Opening the CATDrawing Document in CATIA V5

To be able to open only the CATDrawing document in CATIA V5, you need to send it from ENOVIA Content Management for example.
To illustrate this task, you should use an assembly drawing (rather than a part drawing).

  1. In ENOVIA LCA Product Editor, right-click the CATDrawing document and select Send To > Content Management.

  2. In Content Management, right-click the CATDrawing document and select Send To > CATIA V5. The drawing is displayed in CATIA V5. In the case of an assembly drawing, the drawing links are broken.

  3. To solve the links, perform as follows, depending on the save mode you chose when saving the assembly in ENOVIA LCA:

    • For an exploded assembly (i.e. saved as Document not kept & Product Structure Exposed):  right-click the drawing in the specification tree and select Load PDM Context (you can also go back to ENOVIA LCA, right-click the associated CATProduct and CATPart documents in Content Management, and select Send To > CATIA V5). This loads the assembly drawing context (i.e. the CATProduct and CATPart documents used for the assembly drawing creation or for the last assembly drawing update) in the Product Structure editor in CATIA V5. Note that the part instances, which were not referenced by the drawing when you last saved it are not loaded.
    • For a black box or work package assembly (i.e. saved as Document kept in vault & Publications Exposed): select Edit > Links... and choose the Pointed documents tab to visualize the links between the CATProduct and CATDrawing documents. From the list of pointed documents, select the CATProduct and/or CATPart documents you want to open, and click the Open button. This loads the selected documents in the Product Structure Editor in CATIA V5.
      Note: The Publications Exposed mode works similar to that of file-based mode. That is, when you open the CATDrawing, the assembly will be automatically loaded, if the Load referenced documents check box is selected in the General tab, under the Referenced Documents area, available through Tools > Options > General. For more information, refer to Infrastructure User's Guide.

  • The LoadPDM command does not load the technological package referenced by the drawing. The referenced technological packages can be listed via the Edit Links command, and have to be opened from LCA sessions.

  • A drawing references technological packages if it contains views created by selecting FT&A views stored in a technological package (View from 3D command).

Note that you need to load all the parts, which make up a given product if you want them to be included in the drawing views at the next update.
 
  • The drawing links are now solved. The Drafting specification tree indicates whether the drawing is up-to-date or not, whether the drawing is a part drawing or an assembly drawing.
 

  • If the New Evolution and Replace Reference commands have been performed on a part or sub-assembly, the drawing links will not be solved by default on the new evolution of the part. However, you can force the solving of the links on this new evolution by going to Edit > Links > Replace and selecting the root product (PRC).

Creating a CATDrawing from Scenes Saved in ENOVIA LCA (Technological Packages)

   

To be able to create CATDrawing from scenes saved in Technological Packages, save the scene in ENOVIA.
To illustrate this task, you should use an assembly drawing (rather than a part drawing) with a scenes created and saved in ENOVIA.
 
  • View creation
    • For creating views saved in technological package, you need to follow the same procedure as selecting scenes in file based environment and further creating the CATDrawing.
    • It is possible to create all the generative views (front view, isometric view, advanced front view, projection views and detail views).
    • Views of different scenes associated to the same product can be created.
    • Views of scenes and views of the whole PRC can be created in the same CATDrawing, if the scenes are associated to the PRC.
    • Views of scenes and views of sub-parts of the PRC cannot be created in the same CATDrawing.
    • All generated views in the drawing must reference the same 3D exploded product.
  • Annotation and dimension
    • The annotations and dimensions created are associative. They follow the 3D changes when updating the generative views. When creating or updating annotations and dimensions, the 3D transformations (translation, rotation) applied on instances in the scene are taken into account.
  • Overload properties
    • You can overload the properties of elements in the generative views pointing to scenes saved in the technological package.
    • All the overload properties are taken into account when updating or visualizing the views.
  • Opening drawings
    • For more information about opening the drawings, refer to Opening the CATDrawing document in CATIA V5.
  • Edit links
    • Drawing links are broken till the time the technological package is not loaded.
      After the 3D product is loaded, you can automatically load the technological package using the Edit > Links command from the main menu.
      For more information about editing links, refer to Editing Drawing Links in Generative Drafting User's guide.
  • Modify links
    • You can use the Modify Links command to remove the link to the scene in order to represent the whole assembly. It is possible to change the link from one scene to another contained in the same assembly.
      For more information about modifying links, refer to Modifying a View's Links in Generative Drafting User's guide.
  • Updating views
    • For more information about updating generative views, refer to section on updating drawing in Opening the CATDrawing document in CATIA V5.
