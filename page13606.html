Customizing Second XML Format Type Options

  1. Select the XML Export for clash process purpose check box in the DMU Clash - Process tab page (Tools > Options > Digital Mockup > DMU Space Analysis).

  2. In the Data base repository path, specify a relative path, for example:
    E:\users\Export_XML_Clash\XML

  3. In the Style sheet location path, specify a relative path, for example:

    file:///E:\BXX\intel_a\resources\graphic\UnifiedStyleSheet_xx.xsl

    You can choose to export clash results and geometry to:

    • One XML file per conflict plus one header file summarizing results (Federated XML output checkbox selected. This is the default option) or

    • A single large XML file containing all generated conflicts (Federated XML output check box)

    Two new results visualization options are now available:

    • Zoom on interference

    • Transparent background

    Selecting this check box displays a transparent background for pictures retrieved in the XML clash results file.

    By default, this check box is selected.

    • Aggregate files in folders

    A new option to support long filenames is now available:

    • Support for long filenames

    Selecting or clearing this check box modifies the name of the generated xml file is as follows:

    If selected

    Name_used_date_time_interferenceID (e.g.,Federated_ClashPublish_17_9_2007_15_49_17)

    If cleared

    Name of the first object involved in the interference analysis ++ name of the second object involved in the interference analysis.

    By default, this check box is selected.

    Clearing this check box

    A new option to classify your clash result files is now available:

    • Aggregate files in folders

    Selecting this check box creates a dedicated folder in which all the clash result related files (i.e. .xml, .jpg, .xsl, .dtd, etc.) are stored.

    By default, this check box is selected.

    • To have your browser open automatically showing the exported results, ensure your browser is already open on your desktop.

    • Note that on Unix, the Browser automatically opened option is not available, refer to Recommendations

    • Important:  When the XML Export for clash process purpose check box is selected, use exclusively the UnifiedStyleSheet_xx.xsl dedicated to the second XML.

    • On the contrary, clash.xsl is the style sheet to be exclusively used when the XML Export for clash process purpose check box is cleared.

  4. Click Export As in the Check Clash dialog box.

  5. Click OK in the Save As dialog box, XML is correctly created.
    Using DLNames

    If you set the DLName document environment (Tools > Options > General > Document) as your current environment, clicking Export As will open the DLName dialog box instead of the usual Export As dialog box.

    The DLName document environment lets you restrict the access to specific folders referenced by logical names referred to as DLNames. Each folder is assigned a logical name. In this mode, you can only access documents in folders referenced by DLNames.

Filtering Clash Results

If you choose to export clash results to a single large XML file (Federated XML output check box cleared), you can sort the display of computation results in the file. A Filter options entry under Computation Result lets you do so.

  1. Click Filter options to expand and show the form

  2. Set filter options

  3. Click Filter.
    Note that when Second XML Format Type (export for clash process purposes) is set, filters are embedded in the style sheet so that you can filter you clash results directly in the exported XML file (you do not filter clash results before the export).
    This only works when Federated XML output check box is cleared.

XML Grammar

Federated XML Output

You can now use the following characters <, >, &, /., : These characters are supported when exporting results to XML files.

To support special characters the XML grammar has been changed as follows:
  <LinkToInterference Link = "DataBase\wheel#62#60#42#47#92#63#34#124_ (wheel.1) -- Shape 1++wheel#63 (axle.1) -- Shape 1++1.xml" LinkTitle = "DataBase\wheel&gt;&lt;*/\?&quot;|_ (wheel.1) -- Shape 1++wheel? (axle.1) -- Shape 1++1.xml" FirstProductURL = "Picture/28_2_2007_21_6_4_Preview_First_1.png" SecondProductURL = "Picture/28_2_2007_21_6_4_Preview_Second_1.png"/>
  • The Link tag always contains the real path to the sub-XML file (the pointed file) as in the previous XML versions.
  • The new LinkTitle tag contains the hyperlink name.

Refer to Reading XML Clash Files For more detailed information about the XML grammar

 

Exporting Clash Results - Example

  1. Set XML export options in the DMU Clash - Process tab page (Tools > Options > Digital Mockup > DMU Space Analysis).

  2. Select Insert > Clash from the menu bar or click Clash in the DMU Space Analysis toolbar to run a check of type Clearance (3mm) + Contact + Clash between the valve and all the other products in the document (Selection against all).

    The Check Clash dialog box expands to show the results of the initial computation. 

  3. Browse through results using dialog box commands and/or the Results viewer.

  4. Click Export As in the Check Clash dialog box.

    The Save As dialog box appears. It is set to export results and geometry in XML format to the database repository identified in your settings.

  5. Click OK in the Save As dialog box when done.

    A dialog box informs you of the success of the operation. All the necessary files and images are created in corresponding folders.

    By default (Federated XML output), one header XML file summarizing clash results and as many XML files as conflicts are generated in the single database folder or repository.
     

    One PNG (Portable Network Graphics) image per conflict and two small images, each showing one of the products in conflict, are also generated and stored in a dedicated picture folder:

  6. Open your browser and read the summary in the header XML file.

  7. Select an interference (computation result table) to jump to the file giving information about the conflict of interest.

    Notes: 

    • The viewpoints set when visualizing the interferences in the Results window (not the Preview) are saved and exported along with the clash results. If no viewpoint is set, there is an automatic zoom on the interference.

  8. Click OK in the Check Clash dialog box to exit when done.
