Releasing CATIA Parts, Drawings and Assemblies

The Release operation updates files and releases them, including all mapped title blocks and revision blocks in drawings.

A Release operation may be performed in either of the following ways:

  • Releasing a checked-out document
  • Releasing a checked-in document
  

Releasing a checked-out file

  1. Open a drawing, part or assembly in CATIA.

  2. Check out the document if it is not already checked out.

  3. From the SmarTeam menu, select Lifecycle> Release operation.
    The Lifecycle dialog opens.

  4. Click OK.
    The document is released. The document status is automatically changed to Released in the State field. When releasing an assembly, the assembly and all of is components are removed from the work directory.

 

Releasing a checked-in document

  1. Open a checked-in drawing, part or assembly in CATIA (in Read-only mode).

  2. From the SmarTeam menu, select Lifecycle> Release operation.
    The Lifecycle dialog opens.

  3. Click OK.
    The document is released.

Note: When releasing files from a checked-in state, local files are not removed from the work directory. You must remove these remaining components from the work directory using SmarTeam File Explorer.

 

Releasing a checked-in document using Job Server

If Job Server is implemented on your system, CATIA documents can be released directly from SmarTeam - Editor without the need for a CATIA installation on your machine. The updating of the CATIA files is performed on the Job Server machine.  For more information, refer to the Silent Release section in the Job Server Administration Guide.
