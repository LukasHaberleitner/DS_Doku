Filtering Product Annotations 

This task shows you how to filter product annotations.
You can filter out 3D annotations at any product level in a product structure, which are pointing to any broken, unloaded or deactivated part/product instances. To achieve this, you need to select Do not load annotations with missing related geometric elements in Tools > Options > Mechanical Design > Functional Tolerancing and Annotation > Display tab, Annotations in Assembly Context area.
Open an existing product containing at least two parts as shown:

  1. Select the Start > Mechanical Design > Product Functional Tolerancing & Annotation command to launch the workbench.
    The Product Functional Tolerancing and Annotation workbench is opened.

  1. Create a cumulated dimension between the two parts.

  2. Create a flag note on Part2.

  1. Right-click Part2 in the specification tree and select Part2.1 object > Activate / Deactivate component.
    The part is deactivated and not shown in the 3D graphic area. The annotations however are displayed.

  2. Click Update Unresolved FTA Features Filter in the Visualization toolbar.

    This command is not available by default in the Visualization toolbar. It is enabled dynamically during the session in the following cases:
    • Load/Unload of part/product instance.
    • Activate/Deactivate of part/product instance.
    • Activation/deactivation of Do not load annotations with missing related geometric elements option.
    • Change of filtering mode option while the Do not load annotations with missing related geometric elements  option is active.
     

    All impacted product FTA features are checked and, according to the state of pointed part/product instance geometry and the filtering mode, the corresponding annotations are restored or are filtered.

More About Filtering Product Annotations

When you filter an FTA feature, it behaves as if it does not exist (or it has been deleted).

  •  It is neither displayed in 3D nor in the feature tree.
  •  It is not displayed in the annotation set properties.
  •  It is not taken into account by semantic checking rules.
  •  It is ignored by the Edit > Search, Filter, Annotations > Select All (in the contextual menu of a view) and Display Capture (in the contextual menu of a capture) commands.
  •  It is not exported in CGR, 3DXML, STEP formats.
  •  It is not generated in drawings at view creation (View from 3D command) or at view update time.

This applies to all product FTA feature types except captures and views.

The filtering does not modify the actual product or the data. It is only session dependent.

Update Unresolved FTA Features Filter command:

When the option Do not load annotations with missing related geometric elements is selected, the product FTA features are filtered according to the selected filter mode (At least one related geometric element is missing or All related geometric elements are missing).
The command Update Unresolved FTA Features Filter becomes available (enabled) in the following cases:

  • In case of broken part/product instance(s) (since its corresponding document is not loaded), when using File > Desk or Edit > Links if a missing part document or the product document is replaced or reloaded.
  • When part/ product instance(s) that is(are) deactivated. 
  • Part instance(s) whose representation(s) is(are) deactivated. 
  • Part/product instance(s) whose document(s) is(are) unloaded/loaded.

General Remarks

  • Capture:
    When you create a new capture or edit a capture, the filtered annotations are not considered. On already existing capture, if Display Capture is performed then the filtered annotations are not displayed.
  • Display View:
    When you run Display View command on an existing view with filtered annotations, it will only display unfiltered product FTA features and make the view normal to screen.
  • Annotations > Select All:
    During this command, the filtered annotations are not considered.
  • Delete:
    When you run the command Delete on a view containing at lease one filtered annotation, you get a message informing that delete cannot be performed until all the annotations are restored.
