Requirements

  • Please note that all MULTICAx Plug-ins are on a dedicated media. You need this media to install them.
  • Never modify data generated in V5: information could be lost and the MULTICAD Update could result in incorrect results.
  • Part Number must be unique, all parts with the same name are considered as identical.
  • On Windows, make sure the files to translate are not encrypted: push the Advanced button in the Properties dialog box of the file and uncheck the Encrypt contents to secure data option.
  The MultiCAx SolidWorks (S) Plug-in is supported on Windows 32 bits and 64 bits.

The MULTICAx SolidWorks (S) Plug-in allows you to import:

  • SolidWorks native data (part and assembly models).
    This import converts 
    • SolidWorks part geometry into the CATIA Graphical Representation (CGR) or CATPart formats, which renders it readable by V5. 
    • SolidWorks assembly files into a V5 product structure document.

    You can perform this import interactively or from the command line.
  • Parasolid formats (*.x_t and *.x_b): parts only.
  • VDA-FS data (*.vda): parts only.
  When importing SolidWorks part geometry, MULTICAx S Plug-in converts all solid bodies belonging to visible layers in the  SolidWorks part into the CGR-faceted form, with user-specified tessellation parameters. In  addition, parts can optionally be imported in the CATPart data format, which allows access to geometry. Coordinate systems can also be retrieved in the generated CATPart files. Any part user attributes defined in the original part are stored in the CATProduct document.

The assembly import converts the SolidWorks assembly hierarchy into product structure, along with the proper positions. No attributes of the assembly other than hierarchy, positions, and colors of the geometry are addressed in assembly import.

Protocols and attributes

For conversions, the following protocols or attributes apply:

  • The SolidWorks part files must have the .SLDPRT extension
  • The SolidWorks assembly files must have the .SLDASM extension.
  • The VDA-FS part files must have the .vda extension
  • Parasolid formats must have the .x_t and .x_b extension
  • Length units for both parts and assemblies are converted to millimeters. 
  • Assembly-level colors and other features are not supported. 
  • Pure wireframe data  and coorsys are not supported during the conversion.
  • Text and annotations are not supported.
  • SolidWorks configurations are not supported.
 

Installation requirements

MSXML 6.0 must be installed on your machine before running the converter.
 

Licensing requirements

.sldasm, .sldprt, .x_t, .x_b .xmt_txt, and .xmt_bin extensions are not natively supported in V5.
Therefore, DSL licenses (ENOVIA PORTAL, CATIA) or MSP licenses (DELMIA) are required to access these extensions.
