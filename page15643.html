Importing a Rule Base With Links From a Catalog Saved in ENOVIA V5 VPM  

This task explains how to import a Rule Base with links from a catalog stored in ENOVIA V5 VPM. In this scenario, you:
  • create a Product (called saw) in ENOVIA V5 VPM and sends it to CATIA
  • add components to the Product
  • create a Part containing a RuleBase
  • create a catalog that is linked to the RuleBase and saves it in ENOVIA
  • import the RuleBase into the Product
Make sure you have:
  • selected ENOVIAV5 in the View > Toolbars  menu.
  • made the option ENOVIA as Allowed in the Document Environments field (Tools>Options>General>Document.)
To perform this scenario, you will need the following files:
  • Bearing.CATPart
  • Case.CATPart
  • Cog_wheel.CATPart
  • Ring.CATPart
  • RuleBase_Catalog.CATPart
  • Slide.CATPart

  1. Create a Product in ENOVIA V5 VPM (in this scenario the product is named Expert). Note that you can close ENOVIA V5 VPM after creating the Product.

Send the Product to CATIA

  1. To send the product to CATIA:

    • In CATIA, click the Connect icon ( ) in the ENOVIA V5 toolbar. The ENOVIA Logon dialog box is displayed:
    • Enter your user name and your password.
    • Select your role in the Role scrolling list and the workbook and click OK.
    • Click the ENOVIA Search icon (). The VPM Search dialog box is displayed.
    • In the Objects scrolling list, select the Product option.
    • Enter the owner's ID.
    • Click OK when done. The Product ID is displayed in the Result window.
    • Double-click the product. The VPM Navigator view is displayed.
    • Right-click the root product (Expert) and select Open.... The Open Modes dialog box is displayed. Leave the default settings and click OK. The CATIA view is displayed.

Add Components to the Product

  1. Click the root product, select Insert>Existing Component... and select the following documents:

    • Ring.CATPart
    • Cog_wheel.CATPart
    • Bearing.CATPart
    • Slide.CATPart
    • Case.CATPart

  2. Optional: Access the assembly design workbench to add constraints to the saw.

  3. Save the product in ENOVIA. To do so, click the Save in ENOVIA icon (). The Save in ENOVIA dialog box is displayed. Click OK.

Create a Part Containing Expert Checks

This Part is the one that will be used to create the catalog.

  1. Create an empty Part or open the RuleBase_Catalog.CATPart file.

  2. If you opened RuleBase_Catalog.CATPart, go to step 12. If not, from the Start>Knowledgeware menu, access the Knowledge Expert workbench.

  3. Click the Expert Check icon (). Enter the name of the check (Hole_Check) in the Check Editor and click OK.

  4. Enter the following check, and click OK when done.

    H:Hole
    (H.Diameter == 11.0 mm) AND (H.Activity == TRUE)

  5. Click the Expert Check icon (). Enter the name of the check (Fillet_Check) in the Check Editor and click OK.

  6. Enter the following check, and click OK when done.

    H:ConstantEdgeFillet
    H.Radius >= 2mm

  7. Save the Part in ENOVIA. To do so, click the Save data in ENOVIA V5 icon (). The Save in ENOVIA dialog box is displayed.

  8. Click the button and select the line referring to the RuleBase_Catalog.CATPart and click Modify. Click the root product (Expert)  in the VPM Navigator view and click OK. The Part is saved in ENOVIA.

Create the Catalog Linked to the Rule Base and Store it in ENOVIA V5 VPM

  1. From the Start>Infrastructure menu, access the Catalog Editor workbench.

  2. In the Catalog Editor, select Chapter1, then click the Add Family icon ().
    The Component Family Definition dialog box is displayed. Click OK. The new family is added to the tree.

  3. Double-click the family then click the Add Component icon (). The Description Definition dialog box is displayed.

  4. Click Select external feature then select the Rule Base contained in RuleBase_Catalog.CATPart. A catalog containing your RuleBase is created. Click OK when done.

  5. Save the catalog in ENOVIA. To do so, click the Save data in ENOVIA V5 icon (). The Save in ENOVIA dialog box is displayed.

  6. Click the Catalog name and rename the catalog: RuleBase.catalog.

  7. Select the line referring to this catalog and click Modify. Click the root product (Expert) in the VPM Navigator view and click OK.

  8. Close all windows in CATIA.

Import the RuleBase From the Catalog

  1. Click the Search ENOVIA icon (). The VPM Search dialog box is displayed.

  2. In the Objects scrolling list, select the Product option.

  3. Enter the owner's ID.

  4. Click OK when done. The Product ID is displayed in the Result window.

  5. Double-click the product. The VPM Navigator view is displayed.

  6. Load the product from ENOVIA V5 VPM. To do so, right-click the Parts (see picture opposite), and select Open.... The Open Modes dialog box is displayed.

  7. Check Automatic Lock of all opened parts and documents and click OK. The product is displayed.

  1. Click the Catalog Browser icon () in the Catalog tool bar.

  1. In the Catalog Browser window, click the Browse another catalog () icon. The Document Chooser is displayed.

  2. Click the ENOVIA icon. The Search Conditions dialog box is displayed. Enter the name of the revision owner and click OK.

  1. Select the RuleBase.catalog (RuleBase) in the Search Result window and click Open.

  2. Double-click ComponentFamily, then RuleBase in the Catalog Browser until the RuleBase Catalog window is displayed.

  3. Check the Import with Link option and click OK.

A new technological package is created that contains the RuleBase and the checks.

  1. Right-click the Rulebase and select RuleBase object>Manual Complete Solve to solve the RuleBase. The checks are applied.

  2. Right-click the Rulebase and select RuleBase object>Report... to display the report. Click OK in the Select directory window: The report is displayed.
