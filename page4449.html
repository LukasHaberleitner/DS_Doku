Lock Management and User Rights

A user's ability to load an object, modify it, and then save the changes back to the Manufacturing Hub is determined by two primary factors: 

  • the current read/write status of the object, which is determined by the Lock Management system of DPM 

  • the current user's access rights (referred to as User Rights), as defined in Process Engineer

An object is in a read-only state for a user if that user does not have the appropriate access rights (as defined in Process Engineer), or if another user has already acquired a write-lock on the object of interest.
