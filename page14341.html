Customizing Checks by Discipline

This task explains customizing the validation checks by discipline. It will result in only your customized checks displaying when you select a discipline in the Design Validation Report dialog box.
1. Validation checks are customized for a discipline by entering all possible checks you want for that discipline in a text file. The file is XXXRequiredDesignValidationChecks.txt where XXX is the discipline. Thus, for the Piping discipline the name of the sample file is PipingRequiredDesignValidationChecks.txt.

The sample files are in ...startup\EquipmentAndSystems\XXX\SampleData, where XXX is the discipline.

The file is a resource managed by project resource management. Its resource name is XXXRequiredDesignValidationChecks, XXX being the discipline.
2. Each check has been assigned an abbreviation, or key. You need to enter these abbreviations in the Required Design Validation Checks file for a discipline. The abbreviations are listed in Step 3. An image of a portion of the Piping file is shown below.
