How to do an Optimal CATIA PLM usability for Safe Save in ENOVIA V5 VPM from CATIA V5

The objective of the Optimal CATIA PLM usability for Safe Save is to prevent the user from building / editing data in CATIA V5 if they are not supported in ENOVIA V5 VPM. Therefore, in interoperability mode, some CATIA V5 commands are disabled or unavailable and in some cases, rules are applied to restricted commands.
 

Safe Save in Product Specification context
  The aim of safe save is to prevent certain operations in CATIA (in context of assemblies containing Product Specification) which are not supported by ENOVIA V5 VPM when saving them back in to ENOVIA V5 VPM from CATIA.
  The following table explains you about the Safe Save rule/unsupported commands in CATIA, in context to an assembly containing
Product Specification.
 
Command Description Representation Warning Messages
Instance Creation
Insert Existing/Create New

  • Product Specification and  instances under it cannot be created in CATIA and saved into ENOVIA. They can only be created in ENOVIA.

  • Insertion of exploded entities such as new Parts, new Products and new components (also existing  components) can prove to be unsafe unless they are inserted under a Work Package contained by a Product Specification.

  • By implying Safe Save rule, the above mentioned instance creation commands are disabled. These commands are grayed out in the menu.

 Insertion under Product Specification is forbidden.
Copy This command is disabled on Product Specification, and on its parent nodes. The command also includes the Drag and Drop and the Ctrl+C/Ctrl+V actions. -
Cut This command is disabled on Product Specification, and on its parent nodes. The command also includes the Drag and Drop and the Ctrl+X/Ctrl+V actions. -
Paste This command is disabled on Product Specification, and on exploded objects under Product Specification. However CATIA features can be pasted under Part instances. -
Replace The operation is detected and stopped by displaying a warning message. Invalid Replace: Target contains a Product Specification or is a Product Specification itself.
Contextual Links No contextual link can be created with the exploded entity under the Product Specification. When the command Define Contextual Link is invoked a warning message is displayed and the operation is stopped. - Entity not valid for contextual link. Contextual link creation using an entity under Product Specification is not supported.
Save as new version in VPMNAV
 (Versioning)		 Versioning in context of Product Specification is disabled in order to minimize the complexity of keeping the precise overview of several situations, Versioning in VPMNAV save dialog box is disabled whenever a Product Specification is opened in CATIA sessions. In save dialog box, Versioning and Save all Versions options are disabled. -

 
In order to achieve Optimal CATIA PLM usability, the Save In ENOVIA V5 dialog box restricts the choices you can make for save options. For more information about Saving options, please refer to Saving an ENOVIA V5 VPM Document from CATIA V5.
