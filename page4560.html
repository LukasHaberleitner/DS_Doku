Document Templates: Limitations

  • A publication cannot point an object already published more than once. When creating the import link, the published object is looked for and the import is created on the first publication found which might not be the one that has the same name as the input.
    The only information that the Document Template can provide is the final object itself (infrastructure prevents you from specifying the publication, enabling you to select only the pointed object). The publication is then automatically retrieved by the link infrastructure.

  • The document template does not accept inputs (both automatic and manual) if they are not pointed or if they are only pointed by publications or other external references in other parts (example : the result of a Copy-Paste with link will not be detected and can not be manually selected if no additional operation is done).

    Workaround: Create meaningless geometry that uses these inputs in the same part.

  • The document template does not accept geometrical sub-elements of a feature as inputs. For example, faces, edges or vertices of a pad. However the document template accepts extracts from geometrical sub-elements as inputs.
  • The Save in Catalog  command can not be used to either save, create or modify document templates using ENOVIA V5 VPM.
  • Instantiation of document templates is restricted by the Instantiate From Selection method because the original assembly or part must not be open in the session.
