Reconciling Product Structures Containing Parts  

 

The interactive reconciliation window enables the user to define reconciliation rules for documents loaded in a CATIA session in order to import the received data into the VPDM.
It is highly recommended that you refer to the ENOVIAVPM  and ENOVIA V5 VPM best practices and user guides to learn more about these VPDMs.
 

Data Exchanged from the OEM to the Supplier

  1. In CATIA V5, load the relevant information you extracted and that you want to exchange such as a Product structure and CAD data from the VPDM: This enables you to exchange both design Context (read only data) and the Workpackage (data to be designed/detailed by the supplier).

  2. Save on disk using the CATIA Save As... or Save Management... command.

  3. Send the extracted data as a set of CATIA V5 documents to the supplier.

  4. The supplier will create/update/delete data in CATIA V5 and send the updated data back to the OEM.

  5. The data will be re-integrated when received from the supplier into the VPDM using the Supply Chain Engineering Exchange Reconciliation application.
 


Data Exchanged from the Supplier to the OEM

  1. Open the documents to be reconciled.

  2. Click the Launch Reconciliation icon in the current workbench

                                    or

    in the menu bar, select File > Desk > Launch Document Reconciliation.

     

  3. Set the VPDM destination (ENOVIAVPM  or ENOVIA V5 VPM) both for documents and parts. 

  4. Perform queries to see if the object already exists in the VPDM. To do so, select a criterion in the Query tab by selecting a criterion for searching documents/parts inside the VPDM repository among the criteria available in the Query Criteria list. Queries list will be VPDM and object type dependant.

  5. For each object to be reconciled, select the storage mode (Publications Exposed or Structure Exposed) and one of the following reconciliation rules:

    • Overwrite in VPDM
    • Reload from VPDM
    • New Object in VPDM
    • Keep As External

  6. The user displays the search results and selects the VPDM candidate for document/Part reconciliation.
    Note that the characteristics display can be customized.
    The status OK/KO of the characteristic is the result of the publications comparison between the document to reconcile in the CATIA session and the selected document stored in database. Depending on the result, the user decides if he wants the reconciliation to occur or not. See Checking Object Characteristics.

  7. The user selects the reconciliation options among the ones available for each Document/Part in the list. Note that:

    • the New object in VPDM option can be selected without performing a query. This is also true for the Keep as external option on broken links.
    • the 3D Viewer is available for CATIA CATParts\CATProducts and models. It is available for the VPDM with the cache only for CATParts and models after having defined a mapping onto a VPDM object.

     

  8. When you have defined a reconciliation choice for all documents/parts to be reconciled, press the Apply button. Note that nothing will be saved in ENOVIA before you use the File > Save command.
