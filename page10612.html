Conversion of SolidM entities as Result (migrated as a PartBody)

This task shows you how a V4 Mock-Up Solid can be converted (As Result) into a V5 Exact Solid, into a PartBody and therefore, it can be modified by V5 Operators.
Open SOLIDEV4_PRIMITIVEV3isole.model.
 

What is the behavior by default:

  1. Copy / Paste (As Result) SOLIDEV4_PRIMITIVEV3isole.model into a CATPart and a Warnings dialog box is displayed:

    This Warning informs you that a cgr document has been created after the copy / paste operation; it is stored in the same directory as the .model.

  2. Click Close. The CATPart is empty.
  3. If you open the cgr file, you only have access to Visualization (Mesh) and you cannot modify any data.
    The default mode is the as CGR migration.

What is the behavior when you activate the As PartBody migration of the Solid Mock-Up into a V5 PartBody?
  1. Check the option as PartBody in the option entitled Isolated Solids Mock-Up Migration - Feature Result in Tools > Options > General > Compatibility > V4/V5 Space (by default, the as CGR option is checked):
    If you do not the use the setting, the SolidM will be migrated as CGR.
    To obtain this result you can activate the as PartBody option, by selecting the Tools > Options... command, by clicking Compatibility in the list of objects to the left of the Options dialog box and by choosing the V4/V5 SPACE tab.
  1. Copy / Paste SOLIDEV4_PRIMITIVEV3isole.model into a CATPart As Result and a Warning is displayed. If you select the as PartBody option, you will also have the following message after the copy / paste of the SolidM into a CATPart:
  2. In this message, you know that *SOL4 is migrated As Result and no cgr is created.

Limitations:

We advise customers against converting huge V4 Mock-up Solid, which can lead to:

  • High CPU time consuming,
  • Important CATPart size.
 

Improvements:

  • Improvements in Naming Management (Face selection): if one Face is selected, all the other ones belonging to the same Volume will be selected as well.
  • Improvements in the migration As Spec: isolated solidM are migrated As Result. As a consequence, no .cgr file is created.
  • Color Management: previously, a global color was transferred onto the migrated solid. Now, the Faces' color is the color of the V4 sub-volume they belong to.
