Requirements

  • Please note that all MULTICAx Plug-ins are on a dedicated media. You need this media to install them.
  • Never modify data generated in V5: information could be lost and the MULTICAD Update could result in incorrect results.
  • Part Number must be unique, all parts with the same name are considered as identical.
  • On Windows, make sure the files to translate are not encrypted: push the Advanced button in the Properties dialog box of the file and uncheck the Encrypt contents to secure data option.
  The MultiCAx SolidEdge (SE) Plug-in is supported on Windows 32 bits and 64 bits.

The MULTICAx SolidEdge (SE) Plug-in allows users to import:

  • SolidEdge native parts and assemblies. This import converts SolidEdge part geometry into the CATIA Graphical Representation (CGR) or CATPart formats, which renders it readable by V5. Users can perform this import interactively or from the command line.
  • VDA-FS (*.vda): parts only
  • Parasolid formats (*.x_t and *.x_b): parts only. 

When importing SolidEdge part geometry, MULTICAx E Plug-in converts all solid bodies belonging to visible layers in the  SolidEdge part into the CGR-faceted form, with user-specified tessellation parameters. In  addition, parts can optionally be imported in the CATPart data format, which allows access to geometry. Coordinate systems can also be retrieved in the generated CATPart files. Any part user attributes defined in the original part are stored in the CATProduct document.

The assembly import converts the SolidEdge assembly hierarchy into product structure, along with the proper positions. No attributes of the assembly other than hierarchy, positions, and colors of the geometry are addressed in assembly import.

Protocols and attributes

For conversions, the following protocols or attributes apply:

  • Length units for parts are converted to millimeters. 
  • Pure wire frame data is not supported during the conversion.
  • Text and annotations are not supported.
  • Application specific format (e.g. piping data) is not in the scope.

Installation requirements

MSXML 6.0 must be installed on your machine before running the converter.
 

Licensing requirements

.par, .psm, .asm, .x_t, .x_b, .xmt_txt and .xmt_bin extensions are not natively supported in V5.
Therefore, DEL licenses (ENOVIA PORTAL, CATIA) or MEP licenses (DELMIA) are required to access these extensions.
