Migrating the ENOVIA Server

Migration from R21

ENOVIA V5 VPM LCA applications can directly run on the existing database of previous LCA version. After you install the code of ENOVIA V5 VPM LCA applications, you can use the existing database, either from the same machine or from different machine. For more information about the process, see the links provided below:

  • Installing ENOVIA V5 Using an Existing DB from Same Machine
  • Installing ENOVIA V5 Using an Existing DB from Different Machine

Note: You are recommended not to use any of the previous versions of LCA applications once you start working on the current version of LCA applications.

Migration from R20

If you want to migrate from ENOVIA V5 VPM V5R20 to V5-6R2016, you will have to initially migrate ENOVIA V5 VPM V5R20 database to ENOVIA V5 VPM V5R21. For more information about migration from ENOVIA V5 VPM V5R20 to ENOVIA V5 VPM V5R21, refer to the links given below:

Migrating from ENOVIA V5 VPM V5R20 to ENOVIA V5 VPM V5R21 on Windows
Migrating from ENOVIA V5 VPM V5R20 to ENOVIA V5 VPM V5R21 in UNIX
