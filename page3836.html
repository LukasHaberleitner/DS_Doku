Designating Assignment Types

An Implemented Engineering Requirement may be unassigned, or be partially or completely consumed for an operation. Making the designation in the Plan Engineering Requirement Implementation dialog box, enhances the interface and work sessions.

You can select individual ERs or multi-select several at a time. After you have selected ERs, use the right mouse button to show the menu. Chose one of the three types needed.

Un-assign, Partial, and Complete assignment types

When an ER is un-assigned, then it will not be consumed by the designated operation in the Implemented By: box.

When there is Partial consumption, it means the specifications in the ER are partially met at a given operation, and the ER still needs to be processed somewhere else in some other operation in the same work plan or by some other operation is some other work plan downstream. An ER can be Partially consumed multiple times by multiple operations.

When there is a Complete consumption, it means the specifications mentioned in the ER are completely met at a given operation and that the ER is complete and although semantically an ER can be completely consumed only once, the software does not prevent you from assigning an ER to multiple operations with complete consumption type of assignment.

Un-assign Behavior

With this option, all the selected ER will be unassigned from the operation listed in the Implemented By: box.

Changing Complete to Partial Assignment Behavior

When there is Partial consumption, it means the specifications in the ER are partially met at a given operation, and the ER still needs to be processed somewhere else in some other operation in the same work plan or by some other operation is some other work plan downstream. An ER can be Partially consumed multiple times by multiple operations.

If you select this option on some of the ERs which are already assigned as Partial then there will be no change, and those ERs will remain Partially consumed.

You will see the changes reflected in the PPR tree,after you exit the Assign ER command.

Changing Partial to Complete Assignment Behavior

This option will change the implementation type from Partial to Complete for the selected ERs.

If you execute this choice on some of the ERs which are already assigned as Complete then there will be no change.

You can see the changes reflected in the PPR tree.

Additional Information

 
  • Once an ER is changed from Complete to Partial, then this ER will be made available for consumption for other operations because now it is only Partially consumed and can be consumed completely to any other operation.
  • If an ER is already assigned as Complete to some downstream operation in the document and somewhere in the upstream operation, it has been consumed as Partial. You will, therefore, not be allowed to change the ER assignment of the upstream operation. This is because it will result in over-consumption of the ER and this is prohibited.
    If this happens, an error message will be displayed advising you of this condition. You will also see a report showing that the ER that could not be changed and the path of operation where the ER was assigned as complete.
  • Once an ER is changed from Partial to Complete, then the ER will not be available for any other ER for consumption in the same session.
