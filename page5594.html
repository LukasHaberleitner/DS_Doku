Colors and attributes

This option defines how colors and applicative attributes are transferred.
Note: Graphic attributes other than color and transparency (line type, thickness, symbol, ...) are copied regardless of the status of this option.

  • Overloaded faces: This option overrides the standard behavior for colors and attributes and keeps the visual aspect of the original data.
    For example, if the color of one face has been changed like this

    the result of the transfer looks exactly the same.
    • Be aware that this option may be time consuming.
    • If a Body is not up-to-date (update icon displayed in the tree), the color of the subelements will not be kept even if this option is selected.
    • When the color or the opacity of a subelement has been overloaded (at the level of the subelement or at the level of one of its parents), in the result part, the overloaded color or opacity is kept on the subelement, but can be changed at the level of the subelement only, not at that of its parents. If you overload the color or opacity of one of its parents, the color or opacity of the subelement will not change.
  • Container: The whole transferred solid takes the color of the original body.
  • None: The result takes the default color of the body.
 

2D Layout for 3D Design data

2D Layout for 3D Design data can be processed only if they can be visualized in the 3D Viewer. 2D Layout for 3D Design data that are in No-show in the origin documents are transferred and are in No-show in the filtered documents.

In the case of a global transfer (all options are selected),  2D Layout for 3D Design data will be reconnected to the elements that were pointed, with the exception of 2D profiles generated from 2D geometry of 2D Layout for 3D Design views that are no longer linked to filtered geometry. In particular, 2D Layout for 3D Design filters will be transferred and filtered objects will be correctly managed in destination document. No difference will be visible on resulting 2D Layout for 3D Design visualization.

In the case of a partial transfer (some options are not selected), 2D Layout for 3D Design data will be totally transferred, but some 2D Layout for 3D Design objects will point to non transferred objects (e.g. non transferred annotations cannot be pointed by transferred 2D layout filters).

All objects (sheets, views, 2D component references and 2D component instances, 2D geometries, annotations, layout filters etc) will be transferred and visible but in some cases objects will be isolated (e.g. annotations that cannot be rerouted on a translated object). Non translated objects will be removed from transferred filters.

As 3D data and 2D data are transferred in separate steps, profiles in 3D generated from 2D geometry in 2D Layout for 3D Design will be isolated from their 2D geometry in 2D layout after filtering.

All texts are isolated (no more attribute links). In particular, texts under views with scale factor will lose that scale factor.

Links to the catalog of details are kept after filtering. This means that if the catalog is in the input CATPart, the result CATPart will have a link with the input CATPart

  • To prevent the creation of a document with corrupted data, when part of the 2D Layout for 3D Design data cannot be transferred correctly, an error occurs and no document is created.
  • When this option is selected, the input CATPart is modified, but not the data it contains. You need not save the input CATPart.
  • The update status of 2D Layout objects (layout, sheet, view) may change is some cases. For example, if a view is associated to a 3D plane, the corresponding view in the output document will not be up-to-date as the association has been re-created with new 3D plane.
  • Curvilinear dimensions with Offset or Parallel representation modes do not support isolated state. As a consequence, rather than turning to isolated analysis display color, they turn to not-up-to- date analysis color.
 

Annotations

Select this option to transfer annotations from 3D Tolerancing & Annotations and 3D Tags from Generative Shape Design.
Annotation Annotations and 3D Tags can be processed only if they are visualized in the 3D Viewer.
You will find below explanations on what happens.

Non-displayed 3D Functional Tolerancing & Annotation Features

When Keep annotations is selected, the 3D Functional Tolerancing & Annotation data are all transferred even if they are not displayed:

  • The 3D Functional Tolerancing & Annotation features that are in No-show in the origin documents are transferred and are in No-show in the filtered documents.
  • The 3D Functional Tolerancing & Annotation features that are in a given layer in the origin documents will be in the same the same layer in the filtered documents even if the Keep layers and filters definition option is not selected.
  • The 3D Functional Tolerancing & Annotation features are transferred even if the filters applied to the origin documents make them not displayed:
     
    • If the Keep layers and filters definition option is selected, the display of the 3D Functional Tolerancing & Annotation features in the filtered documents is exactly the same as the display of the 3D Functional Tolerancing & Annotation features in the origin documents.
    • If the Keep layers and filters definition option is not selected, all the 3D Functional Tolerancing & Annotation features (that are in show mode) of the filtered documents are visible.
  • Even if the annotation sets are switched off in the origin documents, the 3D Functional Tolerancing & Annotation data are transferred (see Disabling/Enabling Annotations in the 3D Functional Tolerancing & Annotation user's guide). The transferred annotation sets are in the same state (switched on or switched off) in the filtered documents than in the origin documents.

Names of the Transferred 3D Functional Tolerancing & Annotation Features

The transferred 3D Functional Tolerancing & Annotation features are displayed in the feature tree with the same names as in the origin documents and in the exact same order.
The names of the Geometrical Links features of the transferred 3D Functional Tolerancing & Annotation features

are partially kept:

  • The names of Group Of Surfaces and User Surface features are kept.
  • The names of Geometric Component features are not kept. They are getting a default name in the filtered documents.

Old Construction Geometry Features

The old construction geometry features (created before V5R13 and displayed under an OpenBody for 3D Annotations node) are special GSD features.
These features are not fully transferred. They are transferred as datums as any other GSD feature but are no longer editable in the filtered documents. All the 3D Functional Tolerancing & Annotation features that are applied to them in the origin documents are transferred and applied to the corresponding datum geometry in the filtered documents.

New Construction Geometry Features

The new construction geometry features (displayed under the Construction geometries node of the Annotation Set in the feature tree) are fully transferred and can be edited in the filtered documents.

Thread Construction Geometry

Thread construction geometry features are linked to Part Design threaded hole features or thread features or their corresponding Technological Result thread features.
The threaded hole features or thread features are not transferred but the corresponding Technological Result thread features are.

Case of old construction geometry (see above)

The transferred thread construction geometry features are invalid and all the annotations that are linked to them are invalid.

Case of new construction geometry (see above)

If the Technological Result thread features have not been generated, the transferred thread construction geometry features are invalid and all the annotations that are linked to them are invalid.

If the Technological Result thread features have been generated, the transferred thread construction geometry features are valid.
They are linked to the Technological Result thread features (even if they were linked to the threaded hole feature or the thread feature).
All the annotations that are linked to them are valid too.

Note that:

  • Technological Result thread features can be generated on Part Design threaded hole features or thread features, whatever their creation level.
  • The described behavior will work for any new thread construction geometry features whatever their creation level.

Parameters Linked to 3D Functional Tolerancing & Annotation Features

All the parameters of the 3D Functional Tolerancing & Annotation features are transferred.
All the formulas on these parameters are not transferred but the value of the 3D Functional Tolerancing & Annotation parameters are kept.
The attributes links of 3D Functional Tolerancing & Annotation text features are not transferred but the content of the text is fully kept.

3D Functional Tolerancing & Annotation Captures

The result on screen of the display of a transferred capture is the same as the result on screen of the display of the origin capture, except for the following cases:

  • The geometry of the geometrical sets or bodies the capture is displaying has not been transferred (this is the case when the geometry is in no-show in the origin documents).
  • The filtered 3D Functional Tolerancing & Annotation features are invalid while the corresponding origin features are valid (the invalid filtered 3D Functional Tolerancing & Annotation features are displayed in the color that corresponds to the invalid 3D Functional Tolerancing & Annotation features.

3D Functional Tolerancing & Annotation Captures Linked to Cameras

All the cameras that are linked to 3D Functional Tolerancing & Annotation captures are partially transferred:

  • The camera feature itself is not transferred.
  • The view point definition of the camera is transferred as 3D Functional Tolerancing & Annotation capture feature view point attributes.
  • When editing the capture in the filtered document, UndefCam is displayed in the Capture Options panel to let you know that a view point is defined but is not linked to any camera feature.
  • When editing a capture that uses a Camera in the origin document, the name of the used camera is displayed in the Capture Options panel.
    When editing the corresponding transferred Capture in the filtered document, UndefCam is displayed in the Capture Options panel.

Note Object Attribute From a Ditto

Instances of Note Object Attribute From a Ditto are transferred by removing the link (if any) to the document that contains the reference Note Object Attribute. The instances of Note Object Attribute From a Ditto in the filtered documents are linked to the CATDrawing documents that contain the corresponding detail.

Behavior of 3D Functional Tolerancing & Annotation Features that cannot be rerouted on the Datum Geometry of the Filtered Documents

When there is no datum geometry that corresponds to the origin geometry (e.g. when the geometry in the original documents is in no-show – except for the XY, YZ, ZX planes that always exist in the result products), the 3D Functional Tolerancing & Annotation features behave as follows:

  • Annotation features: The annotations features are transferred but are invalid.
    Even if they are invalid, they are displayed exactly as the original features (same position, same values …) except for the color that corresponds to the 3D Functional Tolerancing & Annotation invalid features color (dark brown by default).

3D Functional Tolerancing & Annotation Features Belonging to Selection Sets

If the option Keep selection sets is selected, the 3D Functional Tolerancing & Annotation features belonging to selection sets will be kept in the selection sets of the transferred documents.

  • When 3D Functional Tolerancing & Annotation features cannot be transferred correctly, an error occurs and no filtered CATPart is created.
  • 3D Functional Tolerancing & Annotation features defined at product level are not kept when performing a product to product filtering.
  • V4 Functional Dimensioning and Tolerancing data are not kept when performing a product to product filtering.
  • The graphic properties of element in the sketch used for extraction are not kept.
  • Sketches defining extraction views (Offset Section view and Aligned Section view) are not transferred as plain datums.
    The result is a sketch with elementary features (points, lines, curves).
  • When you filter CATParts  that contain 3D Functional Tolerancing & Annotation features with the option Keep annotation selected, you are asked to save the origin CATPart when you close it. Do not save the origin CATPart, unless you have modified it.

Specifications of Sketches

When it is cleared, the sketch is transferred As Result.
The resulting sketch is not editable.

When it is selected, the sketch is transferred As specified in Part document:

  • The resulting sketch is editable.
  • The sketch support positioning is switched to Isolated definition mode.
  • The sketch parameterization and all its associativity with external features is removed:
    • Constraints and their formulas
    • Output and output profile features
    • All construction geometries
    • All geometries in NoShow.
  • Use-edges (projections, intersection, ...) are isolated and converted to sketched curves.
  • In the case of use-edges with a complex curve mark, the wire isolation cannot keep the sub-element naming, nor the previous orientation of the curves.

More information

  • Filtering applies only to the contents of Bodies, Geometrical Sets or Ordered Geometrical Sets.
  • When an Ordered Geometrical Set is filtered, solids are transferred first, then the surfacic entities are transferred. In the resulting CATPart, you will find the solids at the beginning of the list of the entities of the Ordered Geometrical Set, even though there were surfacic entities before those solids in the input CATPart.
  • The change of an In Work Object is not taken into account. The geometry is transferred in its final status, not in the intermediate one. 
  • Only visible geometries are transferred. In some cases, no empty bodies are created in the resulting CATPart, with the following exception:
    • Since a CATPart is always created with a PartBody, you will always find one in the result CATPart.
  • The option Checking Operation When Renaming: Under the same tree node in Tools > Options > Infrastructure > Part Infrastructure > Display, is inefficient in Product Data Filtering. Therefore be aware that you can find several elements with the same name under one node.
  • If a publication, selection set or a layer refers to a geometry placed in the NoShow in the original CATPart, this publication, selection set or a layer is not transferred.
  • When annotations are transferred, the active view of the resulting CATPart is not necessarily that of the input CATPart.
  • Elements other than the geometry and the axis system or those processed by the options above are never transferred.
