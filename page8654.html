Moving Instances in CATIA V4

  1. In CATIA V4, select Models / Modify / Replace from the menu displayed on the right side of the window.

    The instances' position is modified, according to ENOVIA product model.

The following capabilities are managed synchronously (at move time):

  • several models for one instance is managed

  • same sub-assembly instanced several times is managed

The following capabilities and controls are only activated when the new positions are transmitted to the ENOVIA Application Server :

  • relations between instances (example : symmetry)

  • ENOVIA lock mechanism is taken into account : parts and instances need to be locked before refreshing positions of an instance moved in CATIA.

 

  1. In the ENOVIA Homepage window, select the Engineering Life Cycle folder and double-click the Product Class Editor bookmark.

    The Product Class View window is displayed.

  2. Select  a Product in the list and right-click Open. Your PRC and your product are opened in the Product window.

  3. Right-click the Product Root Class then select the Expand command from the contextual menu.

    All the item instances are expanded.

  4. Right-click a product and select Active Part > Set from the contextual menu.
The Set action in ENOVIA V5 VPM frame enables to define the level in the product structure tree under which the components will move. There is at most one active object, underlined in the product structure tree. Each instance that is a component of the active object moves as a rigid object in the context of the active object. If no explicit active object is defined, each object moves in the context of its next higher assembly.
