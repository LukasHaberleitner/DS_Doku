Accessing ENOVIA V5 VPM Replication

Partial Replication enables the sharing of specific data between different ENOVIA V5 VPM installations within one company and the supply chain. Partial Replication allows you to share work or design at any state of maturity, design part or assemblies in one place and replicate to other sites. This process is mainly an administration task that is done on the server side. For more information about replication, see LifeCycle Administration Guide > System and Data Administration > Partial Database Replication Setup and Administration.
Note: Replication process varies from Reconciliation, as it does not extract data in CATIA file mode to reintegrate in ENOVIA V5. Replication aims to exchange data between two ENOVIA V5 sites.

  The objective to define the scope of replication is the Replication Package. The Replication Package is defined with a selection of an object existing in Database in addition to some options that details out in what depth the linked objects are enrolled.
  You can create a replication package for the objects, that you are going to send to external sites. You also have the ability to simulate the creation of a replication package before creating the actual package. You can see the html report of the objects that will be extracted, thus ensuring the replication package's contents. This creation can also be performed using LCA client. For more information, see LifeCycle Administration Guide > System and Data Administration > Partial Database Replication Setup and Administration > Creating a Replication Package.
  To understand how the ENOVIA V5 VPM Replication Toolbar is used for creating replication package in VPM Navigator, see Partial Replication Process Management.
