Verification 

This tab appears only if you have a DELMIA license.

This page describes the options that determine what is displayed in the geometry area when you use Process Verification and (unless otherwise specified) Process Verification Across Resources.

  • PPR Tree Synchronization

  • Engineering Requirements

  • Output Relation

  • Navigation

PPR Tree Synchronization

Scroll activity to center

When selected, the PPR tree scrolls up and down so that as each activity is highlighted, the PPR tree is centered on that activity. If the activity shown by Process Verification is not visible because it is in an unexpanded portion of the tree, only the parent process is highlighted, unless Expand PPR Tree is also selected.

Selecting Scroll Activity to Center makes available the Expand PPR Tree check box.

Expand PPR tree

Selecting this option results in the PPR tree expanding when the activity Process Verification is showing is not visible in the tree. This option is unavailable when Scroll Activity to Center is not selected.

The Scroll activity to center check box is cleared by default; as a consequence, the Expand PPR Tree check box is unavailable by default.

Engineering Requirements

Licensing
  If you do not have an Engineering Requirements license, the option is not available.

Support Engineering Requirements
When the option is selected, then the shown/hidden behavior default is on as described in in Product Visibility in Process Verification.

When the option is not selected, then PIR relations (Process implements requirements) are not supported. An  Engineering Requirements visualization status does not change due to a PIR relation. Also cross-highlighting for PIR relations is not supported and the cross-highlight dialog box does not contain this relation type.

The Support Engineering Requirements check box is cleared by default.

Output Relation

Support Output Relation (Process Creates Product)

Products assigned with the Product Creates Product relation are shown while running process verification. As a consequence, a product assigned with a PCP relation is shown when you navigate within process verification to the operation to the product is assigned  and to all successive operations.

The Support Output Relation (Process Creates Product) check box is cleared by default.

Navigation

Go to First Child When Using DOWN Button

The default value for process verification is that when you first go to the next level, you immediately go the last process in the next level.  However, by selecting this check box, you can go to the first child process.

Inherit Initial Hide/Show State

If the Inherit initial Hide/Show state check box is not selected, any products or resources assigned to an activity appear in process verification when process verification shows the activity, regardless of whether the resources or products are hidden by the Hide/Show command.

If the option is selected, then the Hide/Show command controls whether the resources and products are shown.

Suppress Link Warnings

In process verification, if you are navigating activities that do not have links, by default, you get a warning message informing you that the activity is not farther linked. If you select this option, the warning options do not appear.

All check boxes are cleared by default.
