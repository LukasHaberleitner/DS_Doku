Producing a Cipher File

This task explains the preparatory steps that are needed to encrypt a file through the DRM capability.
  Prior to encrypting the file, you need to access the DRM settings to define the publishing license that will be used to secure the file.

  1. Select Tools > Options > General > Digital Rights Management:

  2. In the General Settings area, activate the Secure document on 'Save As' operations option.

    The encryption mode is now activated and will be applied when saving documents.

  3. Click Check Environment to make sure that your environment is correctly set.

  4. If needed, click More Options... to access the advanced user interface.

    In this scenario, we use the advanced user interface.

  5. In the Publishing License area, define the list of authorized users or groups with their associated rights:

  6. Click OK to validate and close the Options dialog box.

  7. As User1, Select File > Open... to open an existing clear file.
    In our example, the CATPart file looks like this:

  8. Select File > Save As....

  9. In the Save As dialog box, enter the name and location of the file then click Save.

    The file is saved and encrypted because the DRM capability has been activated in a previous step.

    This protected file will be used in the next scenario to explain how to consume a cipher file.
    To encrypt an assembly with multiple files, you must use the Save Management functionality.
