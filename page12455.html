Saving Data to VPM

This procedure has two parts:
 
  • How to save the current process document to VPM 
  • How to save a process document from an assembly
     
The document must have been read from VPM during the session in which you plan to do the save.  If you load data from VPM, save it on disk, and then attempt to save it to VPM in a separate editing session, you cannot save that file to VPM using this procedure.

The VPM process that users run to load their data onto DPM must also be used to save modified files to VPM.

Save the current document

1.

The current document can be saved by using Tools>Save in Database.

VPM will not ask you for any confirmation; it will simply overwrite the original file.

Save a document from an assembly

1.

Select the ProductStructure workshop; use Start>Infrastructure>Product Structure.

2.

Select Commit in Database.

3.

Select a product. The associated document will be saved in VPM. 

Only  the selected document will be saved in VPM; the associated documents will not be.
