Exporting Clash Results Using Second XML - Products from ENOVIA V5 database

This task explains how to export the Clash results in Second XML format using data coming from ENOVIA database

To do so, perform the following operations:

  • Establishing the DMU V5 / ENOVIA V5 Connection form DMU V5

  • Customizing Second XML Format Type Options

  • Loading Models from ENOVIA V5
  • Detecting Interferences and Exporting Results in Second XML Format

     
Have a DMU V5 session and an ENOVIA V5 session running
 

Establishing the DMU V5 / ENOVIA V5 Connection from DMU V5

  1. Still in DMU V5, select Infrastructure -> Product Structure from the Start menu to launch the Product Structure workbench and establish the connection between DMU V5 and ENOVIA V5.

  2. Display the ENOVIA toolbar by clicking Toolbars > ENOVIA V5 from the View menu.

  3. Click the Connect to ENOVIA V5 icon.
 

Customizing Second XML Format Type Options
 

  1. In your CATIA session, select Tools > Options > Digital Mockup > DMU Space Analysis

  2. Clear the XML Export for clash process purpose check box in the DMU Clash - Process tab page.

  3. In the Data base repository filed, select an absolute path, for instance
    (E:\users\Export_XML_Clash\XML)

  4. In the Style sheet location path, select a relative path, for example:
    file:///E:\BXX\intel_a\resources\graphic\UnifiedStyleSheet_xx.xsl

  5. Set the remaining XML Export options accordingly, refer to DMU Clash - Process tab page (Tools > Options > Digital Mockup > DMU Space Analysis)

Loading Models from ENOVIA V5

  1. In ENOVIA Home panel, select the Engineering Life Cycle folder and double-click the Product Class Editor bookmark. The Product Class View panel is displayed.

    To create a PRC, please refer to the appropriate ENOVIA V5 documentation.

  1. Right-click a Product Class Root (PRC) and select Open from the contextual menu.The PRC is opened in the Product Editor panel.

  2. Right the PRC and select Expand from the contextual menu to expand the product. All instances are displayed.

  3. Right-click the instance of your choice and select Send To > CATIA V5 from the contextual menu. The document is displayed in your DMU V5 session.

Detecting Interferences and Exporting Results in Second XML Format

  1. In DMU V5, switch to the DMU Space Analysis workbench (Start > Digital Mockup > DMU Space Analysis)

  2. Select Insert > Clash from the menu bar or click Clash in the DMU Space Analysis toolbar and run an interference analysis.

    The Check Clash dialog box expands to show the results. 

    The dialog box identifies the number of interferences detected along with the type. By default, results are organized by conflict in the List by Conflict tab. 

    Additional information is provided in the Check Clash dialog box to help you compare clash results with those already stored in ENOVIA V5 database.

  3. Click Export As in the Check Clash dialog box to generate the XML file.
