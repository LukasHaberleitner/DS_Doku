Retrieving Information from ENOVIA V5

This task explains how to work with the Clash command and ENOVIA V5. You can:

  • Retrieve clash results from ENOVIA V5

  • Compare new and old clash results

  • Save clash results in ENOVIA V5.

To do so, perform the following operations:
 

  • Customizing Clash Settings in DMU V5

    • Customizing XML Export Clash Settings

  • Establishing the DMU V5 / ENOVIA V5 Connection from DMU V5

  • Loading Models from ENOVIA V5

  • Detecting Interferences & Comparing Results

  • Saving Clash Results in ENOVIA V5

Have a DMU V5 session and an ENOVIA V5 session running
 

Customizing Clash Settings in DMU V5

  1. In DMU V5, set the clash settings in the DMU Clash tab to retrieve information from PDM (Tools > Options..., Digital Mockup > DMU Space Analysis).

Customizing XML Export Clash Settings

If you want to export clash results (saved in ENOVIA V5 and sent to CATIA) with associated pictures in XML format, we recommend you to perform the step 2 or step 3:

  1. In DMU V5, set the XML export clash settings in the DMU Clash tab to retrieve pictures in your XML file when exporting results in XML format (Tools > Options..., Digital Mockup > DMU Space Analysis > DMU Clash).
     

  2. If you need to export only a few clashes results with pictures in XML format, and if you do not want to select Compute all numeric and graphic results option button, all you need to do is:

    Double-click each interference to be exported in an XML file in the DMU V5 Check Clash dialog box

    This action enables to retrieve pictures in the generated XML file (same behavior as the Compute all numeric and graphic results option but only for the selected interference)

    • When done, send the interference to DMU V5 using the Clash Manager.

  3. Set the remaining XML Export options accordingly, refer to DMU Clash - Process tab page (Tools > Options > Digital Mockup > DMU Space Analysis)

  4. Click Export As in the Check Clash dialog box to generate the XML file.
 

Establishing the DMU V5 / ENOVIA V5 Connection from DMU V5

  1. Still in DMU V5, select Infrastructure -> Product Structure from the Start menu to launch the Product Structure workbench and establish the connection between DMU V5 and ENOVIA V5.

  2. Display the ENOVIA toolbar by clicking Toolbars > ENOVIA V5 from the View menu.

  3. Click the Connect to ENOVIA V5 icon.
 

Loading Models from ENOVIA V5

  1. In ENOVIA Home panel, select the Engineering Life Cycle folder and double-click the Product Class Editor bookmark.The Product Class View panel is displayed.
    To create a PRC, please refer to the appropriate ENOVIA V5 documentation.

  2. Right-click a Product Class Root (PRC) and select Open from the contextual menu.The PRC is opened in the Product Editor panel.

  3. Right the PRC and select Expand from the contextual menu to expand the product. All instances are displayed.

  4. Right-click the instance of your choice and select Send To > CATIA V5 from the contextual menu. The document is displayed in your DMU V5 session.
 

Detecting Interferences & Comparing Results

  1. In DMU V5, switch to the DMU Space Analysis workbench (Start > Digital Mockup -> DMU Space Analysis)

  2. Select Insert > Clash from the menu bar or click Clash in the DMU Space Analysis toolbar and run an interference check.

    The Check Clash dialog box expands to show the results. 

    The dialog box identifies the number of interferences detected along with the type. By default, results are organized by conflict in the List by Conflict tab. 

    Additional information is provided in the Check Clash dialog box to help you compare clash results with those already stored in ENOVIA V5.
 

Saving Clash Results in ENOVIA V5

  1. Identify the results you want to store in ENOVIA V5 in the Keep column of the Check Clash dialog box.

  2. Click OK when done in the Check Clash dialog box.A dialog box appears asking you to confirm that you want to save results in ENOVIA V5.

  3. Click OK in the dialog box to confirm your save.

    Clash results are written to ENOVIA V5 and a dialog box informs you whether or not the operation was successful.

    		Using ENOVIA V5 search capabilities, review clash results, they are stored in the database.
    Commit changes to the ENOVIA V5 database before exiting your ENOVIA V5 application
