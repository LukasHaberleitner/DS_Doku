Creating a Delay Activity

This procedure describes how to add a delay activity to a simulation.
This delay may be required to simulate the processing time for some non-motion activities (such as welding).

A delay lasts for a user-specified time; once the time has elapsed, the simulation recommences without any intervention from the user. The need for no user intervention differentiates a delay from a pause, which halts the simulation until the user takes an action.

  1. Click Delay from the Simulation Activity Creation toolbar.

    The Activity Creation Options dialog box appears asking where to place the activity.

  2. Select the location for the activity and click OK.

    The Delay dialog box appears in the bottom right corner.
    You can reverse Steps 1 and 2.

  3. Enter the length of the delay (in seconds) in the Duration box and click OK. 

    The delay activity appears in the PPR tree.
    The default units shown in the Delay dialog box appear in seconds.  You can modify this default by using Tools > Options > General > Parameters > Units.
      The duration of the delay can be modified by right-clicking the delay activity on the PPR tree with the right mouse button and selecting Definition.  The Delay dialog box appears.
