Automating the Valuation of Reconciliation Rules

 

Once you are familiar with the Supply Chain Engineering Exchange application, you may want to automate the valuation of the reconciliation rules.

Pre-defined rules (global sets) enable you to select a global rule in a list of pre-defined valuations and to apply it to all the objects to be reconciled. They can be activated at any time when you are defining the reconciliation rules. 

Global sets can be run on:

  • the whole structure by selecting the structures root object.  To run a global set on just an assembly in the structure, the assembly must either be opened as the root of the reconciliator session or you can use Locate > Select Sub Tree. 
  • a single selected document.
  • multiple objects by selecting them based on specified criteria using the Locate function.
  • Global Set can be launched in two ways:
    • In the Reconciliator Window, select the Global Set tab. Select one of the options under Select a Global Set. Click Apply. The Global Set on current selection dialog box appears.
    • In the Reconciliator Window, select the Global Set tab. Double-click one of the options under Select a Global Set. The Global Set on current selection dialog box appears.
      Note: This enables to launch the Global Set on current selection dialog box directly instead of clicking the Apply button. Thus, there is a reduction in the number of clicks.

Set storage mode: Enables you to select the Structure Exposed or the Publication Exposed mode.  When you select this option and click the Apply button, a dialog box is displayed, prompting you to select the Structure Exposed or the Publications Exposed mode.

Global Query: Enables you to automate any available query for the VPDM (ENOVIAVPM or ENOVIA V5 VPM) selected. When you select the Global Query option and click the Apply button, a dialog box is displayed, prompting you to choose the query type and apply the implicit rules after the query. This query is used to automate the mapping of all objects to reconcile. If the query returns only one result then this result will be mapped to the object on which the query was run.  A progress bar will be displayed when you click the OK button which will indicate the completion status of the transaction.  It will also enable you to end the operation which can be extremely useful for long processes.  Note that ending the operation will stop the transaction at the time the Stop button is pressed.  Any rules that were established before the Stop button was selected will be set.

 
 

Mapping based on VPDM View: Enables you to perform an automatic mapping based on a comparison of the instance tree in the CATIA view and the VPDM view.  The root product's storage mode must be Structure Exposed to use this global set.  Objects will be mapped if their name and the name of their parent are identical to an object in the VPDM view.  Additionally,  both objects must be either
Structure Exposed or Publication Exposed in order to be mapped by this global set.  When you select the Mapping based on VPDM View option and click the Apply button, a dialog will be displayed with the following options:

  • Query Choice For root assembly Part:  the type of query that will be used to search on the root product
  • Force mapping?:  when selected, objects that are already mapped will be re-mapped based on this functions criteria (i.e. both the object's name and its parent's name are identical to an object in the VPDM view and the references of each of the corresponding nodes also match the reference in the instance tree's node.)
  • Apply implicit rules after mapping?: when selected, the amount of operation to be performed is reduced. Based on the available information, the program will try to identify the most appropriate rule to be used. Thus, the following logic will be invoked:
    • If no object is found as result of the search operation and New is authorized, new rule will be set. For a broken link (document not found), Keep as External rule will be selected.
    • If automatic mapping is not possible (more of one object in result list), no Reconciliation rule will be set.
    • In case, where mapping has been successfully done, Overwrite rule will be the preferred rule except if the rule is forbidden (settings or by the ENOVIA solution) or if object in ENOVIA is not saved. This happens if object is Locked by another user, has a maturity state that indicates that the object is frozen, belongs to an external sites, etc... Here, if authorized, Reload from VPDM rule will be set.

A progress bar will be displayed when you click the OK button which will indicate the completion status of the transaction.  It will also enable you to cancel the operation which can be extremely useful for very long processes.  Note that using cancel will stop the transaction at the time the Cancel  button is pressed.  Any rules that were established before the Cancel button was selected will be set.

 

 

All objects as New: Enables you to set all objects to New. When you select this option and click the Apply button, a dialog box is displayed. If you check the Force mapped object to New option, all the objects, even those that are already mapped, will be set to New.

All objects as Overwrite: Enables you to set all mapped objects with no rule to Overwrite. When you select this option and click the Apply button, a dialog box is displayed. If you check the Force all to Overwrite, the rules that were previously applied to these objects will be overwritten.
 

All objects as New Version: Enables you to set mapped objects to new version. The new version rule can be set on CATPart and CATProduct objects whose storage mode is Publication Exposed and Products saved in Structure Exposed as long as they are not mapped to the Product Root Class (PRC).  Setting the new version rule is similar to setting the Overwrite rule except when the objects are save to ENOVIA V5 VPM, the part is saved as a new version and the corresponding geometry document as a new revision.  When you select this option and click the Apply button, the above dialog box is displayed. If you check the Force all to New Version option, all objects, even those with rules already defined on them, will be set to New Version, if possible.  Checking the Apply rule only on documents option results in all documents (for example, CATDrawings) being saved as new revisions and all publication exposed CATProducts and CATParts being saved as new versions.

 
 

All documents as New Revison: Enables you to set all mapped documents with no rule to new revsion. The new revision rule can be set on CATPart and CATProduct objects whose storage mode is Publication Exposed as well as for CATDrawing, CATAnalysis, CATAnalysisComputations and CATAnalysisResult (also with Publication Exposed storage mode set).  Setting the new revision rule is similar to setting the Overwrite rule except when the documents are save to ENOVIA, they are saved as new revisions.  When you select this option and click the Apply button, the above dialog box is displayed. If you check the Force all document to New Revision option, all the documents, even those with rules already defined on them, will be set to New Revision. 

All objects as Reload: Enables you to set all mapped documents to Reload. When you select this option and click the Apply button, a dialog box is displayed. If you check the Force all documents to Reload, all existing rules will be overwritten and replaced with Reload.

All documents as KeepExternal: Enables you to set all mapped documents to External Reference. When you select this option and click the Apply button, a dialog box is displayed. If you check the Force all documents to KeepExternal, all existing rules will be overwritten and replaced with KeepExternal.

All objects as Overwrite by Delta: Enables you to set all mapped objects with no rule to Overwrite by Delta. The Overwrite by delta rule can be set on CATPart and CATProduct objects whose storage mode is Structure Exposed and mapped either against ENOVIA V5 Part Version or ENOVIA V5 Product Root Class (PRC). When you select this option and click the Apply button, a dialog box is displayed. If you check the Force all to Overwrite by Delta, the rules that were previously applied to these objects will be overwritten and replaced with Overwrite by Delta.

All objects as New Version by Delta: Enables you to set all mapped objects with no rule to New Version by Delta. The New Version by delta rule can be set on CATPart objects whose storage mode is Structure Exposed and mapped against ENOVIA V5 Part Version. When you select this option and click the Apply button, a dialog box is displayed. If you select the Force all to New Version by Delta check box, the rules that were previously applied to these objects even those with rules already defined on them, will be set to New Version, if possible.



Magic Reconciliation: Enables you to execute an automatic reconciliation based on the selection of appropriate VPDM system. When you select this option, and click the Apply button, a dialog box is displayed with the following options:

  • Force all to automatic selected Storage mode?: when selected, objects that have already selected Storage Mode will re-select the Storage Mode based on this function criteria.
  • Force mapping?: when selected, objects that are already mapped will be re-mapped based on this function criteria (i.e. both the object's name and its parent's name are identical to an object in the VPDM view and the references of each of the corresponding nodes also match the reference in the instance tree's node.)
  • Force all to automatic selected Rule? when selected, objects that have already selected a Rule will re-select a Rule based on this function criteria.

By default, all the check boxes are selected.

Reset all:  Enables you to reset all objects mapping and/or rules. When you select this option, and click the Apply button, a dialog box is displayed. You may then decide to reset only the rules or the rules and the mapping. 
When you apply the New, Overwrite, Reload or Reset rule, the root product is not taken into account and the operation must be performed manually.
Before performing this scenario, make sure you have:
  • activated the cache system (Tools > Options > Infrastructure > Product Structure > Cache Management > check option Work with the cache system)
  • have invoked the reconciliation option (View > Toolbars > Reconcile)
  • have activated the ENOVIA V5 toolbar (View  >  Toolbars > ENOVIAV5)
  • have connected to the ENOVIA V5 VPM database (click the Connect User to ENOVIA V5 icon)

To perform this scenario, you will need the following documents:

The Skate-100-ABC-1001.CATProduct document and its sub-products must have been saved in the VPDM in mode Structure Exposed before you begin the reconciliation process. In the scenario described below, it corresponds to the skate board that the manufacturer sends to his supplier for modification. It is made up of:
  • Axle-010-JKL-1031.CATPart
  • Board-010-DEF-1009.CATPart
  • Wheel-010-GHI-1021.CATPart
  • Hub1-010-DEF-1001.CATProduct
The documents listed below are those that will be reconciled using the Supply Chain Engineering Exchange application.
  • Axle-010-JKL-1031.CATPart
  • Board2-010-DEF-1011.CATPart
  • Wheel-010-GHI-1021.CATPart
  • Wheel-Large-010-GHI-1026.CATPart
  • Hub1-010-DEF-1001.CATProduct
  • Hub2-010-DEF-1002.CATProduct
  • Skate2-100-ABC-1002.CATProduct

In this scenario, the manufacturer (you) who receives the modified skate board from his supplier decides to accept all the modifications performed. He decides to overwrite the data stored in the VPDM with the one sent back by his supplier.

  1. In CATIA, open the Skate2-100-ABC-1002.CATProduct in design mode.

  2. Launch the Reconciliation window.

  3. Select the root in the Objects to reconcile tree, and in the Query tab, select the Easy Query and click the Query button.

  4. In the Query window, enter Skate* in the Product ID field and click OK.
    Skate-100-ABC-1001 is displayed in the VPDM Objects view.

  5. In the Global Set tab, click the Global Query option and Apply. The Global Set dialog box is displayed.

    Global Query enables you to run the selected query using the selected query default parameters.

    Mappings are established for all objects for which the name in the database is identical. It is now necessary to manually map the remaining objects.

  6. Repeat steps 3 and 4 for the remaining objects:

    • Board2-010-DEF-1011
    • Hub2-010-DEF-1002
    • Wheel-Large-010-GHI-1026

    The result will be as below:

  7. In the Global Set tab, click the All objects as Overwrite option and Apply. The Global Set dialog box is displayed.

    All objects as Overwrite: Enables you to set all mapped objects with no rule to Overwrite. When you select this option and click the Apply button, a dialog box is displayed. If you check the Force all to Overwrite, the rules that were previously applied to these objects will be overwritten.

  8. Click the Force all to Overwrite check box and click OK.
    All objects are set to Overwrite.

  9. Click Apply to apply the Reconciliation.

  10. Close the Reconciliation transaction and from the File menu, select the Save in ENOVIA command.
