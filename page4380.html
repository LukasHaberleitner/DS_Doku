Understanding Exact Measure Versioning

This task explains how version information is taken into account in Measure item and Measure between commands.

  • Example

About Version Information

Version information provides a stable value for the exact measurement. The measurement value is associated to the V5 level Release N on which the data were created (the measure is strictly V5 level Version dependent); it does not depend on the V5 level Version on which the data is loaded.

Example

Let's take a simple assembly comprising two pads: pad1 and pad2 contained in a part document (Part1)

  • Open this part on V5 level Release 17.
  • Create a measure MeasureMB.1 between the two pads in exact mode and save it.
  • Save the Part.1
  • Now, open the Part.1 on the next V5 level (Release 17+N)
     
  • If you update MeasureMB.1, the results will be identical to the values created on V5 R17 level
     
  • On the next level (V5 Release 17+N), if you edit MeasureMB.1 and change the inputs, the measure calculation will be based on the V5R17 measure functionality. But if you create a new measure, the calculation will be based on the V5R17+N measure functionality.
