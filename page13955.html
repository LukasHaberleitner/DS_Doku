Text ID Mapping

Concept

  Title Block text ID mapping allows users to map text and legacy data more easily. Texts in the Title Block can now be mapped directly from/to CATIA.

Until now, the Title Block mapping was based either on the CATIA property mapping mechanism or on a user-written CATIA macro. In V5R19, the only constraint is that the text id must be defined according to SmarTeam rules; the mapping is performed automatically.

In a CATIA drawing, texts are mapped to SmarTeam attributes that can be displayed in the SmarTeam profile card. The user can update the text that is mapped to SmarTeam in the drawing by modifying the text directly or by updating the drawing based on SmarTeam attribute values.

 

Procedure

The user uses the Title Block defined by the Administrator. The user can change the text in the Title Block and, after a Save operation is performed, the Title Block is updated.

Note: The Text ID mapping only takes place for texts located in the background sheet.

Mapping Order

If the same attribute has been mapped in different ways from/to CATIA the mappings take place in the following order:

  1. File properties mapped from CATIA to SmarTeam

  2. CATIA Text name mapped from CATIA to SmarTeam

  3. CATIA knowledge parameters mapped from CATIA to SmarTeam

  4. CATIA knowledge parameters mapped from SmarTeam to CATIA

  5. CATIA Text name mapped from SmarTeam to CATIA

  6. File properties mapped from SmarTeam to CATIA

For information on defining Title Block text ID mapping, refer to Configuring Title Block Text ID Mapping.
