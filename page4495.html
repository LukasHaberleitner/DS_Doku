Controller Profiles

Every robot controller has certain profiles each of which define characteristics of different aspects of the controller. By default, controllers by the name Default are defined for Motion, Tool, Accuracy and Object Profile.

A Motoman robot must have Tool Profiles created before uploading.

Please see Device Task Definition: Workbench Description: Robot Controller Toolbar for more information on controller profiles.

Applicative Profiles

User Applicative Profiles can be used to define Process information.

Please see Device Building: Advanced Tasks: Creating and Editing Profiles: Creating Applicative Profiles for more details.

Action Libraries

Action libraries consist of a sequence of activities that can be used to perform specific operations such as spot welding. The MRCSpotWeld action library is provided for Motoman controller.

Please see Device Task Definition: User Tasks: Device Task Programming Tasks: Creating and Modifying an Action Library for more details.

Robot Parameter Data

Robot parameter data informs the translator about the user's special requirements. See Robot Parameter Map section for supported parameters.

The following procedure describes how to define device parameters.
Open the MotomanStart.CATProcess file.

  1. Right-click robot SK6.1 in the PPR tree and select Properties. The properties dialog box opens.

  2. Click the Define other properties button.
    The Define Other Properties dialog box opens.

  3. Select the type of parameter from the list and click the New Parameter of type button.
    The new parameter appears in box listing the parameters (for example, in the dialog box above, ProgramFileEncoding has been added).

  4. In the Edit name and value field, enter a name and value as required.

  5. Or instead, you can skip steps 3-4 and import parameters from a .txt file using External properties. The .txt file must have the following format:

    parameter_name         parameter_value
    ....                           .....

Weld Guns

A servo weld gun is defined as an external axis of a robot, and used with MRCSpotWeld action library .

Servo guns are supported by generating an SVSPOT command following a motion statement. Each MRCSpotWeld and MRCSpotDry action should follow a robot motion operation and be followed by another robot motion operation. In addition to robot positions, the leading robot motion specifies the gun position before the weld and the following one specifies the gun position after the weld.

The Motoman MOVC command and DELMIA CircularVia/Circular moves

A Motoman program can have a consecutive series of MOVC commands. The first MOVC behaves as a MOVL. Each subsequent MOVC command looks ahead one target(N+1) and moves along the arc formed by targets N-1, N, and N+1 to target N (not the entire arc, only the segment from target N-1 to N).

The DELMIA motion planner uses the robot's current position, a CircularVia target, and a Circular target to determine the arc of a circular move and moves across the entire arc.

Because Motoman and DELMIA calculate the arc of a circular move differently, it is advised that the consecutive number of MOVC commands be limited to 3 for upload. In teaching robot tasks in DELMIA, Joint or Linear moves should be inserted between arcs formed by CircularVia-Circular moves.
