Managing 3D Annotations in Annotation Set

This topic provides information about the conversion of V6 3D Functional & Tolerancing Annotation (FT&A) features in a representation into a V5 CATPart or CATProduct document using the Downward Compatibility batch.

 

Conversion as Result

The FTA result features are light features that are dedicated for review and are not editable. From R18 they are generated by Multi-cad applications or by exporting V6 products and 3D shape representations to CATIA CATProduct and CATPart documents using the downward compatibility tool.

  • All modifications in V6 data will be applied to V5 data during the next conversion.

  • The associativity is lost immediately after any modification is performed after CATDWC.

All General Imported but:
  • Annotations cannot be edited
  • Annotations cannot be copied or included in Power Copy feature
  • Annotations cannot be mirrored
  • Annotations cannot be involved in V5 FT&A features
  • Redo or Undo commands are not available

Otherwise, annotations can be deleted individually using the Delete_Result_Feature contextual menu command.

The failure mode information associated with annotations is displayed in a tooltip when you hover over the annotations imported as a result.
The tooltip displays the failure mode severity and the related information in descending order of criticality level (Very High to Least Important) between the following:
  • Very High
  • High
  • Medium
  • Low
  • Least Important
All the failure mode information associated with the same level of criticality is merged and displayed together.

  Linked with an Engineering Connection. Annotations are imported in an Annotation Set result under the product containing the Engineering Connection, existing links with the geometry are not imported.
  Within 2D Layout for 3D Design views Not filterable.

Restricted Area Restricted Area  Not imported.
Capture Capture Imported but without links with bodies, geometrical sets or product instances.
Result Annotation Set Annotation Set Can be displayed and filtered using the following commands:
  • List Annotation Set Switch On/Switch Off (Visualization toolbar)
  • Annotation Set Switch On/ Switch Off (contextual command)
  • Filter (Visualization toolbar)
  • Hide/Show (View toolbar)
  • Display Capture (contextual command)

Can be saved as:

  • .cgr
  • .3dxml

The tooltip is displayed as FTA data in CGR and 3DXml document for hidden text, and not for general tolerance.

The Result Annotation Set can also be kept in the filtered document when using the product data filtering capabilities. In all these cases, they will behave the same as when loaded in the session from these types of documents.

Conversion as Specification



The table below summarizes Functional Tolerancing & Annotation features and whether they can be converted as specifications using Downward Compatibility batch in V5 using the Convert Part as Specification.
  • When at least one feature in the data (that is not necessarily a functional Tolerancing & Annotation) can only be converted as result, then all features in the data will be converted as result.

  • When thread CG is created on TR thread feature in V6, the V5 data exported by CATBack as Spec needs to be updated manually in V5 session.
