Managing Document Links

This task explains how to manually create or delete a link between documents. It also explains how to query document links.
Linked documents will display in the Impacted On/By panes in Enovia LCA dialog boxes.

To link documents, the root of both documents must be published. See Managing Logical Publications and Connections.

 

 1. You can manually link documents.

Make the target document active in the specification tree, and from the menu bar, select Tools - Manage Document Links.

The Manage Document Links dialog box displays. (In the image below the user has already selected the document to which he is linking.)

Select the root of the document that you are linking to (shown in the Publication column), and click the Create Link button.

You can see that Yes displays in the Link To column, indicating your documents are linked.

Yes displays in the Link From column when the selected document is pointing to the document.

You can also delete a link using the procedure described above.

The Open button is enabled when the selected cross–document connection points to an exposed publication (owned by a document(s) in ENOVIA) and is not loaded in the current session (shown above). In this case the Document displays as NOT IN SESSION. You can have one of three results when you click Open: 1) If only one linked document is found, it will open the linked document and, after loading, update all the document column. 2) If there is no linked document found, an error message will display. This implies that the publication is no longer referred in any document. 3) If more than one linked document is found, then the Open From ENOVIA dialog box displays, which allows you to select a document to open.

2. You can query a document to determine its document links.

To query a document, make it active and, from the menu bar, select Tools - Manage Document Links.

In the Publication column in the Manage Document Links dialog box, you can see the published root of the linked documents.
