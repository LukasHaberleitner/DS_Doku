Extended Engineering Language (X-EKL)

This language inherits the behaviors and components of the Mathematical Engineering Language (M-EKL), of the Core Engineering Language (C-EKL) and of the Advanced Engineering Language (A-EKL).

V5 applications deliver sets of functions and methods that can be triggered on their objects. X-EKL contains those functions and also enables you to:

  • Create user functions using BKT and CAA. For more information, see BKT User's Guide and CAA V5 Encyclopedia.
  • Create types packages to extend the Knowledgeware Language. For more information, see CAA V5 Encyclopedia.
   
 Note that 2 functions (or methods) may have the same names if they have a different list of arguments. In Object oriented programming, the Knowledgeware language supports the function/method overloading but it does not support polymorphism.
