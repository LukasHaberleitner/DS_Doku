SmarTeam CATIA Integration Settings

 

This page provides the different settings available for working with SmarTeam CATIA Integration product.

The following settings will be found in SmarTeam System Configuration Editor:

  • CATIA.Refresh_Screen
    If set to FALSE, the profile card of the document will not be displayed after:

    • the first SmarTeam Save operation performed from CATIA

    • Life cycle operations (Check in, Release, Check out, New release etc.)

    Values: TRUE/FALSE

  • CATIA.LinkAllToMainClass
    Specifies whether CATIA documents are linked to the Project after a Save operation.
    Values: YES/NO

  • CATIA.ExposeMode
    Exposes CATIA internal objects when saving an assembly or drawing inside the system. If set to YES, internal components and drawing sheets are saved in SmarTeam as distinct objects.
    Values: YES/NO

    Note: To work with CATIA.ExposeMode=YES, the user must select the Enable CFO for Lifecycle operations checkbox and select the Force Checkout option in Application Tools Setup. This is relevant when working with CATIA Product, Drawing and Sheet file types.

  • CATIA.SaveInstanceData
    Exposes instance information (name + position matrix) when saving an assembly inside the system.
    Values: TRUE/FALSE

 

The following settings will be found in Save dialog:

  • Add To Desktop
    Specifies whether CATIA documents are placed on the Project Desktop after a Save operation.
    Values: YES/NO

 

The following setting will be found in SmarTeam Menu:

  • Batch Mode
    Saves the components of an assembly without displaying each profile card.
    Values: TRUE/FALSE
     

  • Out Of Process
    Sets CATIA in a process separate from SmarTeam. Must be set to YES.
    Values: YES/NO
