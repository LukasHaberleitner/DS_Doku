Cgr Management  

   
 

The Cgr settings page concerns the following settings categories:

  • General
  • Applicative data

The cgr settings enable you to define the policy for the generation of cgrs into the cache.
 

 

General

Save level of details in cgr

  
  Activating this option will cause level of details to be added to generated cgrs.

By default, this option is activated.
 

  
     
 

Save lineic elements in cgr

  
  Activating this option will cause the following lineic elements to be added to generated cgrs:
  • lines
  • edges
  • axis systems
  • planes
  • wire-edges

By default, this option is not activated.
 

  
     
 

Optimize cgr for large assembly visualization

  
 

When this option is activated, only a subset of each cgr data content is loaded to reduce memory consumption and improve loading performances. This subset consists of the minimum amount of information (such as vertices, normals and triangles) necessary for the visualization of the geometry.

This mode is useful to save RAM usage for extremely large data set that does not fit in RAM, at the expense of a lower frame rate.

Note the following:

  • This subset is sufficient for most DMU review scenarios performed on large assemblies
  • If a DMU command requires more information from some cgrs, the entire data content of the concerned cgrs will be automatically loaded.
  • This option has an effect during session time only, it has no impact on the persistent content of the cgr.
  • This option is only compatible with cgrs computed in V5R14 or later.
  • In order to upgrade old cgrs, see the DMU Navigator User Guide, Running the CATDMUUtility Batch Process,
    option -force    .
  • As minimum amount of information is loaded at start, rotation / zoom performance is expected to be low in visualization mode till all the information of geometry is loaded. After this, rotation performance is similar to that without this option checked.

By default, this option is not activated.
 

  
     
 

Applicative data

 

Save density in cgr

  
  Activating this option will cause density data to be added to generated cgrs. When this option is changed, Cache needs to be cleaned using DMU Navigator along with regeneration of CGR files.

Note: This option will only be available if you have a solution which allows Measure Inertia.

By default, this option is not activated.
 

  
     
 

Save V4 Comment Pages in cgr

  
  Activating this option will cause V4 comment pages to be added to generated cgrs.

Note: This option will only be available if you have a solution which allows Measure Inertia.

By default, this option is not activated.
 

  
     
 

Save V4 Filter Layers in cgr

  
  Activating this option will cause V4 filter layers to be added to generated cgrs.

Note: This option will only be available if you have a solution which allows Measure Inertia.

By default, this option is not activated.
 

  
     
 

Save FTA 3D Annotation representation in cgr

  
  Activating this option will cause FTA 3D annotation representations to be added to generated cgrs.

Note: This option will only be available if you have a Tolerancing license.

By default, this option is not activated.
 

  
     
 

Confirming chosen options

  
 

  1. When you have finished customizing options, click OK to confirm.
