Useful Tips and Limitations

  • Manual modifications of Knowledge Templates instances may be lost when re-executing the Knowledge Pattern. There is no way to forbid such modifications by the user.
  • If sub elements of instances are used to build geometry, this geometry will not be stable when re-executing the pattern. Links to sub elements may be lost.
  • At least in the first release of this development, it will not be possible to embed the Knowledge Pattern feature in a User Feature. But it will be possible to embed it in a PowerCopy and a Document Template.
  • It is highly recommended not to use the Memory Performance option for Knowledge Patterns that are integrated to update. No undo would be available after update (if the pattern is executed during this update).
  • Knowledge Patterns and PowerCopies: Copied objects can be retrieved using GetAttributeObject.  For example, when instantiating a power copy using the InstantiateTemplate function, copied objects can be retrieved using GetAttributeObject on the object returned by InstantiateTemplate.
    The expected name is the name used when defining the PowerCopy.
  • When using several Knowledge Pattern features, it is strongly recommended to instantiate the generated objects into independent bodies.
    If not, already generated objects will be impacted.
  • In samples containing scripts, object names may differ depending on the language used. To find the object name, double-click the object in the specification tree to insert it in the script.
    When working with knowledge patterns, if you want to set the destination, double-click the geometric set that contains the objects in the specification tree.
  • When you work with templates located in the .CATPart that will contain the Knowledge Pattern, you do not have to store this template in a catalog. To do so, just use the CreateOrModifyTemplate method when working with a User Feature or InstantiateTemplate when working with a PowerCopy. This will ease the prototyping and you will not have to point UDFs in ARM catalogs.  To find out more, see Generative Functions.
    Example:
    udf = CreateOrModifyTemplate("UserFeature1", `Geometrical Set.1` , try , i)
    udf.SetAttributeObject("plane", `xy plane` )
    set p = List.1.GetItem(i)
    udf.SetAttributeObject("point", p)
    EndModifyTemplate(udf)
  • If you perform modifications impacting the links of the template documents and/or the structure, edit the document template.
