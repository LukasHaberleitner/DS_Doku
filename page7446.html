Migrating Version 4 Data 

This task shows how to migrate Version 4 3D dimensioning, tolerancing and annotation data to Version 5 without migrating the geometry. See Version 4 Functional Dimensioning andTolerancing Data Migration.
Launch the Product Functional Tolerancing and Annotation workbench (Start > Mechanical Design > Product Functional Tolerancing and Annotation). Select Product in the specification tree, and right-click Components > Existing Component. In the dialog box, select the V4data.model document and click Open.

Make sure that the 3D Annotation Query Switch On/Switch Off icon is activated.

 

Migration at Product Level

  1. In the specification tree, expand the V4data and then the *MASTER nodes under Product.1.

  2. Copy the *TOLP2 annotation set which is located under the *MASTER node, by selecting it and pressing Ctrl+C for example.

  3. Paste the annotation set right under Product.1, using Ctrl+V for example.

  4. Wait for the migration to complete. This may take some time, depending on the amount of data to handle.

    When the migration is over, a dialog box appears, informing you that some data could not be migrated, and that a capture was created for non-migrated data. Click OK to close it.
    Also, a new annotation set, Copy of *TOLP2, is created under the Product.1 node in the specification tree.
  • The created annotation set uses the V4 parent standard.
  • Migrated annotations are associative to V4 geometry.
 

  1. Under the Copy of *TOLP2 annotation set node, expand the Captures node and click Non migrated annotations of *TOLP2 to visualize the V4 data that could not be migrated.

 

Migration at Part Level

CATIA V4 dimensioning, tolerancing and annotation data can also be converted in CATIA V5 at CATIA V5 part level. The converted data is associative to the converted CATIA V5 geometry.

The conversion of CATIA V4 data is possible in two modes:

  • Interactive
  • Batch
 

Interactive Mode

  1. Select the *MASTER, right-click and select Copy.

  2. Open a new CATPart document.

  3. In the new CATPart, right-click the PartBody (root node in the feature tree) and select Paste Special.... The Paste Special dialog box is displayed.

  4. Select CATIA_SPEC in the Paste Special dialog box.

  5. Click OK.

  6. When the migration is over, a Partial Migration dialog box appears, informing you that some data could not be migrated. Click OK to close it.

  7. An annotation set is created in the converted CATPart. It contains converted CATIA V4 data associated with appropriate CATIA V5 geometry.
 

Batch Mode

  1. Start a Version 5 Session.

  2. Select Tools > Utility command. The Batch Monitor dialog box opens. A list of Version 5 batches along with their description gets displayed in the Type column.
    For more information about the Batch Monitor, please refer to Running Batches Using the Batch Monitor, in the Infrastructure User Guide.

  3. Select PLMV4ToV5Migration. This utility allows you to convert CATIA Version 4 models to CATIA Version 5 models. The PLMV4ToV5Migration dialog box is displayed.



    Note: Only the Complete mode is proposed to migrate annotations. Also, do not select any Structuring Element option (Layer, Solid, Set or Ditto) under the Selected Operation area.

In the Selected Operation area, you can choose following Option:

  • Check: Checks CATIA V4 Model data before copying it to CATIA V5.
  • Migrate: Migrates the V4 Model To CATIA V5.
 

  1. Click the Open button to select the file or files to be migrated. The File Selection dialog box is displayed.

  2. Click Member... button in the Input Options field to select the files to be migrated.

  3. Select the target directory to save the migrated file in the Default target directory.

  4. Click OK.

  5. In the PLMV4ToV5Migration dialog box, select the target directory in Target Directory, to save the result HTML file. A dialog box is displayed. Browse to the directory, select it, and click OK.

  6. Click Run. The batch execution starts. After the batch execution, the converted CATParts are saved in the target directory and the output HTML report file can be viewed.
