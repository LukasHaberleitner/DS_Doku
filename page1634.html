Fixing a Problem

When testing a user workbench, you may encounter some problems. To solve them:

  1. From the Tools>Options-Knowledgeware>Business Knowledge Template menu, change the BKT Access Rights from All Users to Expert.

  1. Expand the sequential combination seen as failed in the specification tree (or in the specifications view). Edit the failed behavior ( identified by the icon), modify its inputs to fix the problem.

 

  1. Right-click the behavior and select Operate to operate it. In Expert mode, you can follow up the execution of the behavior thanks to warning windows (see graphic opposite).

 

  1. If the problem is solved and the behavior operates well, open the .CATfct file corresponding to your workbench, make the corresponding changes and save your file.

Behaviors seen in the user workbench are instances of those defined in the .CATfct file. For the corrections made in the user workbench in Expert mode to be saved, the correction must be made in the original .CATfct file.

  1. Go back to your user workbench.

  2. From the Tools>Options>Knowledgeware>Business Knowledge Template menu, change the BKT Access Rights from All Users to Expert.

  3. Start from an empty document and replay your scenario.
