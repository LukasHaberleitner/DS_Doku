Search for a Process, Product, Resource or Work Station Component

Search can be used to locate processes, products, resources or work station components in the database. User attributes defined in Process Engineer for the selected plantype can also be used as search criteria.

  1. Open a process from the Manufacturing Hub (as described in the section Inserting a PPR from a Project).

  2. Click Search in Manufacturing Hub   located in the Manufacturing Hub Commands toolbar, or select Insert > Manufacturing Hub Object > Search from the menu bar.

    A PPR Search dialog box is displayed:
     
    The search attributes displayed are based on what has been defined for the current project in Process Engineer. Attributes that have been set with the Display in Finder option are shown in the PPR Search dialog box.

  3. Select the component type for which to search.

    The upper list allows for the selection of the component type to be searched for: Process Component, Product Component, Resource Component, or Work Station Component.
     

  4. Select a Plantype.

    The second combo box allows for the selection of a specific plantype to search, as well as an All Plantypes option to search all available plantypes:
     
    Each component type (Process, Product, Resource, Work Station Component) provides a default set of attributes which may be used to define the search criteria. In addition, any user-defined attributes defined in Process Engineer for the selected plantype will also be displayed:
     

    For each search parameter, a check box, list, and value field are provided. These provide the following functions:

    Check box

    Clicking on the check box next to a parameter's name activates it, and includes the values set for that parameter in the current search. Any number of the displayed parameters can be activated for a given search. The following image shows an activated parameter (Component Name), with a deactivated parameter below it (Component Number):

    Combo box

    The combo box can be set to =, !=, or LIKE. These are defined as follows:

    • =
      Return only those components that exactly match the value in the value field for the current parameter.

    • ! =
      Return all components that do not exactly match the value in the value field for the current parameter.

    • LIKE
      Return all components that contain the current value in the value field as a substring for the current parameter.

    • NOT LIKE
      Return all components that do not contain the current value in the value field as a substring for the current parameter.

    Value field

    Data entered in the value field represents what is to be search for with regard to that particular parameter. In the above example, all Work Station Component names would be searched for the substring "EX100."

  5. After entering the values for the parameters against which you wish to search, click the Search button to begin the search.
    When the search is complete, the number of components found that matched the entered criteria is displayed in the No. of Results field, and the Search Results section is populated with the search results.

  7. Load one or more of the search results into the current session by selecting the search results of interest and clicking the Load button.
     
    The Load button is available for Product, Resource, and Work Station Component search results. If the search is performed for Process Components, the Load  button is not available.
    A product, resource or work station component may also be loaded by double-clicking the entity in the Search Results list. Processes listed in the Search Results list cannot be loaded in this manner.

 
  1. Click Filter tab to show/hide the user attributes in search criteria.

    By default, all the attributes are selected in Filter tab to show them in search criteria.
  2. Click Advanced tab to specify the additional search criteria.

    All the search conditions are displayed in a list (one condition per row).  Search condition can be added, edited, and removed using Add, Edit and Remove buttons respectively.
    1. Select attribute from attribute list and click Add.
      The Additional Search dialog box appears.
    2. Specify Operator value and Condition.
    3. Click OK to add a search condition.
    4. Select row and click Edit too edit a search condition.
      Modify the values in Additional Search dialog box and click OK to reflect the modified values in selected row.
    5. Select row in the list and click Remove to delete selected condition from the list.
    6. Select row in the list and use Up and Down to move the search conditions in list.
      If Condition is not specified for the last row in list, then a warning message is displayed if you tries to move this last row Up\Down in the list.
 

Improving resource efficiency with multiple search queries

In the past, performing a search query on a large database containing a many objects could result in excessive memory consumption. To alleviate adverse effects on system resources, an environment variable can be used to break each search query into a set of multiple sub-queries, where smaller subsets of database objects are successively searched instead of all at once.

The environment variable DLM_MHI_PARTIAL_SEARCH_RESULT_DISPLAY should be set to the number of objects that are to be searched for each sub-query. The number of sub-queries is then determined accordingly. For example, if a database contains 100,000 objects, setting the following values results in the corresponding number of queries as shown:
 

Variable Value Number of Queries
10 10,000
100 1,000
1,000 100
10,000 10
25,000 4
50,000 2

The value of DLM_MHI_PARTIAL_SEARCH_RESULT_DISPLAY should be set such that the minimum number of successful queries are required. In the above scenario, if the search fails for 100,000 results, the recommended values of the environment variable would be 10,000 or 25,000. A value such as 1000 or 100 would result in large number of queries, which is undesirable.

The value of this environment variable should be set so as to optimize performance. The optimal values will differ depending on the various search criteria involved.

Notes

  • Unfiltered objects may be loaded from a search result, however changes made to such objects or any relations created with such objects in DPM are not saved back to the Manufacturing Hub. Any relation that is created in DPM to an unfiltered object does not get saved to the Manufacturing Hub. A message is displayed as a reminder of this when an unfiltered object is loaded.

  • AST Manufacturing Assemblies are not displayed in the search results when searching for a Product Component or Manufacturing Assembly.

  • The Reset button can be used to clear all fields of the General Search dialog box.

  • The order of attributes is same in DPM and DPE for Search and Filter tab and is based on Position in Finder property of an attribute. The Position in Finder property can be changed with DPE configuration tool. For Additional Search tab all attributes are sorted alphanumeric in Attribute name combo box. You can add additional search criteria for attributes in any order.

Limitations

  • Date-specific attributes can not be searched using "!=" relations. Therefore, the "!=" option is not displayed for date-based parameters.

  • The search attributes of pointer data type configured with the Display in Finder option in DELMIA Process Engineer, will not have their values displayed in the V5 PPR Hub Search dialog box. These attributes cannot be used for the Search operation.
