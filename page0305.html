Submitting Batch Jobs Locally and Remotely

The AnalysisUpdateBatch dialog box enables you to submit batch jobs locally or to a remote computer. Batch job submission is similar to the same process in the Generative Part Structural Analysis (GPS) workbench, but it also requires the following options and settings:

  • The job path you specify must be the shared network where all computation and scratch files are generated.

  • For Abaqus for CATIA V5 jobs, the AnalysisUpdateBatch dialog box disregards the directory you specify in the Folder to Save Computed Data field. Instead, Abaqus for CATIA V5 uses the directories specified in the job definition.

  • For remote submission, you must have an Abaqus solver installed and licensed at the remote computer that you specify.

  • In many Abaqus for CATIA V5 analyses an output database file of analysis results is attached to the .CATAnalysis file. If you do not attach the previous analysis results, you must perform the following additional steps to include the analysis file in a batch job:

    1. Create a new analysis case and steps of the same type. Abaqus for CATIA V5 automatically creates a No Results solution step, which is required for batch update.

    2. Copy all features from the previous analysis case, and paste the features into the newly created analysis case in the appropriate step.

    3. Delete the old analysis case. Save this analysis file and submit the analysis for batch update.

This task describes how to perform a batch update for an Abaqus for CATIA V5 analysis file. For a complete description of all batch job options, see Computing Using a Batch, which is included in the documentation for the Generative Part Structural Analysis (GPS) workbench.

  1. Select Tools>Utility.

    The Batch Monitor dialog box appears.

  2. From the option on the Utilities tabbed page, double-click AnalysisUpdateBatch.

    The AnalysisUpdateBatch dialog box appears.

  3. Specify the name of the .CATAnalysis file that you want to submit.

  4. Specify the location where the batch job should be run:

    • Choose Run Local to run the analysis on your local computer. You must have an Abaqus license on your computer to run the batch job locally.

    • Choose Run Remote and specify the name of the remote host computer to run the analysis on a remote computer.

  5. Click Run to submit the batch analysis job and to close the AnalysisUpdateBatch dialog box.
