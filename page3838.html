Automatic Re-planning

The Automatic re-plan of unchanged Engineering Requirements command is used to perform an automatic re-plan for unchanged Engineering Requirements in a part, manufacturing assembly, manufacturing kit, or black box in a DPM session.

This is useful in scenarios where Engineering Requirements have been planned inside a part, but after performing tasks such as copying and pasting a part, changing its effectivity, etc., the unchanged Engineering Requirements need to be re-planned. This is also true for a black box scenario, where after changing a single component inside a black box the whole black box must be re-planned.

Command behavior in CMC and non-CMC projects

The Automatic Re-plan command behaves differently for projects that are Change Management Controlled (CMC) and those that are not.

In a CMC-based project, it is checked whether the project was loaded within the context of a given action proxy and modification statement. An action proxy and modification statement must be selected when loading a project to use the Automatic Re-plan command in a CMC project. When a modification statement has been specified, the system automatically copies the relevant relations from the source object to the target object, and the effectivity on the new relations is set per the modification statement. The effectivity on the relations for the source object is also reduced to the appropriate levels.

For a project that is not CMC, you are prompted to select the source and target objects, and then specify a new effectivity range that is applicable for the new relations that are created. The system then copies the relevant relations from the source object to the target object, and the effectivity on the new relations is set by you, and the effectivity on the relations for the source object is reduced to the appropriate levels. You also have the option to delete the old relations for the source object.

When the command has completed, a report is displayed that indicates the number of the various planned and unplanned components in the source and target objects.
  • Both source and target parts must be available in the same DPM session for automatic re-planning.
  • In a CMC-based project, an action proxy and modification statement must be selected when loading a project to use the Automatic Re-plan command

Using Automatic Re-planning

  1. Load a Manufacturing Hub project in DPM having both the old and new ARM parts. For CMC projects, select an action proxy and modification statement prior to loading the project.

  2. Click Automatically Re-plan unchanged ERs .

  3. In the PPR tree, select the source object.

  4. Also in the PPR tree, select the target object.

  5. For a non-CMC project, the Select Effectivity dialog box is displayed and prompts you to enter effectivity information for the new relations, and also given the option to delete the old relations. (In a CMC-based project, this dialog is not shown.) Enter the appropriate information and click OK.

    A status message is then displayed upon successful completion of the command.
