Reporting and Printing

 

Generating a  Reconciliation Report

At any time during the reconciliation process, you may use the report functionality to generate a report of the current reconciliation choices you have defined.  When you select the report function, an xml file is saved to a user specified directory and is launched in the default web browser (the browser must be compatible with xml and xsl).  The xml file references a style sheet which controls how the information will be displayed in the browser. The browser can also be used to print the report. 

Customizing a  Reconciliation Report

Using Tools > Options > Infrastructure > Reconciliation, in the Report Generation section, you can customize:

  • the layout and content of the report by specifying the style sheet (input xsl) that is used to translate the generated xml file into the displayed html file

  • the output directory the generated report xml file will be saved to.

Note:  the Output directory and Input XSL may be specified using DLName by entering CATDLName:://My_DLName in the appropriate text box. DLName is a data storage strategy enabling users to store data only in directories referenced by the specified DLName.  For more information about setting up and using DLName data storage, please see the CATIA Infrastructure User Guide. 


 

How to Generate a Reconciliator Report and Print

  1. In VPM Navigator, open a reconciliator session with the data you want to generate a report of.  This scenario uses data from Use Case 1: Reconciling a Product Structure to illustrate the reporting function.

  2. You may generate a report at any point during the reconciliation process.  In this scenario, the following mappings and rules have been established:

    • all the instances have been mapped to their corresponding VPDM objects except for Hub1-010-DEF-1001.

    • the following rules have been set (note that no rule has been set for Wheel-010-GHI-1021):
      • Board2-010-DEF-1009: Overwrites the board stored in the VPDM
      • Axle-010-JKL-1031: The manufacturer wants to keep the original axle
      • Hub2-010-DEF-1002: This object is new and does not exist in the VPDM
      • Wheel-Large-010-GHI-1026: This object is new and does not exist in the VPDM.

  3. Click the Report icon to generate a report.

    Your default browser displays the report and the xml file is saved to the location specified in the Report Generation: Ouput directory.

     

     

    The default report:

    • specifies the VPDM system the reconciliation will act upon.

    • details each object's name, type, storage mode, current rule definition and file location

    • shows the structure being reconciled with the VPDM

    • shows a legend of all the icons is also given.

    The xml file may include some additional information including object attributes and mapping information.

When launching the report function from the Compare View, the Instance section of the default report (i.e. the report generated with the default xsl style sheet) will contain additional comparison information.
 

  1. Print the report using the browser's print functionality.
