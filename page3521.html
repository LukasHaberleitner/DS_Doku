Displaying Child Parts

By using Check assignment of child parts the child parts of the selected product node can be displayed in the dialog List Associated Activities regardless of whether the parts are linked to a process. In addition you can display Engineering Intents that are linked to processes with the relation Process Implements Requirement.

See Displaying child parts in the dialog List Associated Activities.

In the dialog Tools > PPR Tools > Part Consumption you can set the relations that are to be checked; you can also activate the option Check assignment of child parts, if the children of the selected product node should be checked and displayed.

Part Consumption

Set the options before starting the session. Assignment of child parts is deactivated in the standard configuration.

  • Click Tools > PPR Tools > Part Consumption.

 

  • Under Highlight Unassigned Parts set the color with which the unlinked products are displayed in the 3D viewer.

  • If you activate Check assignment of hierarchical parents, the hierarchical structure is checked upward to the highest parent node for the selected product node. If there is a product node link to a process in the hierarchy, the linked parts are displayed under the column Parent in the results dialog for the List associated Activities command.

Example: Products – parent node linked with process.
 


With the two arrows  you can assign relations to the two windows Available Relations and Relations to check.

  • In order to display Engineering Intents, select the relation Process Implements Requirement under Available Relations. Then click on the rightward arrow in order to assign the relation to the window Relations to check.

  • Activate Check assignment of child parts in order to display the child parts; if Check assignment of child parts is deactivated, only the selected product node is taken into consideration – child parts are not taken into account.

  • Click OK after you have set the options.

Engineering Intents are displayed only if they are linked to a process with the relation Process Implements Requirement.
