Importing a Product

This task explains how you import a product, and work packages, from ENOVIA.
You need to import the product because you will be creating work packages under it. The top level directories - product class root and product class - remain in ENOVIA, only the product is imported. You must import the product (and existing work packages) each time you start a new session.

 

 1. In the Product Editor, right click on the product and select Send To - XXX, XXX being the application you are sending it to (such as CATIA V5). The product displays in the specifications tree.
2. You also need to import all the work packages that you need from ENOVIA to this application each time you start a new session. The process is as described above (select the document associated with the work package and go to Step 2 above). Note that if you send a work package then the product is also sent - you do not need to send both.

You can use filters to determine which work packages you need. It is best, though not essential, to import all the work packages that you need in one operation.
