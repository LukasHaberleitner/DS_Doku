Customizing Pre-V5R18 Databases to use Automatic Pre-selection

This section provides mandatory operating instructions to system administrators who need to ensure that the automatic pre-selection mechanism works when using SmarTeam databases versions anterior to Version 5 Release 18.

This task shows you how to integrate a dedicated attribute in the SmarTeam database. For more information about automatic pre-selection, see Automating Reconciliations Using the Automatic Pre-selection Command.

  1. Launch the SmarTeam Data Model Designer tool.

  2. From the File menu, select Modify Database Structure.

  3. Select your database.

  4. Create the CAD_TIMESTAMP new attribute for the Documents super class.
    Note that CN_ is automatically added.

  5. Modify the attribute definition as follows: set Long Integer as the attribute type and enter 4 as the size.

  6. Save the modification in the database.

  7. Launch the Integration Tools Setup utility.

  8. Open the Special Attribute group tree.

  9. Create an attribute property named CAD TimeStamp.
    You must enter the exact name: as indicated, use one blank character and upper/lower cases. The application only checks this attribute property name and retrieves the database attribute name from it.

  10. Add an attribute mapping by selecting the Documents class and the CAD_TIMESTAMP attribute previously created in SmarTeam Data Model Designer.

For reference information about SmarTeam Data Model Designer, see the SmarTeam-Editor documentation.
