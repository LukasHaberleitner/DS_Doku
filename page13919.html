Enabling Design Compare

 

This section explains the steps necessary for enabling the Design Compare functionality and describes the script that must be attached in order to enable the loading of different revisions of the same CATIA Part or CATIA Product at the same time.
 

Enabling Design Compare for CATIA Parts

  1. In Script Maintenance, select CATIA Part from the Documents class. In the right pane, open the User Defined tab.

  2. In the Operation column, add a key: Design Compare.

  3. Double-click in the Function Name cell of the Design Compare operation. The Script Browser window opens.

  4. In the File Name column, select DesignCompare.bs.
    In the Functions column, select DesignCompare.

  5. Click Compile.

  6. After clicking OK, the DesignCompare script is hooked to the database:

Note:   Following this procedure, you may compare several CATIA parts stored with CATIA Part class in the database. To compare several CATIA parts stored with other classes, e.g., CATIA Standard Part class, repeat the same steps for other CATIA Part files classes.

 

Enabling Design Compare for CATIA Products

The Design Compare for CATIA Product function is performed in the same way as the Design Compare for CATIA Part function.
