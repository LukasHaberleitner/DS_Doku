Design Compare

 

This section describes the functionalities related to Design Compare. This part of the solution groups several scripts and dll files in order to load different revisions of the same CATIA Part or CATIA Product at the same time. 
 

A CATIA Document is identified by its Universal Unique Identifier (UUID) and by its File Name. The combination of these two identifiers prevents duplicate loading of the same file. Similarly, different product/part revisions of the same file cannot be loaded in the same CATIA session.

However, the Design Compare option offers the following functionalities to you to perform a comparison of different revisions of a file:

Part Comparison:  Allows you to load different revisions of the same CATIA Part at the same time in one CATIA session. Product Comparison:  Allows you to load different revisions of the same CATIA Product at the same time in one CATIA session.

Both of these functionalities copy the document and change its file name. Selected parts are copied into the work directory and are renamed automatically based on the following rules:

  • File Name is renamed to FileName_RevisionNumber.

  • Part Number is renamed to PartNumber_RevisionNumber.

Note: “Part Number” is also changed to facilitate recognition of different revisions.

 

For more information, refer to Part and Product Comparison.
