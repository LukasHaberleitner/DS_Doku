Computing Using the AnalysisUpdatePLMBatch

This task shows how to update, compute and save in a database a .CATAnalysis document using the AnalysisUpdatePLMBatch.

For more details about computation, refer to Computing Object Sets.

To know how to use the batch monitor, refer to Running Batches using the Batch Monitor in the Infrastructure User's Guide.

  1. Open the Batch Monitor.

    For more details about the use of the Batch Monitor, refer to Running Batches using the Batch Monitor in the Infrastructure User's Guide.

  2. Double-click AnalysisUpdatePLMBatch in the Batch Monitor.

    The AnalysisUpdatePLMBatch dialog box appears.

    • Document Selection: lets you select a .CATAnalysis file you want to update, compute, and save in the database.
      Multi selection is not available: you can update and compute only one .CATAnalysis file using the batch.
    • Compute Option: lets you select the computation type you want to perform. Select:
      • Mesh only: to update only meshes, properties, materials, preprocessing specifications (restraints, masses, and loads) without requiring the computation of solutions.
      • Compute all: to compute all the solutions.
      • Adaptivity: to compute with adaptivity. See Computing with Adaptivity.
    • Save Management: lets you rename the computed .CATAnalysis document and its associated computation and result files (.CATAnalysisComputations and .CATAnalysisResults) that are generated when you compute the .CATAnalysis document.

      If the Automatic clearing computation data check box is selected in the External Storage settings, the computation file will be deleted after computing the .CATAnalysis document.
    • Output: lets you define the name and the path of the report file. The report file contains information about the computation.
    • Licensing Setup...: lets you select a license authorizing the use of the batch you want to run.
    • Save: lets you save the xml file in the desired location.
    • Run: lets you run the batch.
    • Cancel: lets you return to the batch monitor without launching the analysis PLM batch.

  3. Click to search in the database and select the .CATAnalysis document you want to compute.

    The Search Documents in ENOVIA V5 VPM dialog box appears. For more details about this dialog box, see Searching for Documents in ENOVIA V5 VPM in the Infrastructure User's Guide.

    Multi selection is not available: you can update and compute only one .CATAnalysis file using the batch.

  4. Select one of the following computation types:
    • Mesh only to update only meshes, properties, materials, preprocessing specifications (restraints, masses, and loads) without requiring the computation of solutions.
    • Compute all to compute all the solutions.
    • Adaptivity to compute with adaptivity. See Computing with Adaptivity.

  5. If you want to modify the Output options:

    1. Click to modify the target directory of the report file.

    2. To modify the name of the report file that give the general status of the computation, enter a new name in the Report name box.

    3. To add a suffix to the detailed report file, enter a string in the Append string to document report box.
      The name of the report file will be: [Name_of_document][suffix].htm

  6. Click Run in the AnalysisUpdatePLMBatch dialog box to launch the batch execution.

  7. To visualize the status of the batch computation, you can:
    • Open an HTML version of the report by clicking in the AnalysisUpdatePLMBatch dialog box.
      This button is available only once a report has been generated.
    • Select the Processes tab in the Batch Monitor.

    If the batch computation is successful, the selected .CATAnalysis document is computed and saved in the database with its associated documents (.CATAnalysisComputations and .CATAnalysisResults).
