Migrating Pre-R18 Features

This task shows you how to migrate pre-R18 features to have better integration and stability of these features.

The old subdivision surfaces that need to be migrated are indicated in brown indicator in the specification tree.

Open the Migrating.CATPart document.

  1. Double-click the subdivision surface from the specification tree or the 3D geometry to edit it.
    Or, right-click the subdivision surface from the specification tree to display contextual menu and select Subdivision Surface.x object > Migration.

    The Old Featured Detected message box is displayed indicating that migration of surface is required.

  2. Click Yes to confirm that you want to proceed with the migration.
    The Migration's End message box is displayed after the completion of the migration.
    If you click No in the Old Featured Detected message box then the migration of old surfaces will not be done.

  3. Click OK.

  4. Update the part.
    You can see that the specification tree is updated.
  • If there is more than one subdivision surface or curve in the CATPart, the migration of any one surface will automatically migrate the others.
  • If you select a command prior to selecting the old subdivision surface, a message is displayed as a tooltip in the geometry area indicating that the old feature is no longer handled and needs a migration.
  • If you select an old subdivision surface prior to selecting a command, the Old Featured Detected message box is displayed indicating that the migration of the surface is required.
  • The new migrated features are created at the same place as the old features.
  • The name of the feature does not change after the migration.
  • The general graphic attributes and material are retained after the migration.
  • Publication and colors applied to internal cells may be lost after the migration.
  • If an old subdivision surface needs an update, then clicking Update will turn the subdivision surface to not-up-to-date. The Old Featured Detected message box is displayed asking for migration. After the surface is migrated, it is updated.
