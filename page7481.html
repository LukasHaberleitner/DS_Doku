Creating Principal and Complementary Annotations

You can specify the principal and complementary annotations and describe the complete information with several characteristics for requirement generation. This helps manufacturing process in controlling the requirement generation in a more specific way.
Open a 3D shape with semantic and non-semantic annotations created on it

  1. Right-click a non-semantic annotation and select Associate to Principal Annotation.

    You can select several annotations at a time. In this case, valid annotations are associated and a warning message is displayed.

  2. In the specification tree, the complementary annotation node is displayed as a child under the node of its principal annotation. For more information about principal and complementary annotations, see FTA Principal and Complementary Definitions.

    • The principal and complementary annotations will maintain their relation after copying the annotation set in target data in product data filtering, tolerancing schema, annotation set power copy and import annotation set processes. For copy paste and power copy of individual annotations in the annotation set, the link between principal and complementary features break. These features are then treated as normal features.
    • After creating principal and complementary association, the positional links from the complementary annotations to the principal annotation are created when the following criteria are met:
      • Complementary annotations are in same view as that of principal annotation.
      • Complementary annotations do not have existing positional link with any other annotations.
    • If complementary and principal annotations are in different views, even after principal-complementary association, the views of the annotations remain unchanged.
    • The manufacturing property of complementary annotation is non-editable when it is associated to a principal annotation.

  1. Right-click the complementary annotation and select Dissociate from Principal Annotation.

    • If you select several annotations and at least one annotation is a complementary annotation, the Dissociate from Principal Annotation is displayed.

    • After dissociation, the existing positional link between complementary and principal annotation remains as it is.
