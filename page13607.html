XML Export: Recommendations and Warnings

Prior to exporting Clash Results in XML Format, it is recommended that you read carefully the following sections:

  • Recommendations

  • Important Warnings

Insert the following sample cgr files: ATOMIZER.cgr, BODY1.cgr, BODY2.cgr, LOCK.cgr, NOZZLE1.cgr, NOZZLE2.cgr, REGULATION_COMMAND.cgr, REGULATOR.cgr, TRIGGER.cgr and VALVE.cgr.

They are to be found in the online documentation filetree in the common functionalities sample folder cfysm/samples.
 

Recommendations
 

  • Use Internet Explorer 6.0 or higher on Windows, and Netscape 7.0 (or higher) or an equivalent browser complying with the W3C standard for XSLT 1.0 transformations on UNIX.

  • When renaming products or adding comments to your clash results, you can now use the following characters <, >, &, /., : as these characters are now supported in XML export.

  • By default, the UTF-8 encoding is used at XML file generation.
    <?xml version="1.0" encoding="UTF-8"?>

  • Note Browser automatically opened option is available only when working under Windows operating system. It is not available under Unix.

  • Before exporting interference results saved in ENOVIA LCA and sent to CATIA, you need to set specific options in Tools > Options > Digital Mockup > DMU Space Analysis > DMU Clash tab to make sure your generated XML file will contain pictures.

  • For more information, read Retrieving Information from ENOVIA LCA - Customizing XML Export Clash Settings section in the Interoperability chapter.

Important Warnings

Two xml types coexist in DMU Space Analysis and their behaviors differ from each other when exporting clash results.

  • First XML type: when the XML Export for clash process purpose check box is cleared. Any filter applied to the results in the Clash dialog box is taken into account so you can use the filter to export only pertinent results.

  • Second XML type: when the XML Export for clash process purpose check box is selected in the DMU Clash - Process tab page (Tools > Options > Digital Mockup > DMU Space Analysis,  the results are not filtered in the XML file.

  • Second XML type: to ensure the availability of Japanese language support, select the Support for long filenames check box in DMU Clash - Process (Tools > Options > Digital Mockup > DMU Space Analysis).

  • Batch processing XML export: only the second XML type is supported.
    Images are not created while exporting Interference results in batch-mode.

  • Style sheets: note that the clash.xsl style sheet is dedicated to the first XML type and the UnifiedStyleSheet_xx.xsl style sheet is dedicated to the second XML type.

  • Browser support: specify the  file:/// prefix before your relative path in the Style sheet location path (in Tools > Options > Digital Mockup > DMU Space Analysis > DMU Clash tab).

  • Wireframe visualization mode is not retrieved in the picture of the XML file.
