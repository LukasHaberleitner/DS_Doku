Exposing Knowledgeware Objects from Multiple CATfct Files

This task explains how to expose Knowledgeware objects from multiple CATfct files by defining the project resource EQUIPSYSTEMS_USERDICTIONARIES in the project resource management file (PRM).
Typically, you do not need to expose Knowledgeware objects from more than one CATfct file per application. This would be necessary if, for example, you needed to use Piping equipment and HVAC equipment from separate CATfct files.

Knowledgeware provides certain functions that are used by all applications. For more information, see Using Knowledgeware Packages.

1.

 To expose Knowledgeware objects from one or more CATfct Files, define the project resource EQUIPSYSTEMS_USERDICTIONARIES in the project resource management file (PRM).

Modify the resource EQUIPSYSTEMS_USERDICTIONARIES and point it to a text file called userdictionaries.txt. This text file contains a list of CATfct files to be loaded by Knowledgeware.

  2. Set a new environment variable by opening the Environment Editor dialog box and entering the PRM file name against the line EQUIPSYSTEMS_PROJECT.

For example, if you name the PRM file ProjectXyz, you enter EQUIPSYSTEMS_PROJECT= ProjectXyz.

Make sure that the PRM file is located in the directory path pointed by the variable CATDisciplinePath. For more information, see Understanding Project Resource Management.
