Save Documents

Save Documents> Choose SmarTeam project at first save
  • Clear the check box to display the Project Manager window only when the root assembly is saved in SmarTeam for the first time.
  • Mark the check box to display the Project Manager the first time a Save to SmarTeam operation is performed.
    Note: When performing Save to SmarTeam on a part under a root object, the PM window will appear even if the root object was previously saved.

Save Documents> Load the contextual assembly
  • Clear the check box to prevent the SmarTeam Save operation from loading the context during the update of design links, in order to improve performance
    and memory consumption.
  • Mark the check box to load the context during the update of design links.

Save Documents> Compute SmarTeam quantity based on instances activity
  • Clear the check box to ignore the instances activity information.
  • Mark the check box to send the instances activity information to SmarTeam to allow it to compute the Quantity attribute based on this information.

Save Documents > With Unique File Name

  • If Saving a document in SmarTeam is expected to generate a new file name for it (for example, by setting SaveWithUniqueFileName=YES in configuration editor), then that document must be loaded in CATIA session.
