Generating a Part Catalog Report

This task shows you how to generate a part catalog report. The report displays values for part attributes that are specified by you.              

1. Create a file that contains a list of the attributes whose values you want to see displayed in the report. Use a text editor like WordPad to create the file. Create two column headings: Keyword Name and Possible Value. The keyword names you can have are:
  • CATIA_PartAttributeName. Used for part attributes.
  • CATIA_ConnectorAttributeName. Used for connector attributes.
  • ENOVIA_Attribute. Used for ENOVIA attributes. This only works if you have set up the ENOVIA-CATIA attribute mapping table as described in ENOVIA documentation.
  • CatalogName. Use this if you want to enter the names of catalogs from which to generate reports. You can also select a catalog using the procedure described below.
  • CatalogReportPath: Use this to enter the location of the output file. You can also specify this using the procedure described below.

Enter the attributes as shown below: under the column Keyword Name enter the correct keyword. Under the column Possible Value, enter the attributes as defined in the parts catalog. You can have as many entries as you want.

NOTE: You can also enter the names of catalogs from which you want to generate reports. If you do this, then the Catalog Name field explained in Step 5 below will not be available. The catalog name(s) should be entered in the following format. The directory path is for the sample catalog - replace with your own path if needed:

Keyword Name: CatalogName

Possible value:  ...\intel_a\startup\EquipmentAndSystems\Piping\PipingDesign\ComponentCatalogs\
Parametric\PipingParts.catalog. If you are generating a report from a catalog in ENOVIA, then use the catalog name only, not the directory path.

You can enter the location of the output report also, in which case the Output Directory field explained in Step 4 will not be enabled. To enter the output directory use the Keyword Name "CatalogReportPath" and enter the path in the Possible Value column.

2.

 Click Tool > Report > Part Catalog Report in the menubar to display the Part Catalog Report dialog box. In the image below values have been added to the fields. When the box first displays the fields will be blank.
3. In the Report Template File field, navigate to the attributes file you created in Step 1.
4. In the Output Directory field, enter or navigate to the directory where the report will be generated. See note above.
5. To select a catalog from which to generate a report, you must open the catalog in your current session. All catalogs that are open will display when you click the down arrow next to the Catalog Name field. See note above.
6. Click OK to generate the report. Two reports will be generated and placed in the output directory: PipingParts_Report and PipingParts_ConnectorReport. All Piping Parts reports (for both 3D and 2D applications) will contain the following five columns:
  • Document Name
  • CATFct name
  • Chapter
  • Family
  • PartType.

These columns will be followed by the attributes you entered in the attributes file.

Similarly, all connector reports always contain the following column headers:

For 3D applications:

  • Part Type
  • PartNumber
  • Connector Type
  • Connector Flow
  • Face
  • Alignment
  • Orientation
  • Datum

For 2D (Diagrams) applications:

  • Part Type
  • PartNumber
  • Connector Type
  • Connector ID
  • Connector Flow
  • Associated Size Attr

These columns are followed by the attributes you enter in the attributes file.

All ENOVIA-related attributes are also contained in the Connector Report.
