Checking Plies Groups for Coincident Edges

This task shows you how you can check plies groups to identify coincident edges in ply or cut-piece contours. The analysis is performed on all the plies groups you have selected or that are found under the stacking.
Open a part containing a stacking or plies group, for example GridSynchroLogical.CATPart from the samples directory.

  1. Right-click a stacking or a selection of plies group and select Coincident Edge Analysis.

  2. Select either a group of EOP or geometry under one group of EOP.
    Multi-selection is supported.
    Coincidences found on EOP are ignored.

  3. Select the Auto Computation check box.
    When several plies to check are full plies, either create EEOP/MEOP from the full plies, or select Auto Computation for more relevant results.
    Selection of EOP is no longer possible.
    A temporary external contour is computed. Coincidences on this contour are ignored.

  4. Click Run Coincidence Check.
    Potential coincidences are computed and identified with a 0.1 mm tolerance.
    Results, if any, are displayed:
    • In the dialog box: Index, first and second features causing the coincidence, type of the coincidence.
    • In the 3D Space, overlap coincidences are displayed in red, butt coincidences in blue.
      • Overlap coincidence
      • Butt coincidence
    • If EEOP have been selected, EOP boundaries are displayed in orange. Only coincidences not on EOP boundaries are displayed.

  5. Select the required option to filter the results.

  6. Select one coincidence in the dialog box.
    A reframe is performed and information is displayed in the 3D Space.

  7. Alternatively, Select several coincidences in the dialog box to display their information in the 3D Space.
    Right-click a coincidence in the dialog box to perform a reframe or a center graph.

  8. Select one coincidence in the dialog box and the Display Plies check box.
    Tip: Hide the support surface.
    The ply or cut-piece shell of the first coincidence element is displayed in white,
    that of the second element is displayed in black. Overlap shell, if relevant, is displayed in grey.
