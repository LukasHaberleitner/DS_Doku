Creating Move Activities with Imported Tracks

This procedure describes how to create move activities using geometry that already has a track created. 

Depending on the CAD program used to create the original product geometry, a product may have been created with a track. A track is the path that a move takes.  If so, that track can be imported and used to create a move activity.

If, in addition to using DELMIA DPM Assembly Simulation, you have the CATIA DMU Fitting product, you may have used the DMU Fitting commands to define a path for a product.  The product may also contain other activities in a sequence, such as visibility activities.  Using Imports the Track does not import any other activities in a sequence; it imports the specified track or tracks and creates move activities from the tracks.
Importing a track created with a product requires that the product data containing the track already be loaded into this process. If the product data does not have track already defined, use the regular Create Move Activity .
 

  1. Click Imports the Track .

  2. On the PPR tree, select either the process or the activity before which you wish the move activity to appear.

  3. Select the product that already has a track defined as part of it.

    The Import Track dialog box appears:

  4. Select a track (more than one may appear), and then click OK.

    A new move activity appears in the process list.
You can select multiple tracks for importation by holding down the Ctrl key when you use the mouse to select the tracks.  Clicking OK imports the tracks, as move activities, into the process in the same order in which they appear in the Import Track dialog box.

The track is imported relative to its part's current position; the track does not necessarily retain the placement it had when it was created.
