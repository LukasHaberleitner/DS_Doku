Migrating V4 Models to V5

This section explains how you can migrate your V4 models to V5.

To explain the process briefly, you have to convert the classes and attributes in your V4 Project Registration Model to an XML file and ftp it to a V5 directory. You have to do the same with the classes and attributes in your V5 Feature Dictionary.

A tool compares these two XML files and then imports the classes and attributes that do not exist in V5 from the V4 XML file and makes them available to you in V5, after which you can migrate the model.

You can also copy the geometry of a single piece of equipment from a V4 model and paste it into a V5 document.

In other applications, such as Piping Design and HVAC Design, a migrated discipline is required before migrating models. For Equipment Arrangement this is optional. If you want to learn how to store your data in a location other than in the sample directories provided with this application see Creating a Directory Structure.

You should make sure that your resources are correctly referenced in the project resource management file.

To learn how to customize line thickness, see documentation for Interactive Drafting in the section Advanced Tasks/Manage Standards. Once defined, line thickness is applied to elements as a property.
