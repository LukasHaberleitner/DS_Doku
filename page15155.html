Reconciliation Toolbar

The reconciliation toolbar enables you to perform actions on objects to be reconciled.

Magic Reconciliation: Enables you to launch a Global set with its default options automatically, without any confirmation.
Save session: Enables you to save the content of the Reconciliation session.
Note: The information is captured in an XML, similar and compliant with the generated batch utility files.
Restore session: Enables you to restore the content of the previous Reconciliation session such as selection of VPDM system, storage mode, mapping with a VPDM object and Reconciliation rule.
Information such as selection of view, assembly structure filter or display state will not be preserved.
Overwrite in VPDM: The object loaded in the CATIA session overwrites the one stored in the VPDM repository.
Reload from VPDM: The VPDM object version is kept. This functionality is applicable for both Structure Exposed and Publication Exposed.
New Object in VPDM: The object is considered as a new one.
Note: This reconciliation rule is available only if the object is saved in Publications Exposed mode.
Keep as External: The broken link will be accepted as "broken" during the reconciliation process. After saving the documents in the database, the external reference will:
  • remain "broken" if it belongs to a document saved as Publication Exposed.
  • definitively disappear if it belongs to a Structure Exposed document.
New revision : The document loaded in the CATIA session overwrites the one stored in the ENOVIA V5 VPM repository and the document will be saved to ENOVIA V5 VPM as a new revision.
New version : The object loaded in the CATIA session overwrites the one stored in the ENOVIA V5 VPM repository, the part will be saved to ENOVIA V5 VPM as a new version and the corresponding geometry document will be saved as a new revision.
Overwrite by delta: This rule is applied when the Part is set to Structure Exposed from Storage mode and is mapped on either an ENOVIA V5 Part Version or an ENOVIA V5 Product Root Class (PRC).
New version by delta: This rule is applied when the Part is set to Structure Exposed from Storage mode and is mapped on ENOVIA V5 Part Version.
Filter on PRC: Allows you to apply a filter to the VPDM view.
Locate: Enables you to query the VPDM in order to establish mappings with the CATIA View.
Report : Generates a report of the current reconciliation choices you have defined. The report is saved in an xml file and displayed in your default web browser. The report can be printed directly from the web browser.

Storage Mode: See Selecting the Storage Mode.
Save Application Container (ENOVIA ENOVIAVPM only): When CATProduct documents are saved in Structure Exposed mode, you can decide to save the applicative data using an application container.

Destination PDM: Enables you to select the destination PDM. By default, the destination PDM is the one selected in the Tools>Options>Infrastructure>Product Infrastructure-Reconciliation window.
View Type: This scrolling list enables you to select a view type.
Part & Document Both tabular and graph views represent the CATIA reference view. All CATIA documents, products and components that belong to the reconciliation transaction are represented.

In this view mode,

  • selecting a Part in the objects to reconcile tree implies that you want to work in Structure Exposed mode.
  • selecting a document in the objects to reconcile tree implies that you want to work in Publications Exposed mode.

Document Both tabular and graph views display the objects to be reconciled as a tree of documents. Note that mapping cannot be fully done from this view (i.e. if CATProducts, whose storage mode is Structure Exposed, have components, the components cannot be mapped from this view. The Instance view or Part & Document view must be used).

Instance Both tabular and graph views represent the CATIA Product structure view, i.e. the objects presented with instances. This is the default view.
Compare Both tabular and graph views represent a merged CATIA/ENOVIA V5 VPM Product structure. In this mode, you will identify identical instances, modified positioning, availability only in CATIA or only in the VPDM.
This comparison will enable you to forecast the impacts of the reconciliation on the Product structure stored in the VPDM after applying the reconciliation and saving the data both in CATIA and in the VPDM system.
The Compare view will be available at any time (even before selecting any reconciliation rule) and the objects for which no reconciliation rule is defined will be analyzed as New objects.
In this context, only CATIA objects can be selected.
Note:
  • The compatibility of XML files used for memorizing Reconciliation session state has a time limit. These data could be incompatible between Releases, SPs, or even after Hot-fix installation and therefore not to be referred as reference data.
  • If the XML content reconciliation information exists in current Reconciliation session, nothing will happen.
  • The mapping consistency is not checked against the VPDM system.

3D Viewer: For all PDM, this tab enables you to visually compare the object loaded in the CATIA session and the one stored in the VPDM. In the VPDM 3D View area, 3D visualization is available for CATParts, CATShapes, CATIA V4 models and native cgr. For all PDM, 3D visualization is available in the VPDM 3D View area, if you activate the cache by selecting the Work with the cache system check box. For more information about Cache Activation, see Cache Management.

For ENOVIA V5 reconciliation, the 3D visualization is available in VPDM 3D View area even without activating the cache; as soon as .cgr files are available in ENOVIA V5 server. These .cgr files are generated from the ENOVIA V5 server by enabling the file introspection process on the corresponding V5 server. If CATIA cache and file introspection are not activated on the server, 3D visualization is generated on the fly and is available in VPDM 3D View area.

In the CATIA 3D View area, 3D visualization is available for CATProducts, CATParts, CATAnalysis documents and CATIA V4 models.

List View

This view is a tabular view of the reconciliation tree.
Compare Is displayed only if you select the Compare option in the View type scrolling list. It indicates the status of the objects after saving them in the VPDM.
ID Indicates the name of the displayed object.

Mode

 Indicates the storage mode applied to the documents: Publication Exposed or Structure Exposed.
Rule Indicates the rule applied to the document, for example, Overwrite in VPDM, Reload from VPDM, or New Object in VPDM.

Type

 Indicates the type of the displayed object: the values are different depending on the view you have selected (Instance, Document, Part & Document, Comparison)
Document Indicates the name and the path of the document.
Environment Indicates the selected database (ENOVIAVPM or ENOVIA V5 VPM).

Characteristics




This tab enables you to access the comparisons result after comparing the document loaded in the CATIA session and the document stored in ENOVIA.
  • The Characteristics column displays the characteristic currently selected for comparison. If there is more than one in the list, they will be shown one after another.
  • The Status column displays the status of the characteristic.
  • The CATIA column displays the detailed comparison value from CATIA.
  • The PDM column displays the detailed comparison value from VPDM.

Ambiguities

 

This tab enables you to identify the types of ambiguities occurred while reconciling an object. The ambiguity types are as follows:

  • Persistency Ambiguity: This ambiguity takes place if multiple Storage Modes are available for an object to be reconciled even when the default storage mode is applied. The object will be displayed in the Ambiguities tab informing that more than one storage mode is possible. Persistency Ambiguity can be resolved by selecting the appropriate Storage Mode. For more information about Storage Mode, see Selecting the Storage Mode.
  • Mapping Ambiguity: This ambiguity takes place if multiple query results are available for an object to be reconciled even when automatic mapping is applied. The object will be displayed in the Ambiguities tab informing about the multiple query results. Mapping Ambiguity can be resolved by selecting the appropriate PDM object from the Query result list if not mapped.
  • Rule Ambiguity: This ambiguity takes place if multiple rules are available for an object to be reconciled even when default rule is applied. The object will be displayed in the Ambiguities tab informing about the multiple rules. Rule Ambiguity can be resolved by selecting the appropriate rule if no default rule is selected.

Note: The list of ambiguities disappear from the Ambiguities tab once you manually resolve the issue or if you deliberately set a rule or explicitly confirm the default options.

Contextual Menu

This contextual menu is available when you right-click an object in the CATIA View.
Overwrite: The object loaded in the CATIA session overwrites the one stored in the VPDM repository. Note that when you work with ENOVIA V5 VPM, the root node must be designated Overwrite.
Reload: The VPDM object version is kept. This functionality is applicable for both Structure Exposed and Publication Exposed.
New: The object is considered as a new one.
Keep External Reference: The broken link will be accepted as "broken" during the reconciliation process. After saving the documents in the database, the external reference will:
  • remain "broken" if it belongs to a document saved as Publication exposed.
  • definitively disappear if it belongs to a Structure Exposed document.
New revision : The document loaded in the CATIA session overwrites the one stored in the ENOVIA V5 VPM repository and the document will be saved to ENOVIA V5 VPM as a new revision.
New version : The object loaded in the CATIA session overwrites the one stored in the ENOVIA V5 VPM repository, the part will be saved to ENOVIA V5 VPM as a new version and the corresponding geometry document will be saved as a new revision.
Overwrite by delta: This rule is applied when the Part is set to Structure Exposed from Storage mode and is mapped on either an ENOVIA V5 Part Version or an ENOVIA V5 Product Root Class (PRC).
New version by delta: This rule is applied when the Part is set to Structure Exposed from Storage mode and is mapped on ENOVIA V5 Part Version.
Reset: Enables you to reset the reconciliation rule and the mapping for the current object.
Locate: Enables you to query the VPDM in order to establish mappings with the CATIA View.
