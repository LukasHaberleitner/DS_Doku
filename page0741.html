Multiprocess  

 

   
  Multi-processes concern the CATDMUUtility processes that are created as child processes for the tessellation of documents when performing a File -> Open of a CATProduct or when running the CATDMUUtility batch process using the -mp option. This new feature enables the management of possible coredumps and / or looping problems during tessellation such that the impact is localized to the specific document that cannot be tessellated; the state of a V5 session will no longer be impacted by these problems and a CATDMUUtility batch process will run to conclusion. A log file indicates which documents could not be tessellated.
  
You must have activated the variable Multi process visualization mode with local cache in the Product Structure options. See Product Visualization.  
 

The Multiprocess settings page concerns the following settings categories:

  • Multiprocess parameters

The Multiprocess parameters enable you to customize:

  • The maximum number of child processes that will be launched.
  • The maximum amount of time a process will be allowed to loop before it is killed.
  • The path of the log file that will report the success or non-success of each tessellation concerned with the opening of a CATProduct.
     

 

Multiprocess parameters

 

Maximum son processes

  
The number of child processes is limited to 1.  
  In the Maximum son processes text-entry field, enter the maximum number of child CATDMUUtility processes to be used during tessellation.

By default, this option is valuated to 2.

  
     
 

Timeout (minutes)

  
  In the Timeout (minutes) text-entry field, enter the maximum time that a single tessellation process will be allowed to loop before being aborted.

By default, this option is valuated to 60.

  
     
 

Log File Path

  
  In the Log File Path text-entry field, enter the path of the log file of the tessellation process or click the Browse icon to choose a path via the File Browser.
