Optimal CATIA PLM Usability for Drafting

When working with Drafting in an ENOVIA LCA context, there are a few circumstances under which a drawing cannot be correctly saved in ENOVIA LCA.

More specifically, a drawing from an exploded product (i.e. an assembly saved as Document not kept & Product Structure Exposed) cannot be saved in the following cases:

  • when the drawing contains views referencing different instances of the exploded product: indeed, all generated views in the drawing must reference the same 3D elements.
  • when the drawing contains views referencing the following DMU components: 3D box or section.
  • when the drawing contains text files referencing knowledgeware attributes.

An error message is now displayed when trying to create views referencing such elements.
Note that this message is only displayed if the product from which you are trying to create the view has already been saved in the ENOVIA LCA database.
