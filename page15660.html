Instantiating a Part Template From ENOVIA V5 VPM Using VPM Navigator  

This task explains how to instantiate using the Document Chooser a part template stored in ENOVIA V5 VPM.
Make sure you have:
  • selected ENOVIAV5 in the View > Toolbars  menu.
  • enabled ENOVIA in the Document Environments field (Tools>Options>General>Document.)
It is now possible to associate non-CATIA (ENOVIA LCA, ...) documents with a template. To do so, make sure you have enabled the desired environment in the Document Environments field (Tools>Options>General>Document.) Your documents will be accessed via the Document Chooser.
To perform this scenario, you will need the following documents:
  • BottomCase.CATPart
  • Battery.CATPart
  • Electronic.CATProduct
  • PlanarCard.CATProduct
  • InteractiveBoard.CATPart
  • ChipAC110.CATPart
  • Chip_AC20.CATPart
  • Chip_AC30.CATPart
  • Capacitor_500.CATPart
  • Capacitor_700.CATPart
  • Speaker.CATPart
  • Body.CATPart
  •  Lens.CATPart
  • Indus.CATPart
  • LCD30-28.CATPart
  • FrontShell.CATPart
  • Main_Shape.CATPart
  • Screen.jpg

 

  1. Create a Product in ENOVIA called MobilePhone. Note that you can close ENOVIA V5 VPM after creating the product.

Send the Product to CATIA

  1. To send the MobilePhone product to CATIA:

    • In CATIA, click the Connect user to ENOVIA V5 icon ( ) in the ENOVIA V5 toolbar. The ENOVIA Logon dialog box is displayed:
    • Enter your user name, and your password.
    • Select a role and a workbook and click OK.
    • Click the Search ENOVIA icon (). The VPM Search dialog box is displayed.
    • From the Objects scrolling list, select the Product option.
    • If need be, enter the owner's ID.
    • Click OK when done. The Product ID is displayed in the Result window.
    • Double-click the MobilePhone product from the Results list.
    • Right-click the root product (MobilePhone) and select Open.... The Open Modes dialog box is displayed. Click OK. The CATIA view is displayed.

Add Components to the Product

  1. Click the root product (MobilePhone) and select Insert>Existing Component.... Select the following documents and click Open:

    • BottomCase.CATPart
    • Battery.CATPart
    • Electronic.CATProduct
    • Body.CATPart
    • Lens.CATPart
    • Indus.CATPart
    • FrontShell.CATPart
    		 

  2. Click the Save data in Enovia V5 icon (). The Save in ENOVIA dialog box is displayed. Click OK.

  1. Open the Main_Shape.CATPart file. This Part contains a document template (Keypad 1).

  1. Click the Save data in Enovia V5 icon (). The Save in ENOVIA dialog box is displayed. Click the button.

  2. Select the line referring to Main_Shape.CATPart and click Modify. Click the root product in the CATIA VPM view and click OK. The file is saved in ENOVIA.

  3. Close all windows in CATIA.

Instantiate the Part Template

  1. Load the product from ENOVIA V5 VPM.

    • Click the Search ENOVIA icon (). The VPM Search dialog box is displayed.
    • From the Objects scrolling list, select the Product option.
    • If need be, enter the owner's ID.
    • Click OK when done. The Product ID is displayed in the Result window.
    • Double-click the MobilePhone product.

  1. Right-click the Part instances then select Open.... The Open Modes dialog box is displayed. Check Automatic lock of all opened Parts and Documents  and click OK. The product is displayed. The mobile phone support is displayed.

  2. From the Start>Knowledgeware menu, access the Product Knowledge Template workbench.

  3. Click the Instantiate from Document icon.

  4. Click the Enovia icon. The Search Conditions dialog box is displayed. Enter the name of the revision owner and click OK.

  5. Select the Main_Shape.CATPart in the Search Result window and click Open.

  6. Value the Inputs by selecting the publications located below the Industrial Design node in the specification tree and click OK. The template is instantiated.
