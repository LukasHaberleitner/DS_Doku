Supporting CGR in Reconciliation  

The Reconciliator enables you to re-integrate the received CATIA data as file documents within ENOVIA V5 VPM and ENOVIA VPM V4. CGR documents and other 3D representation files can be reconciled, despite the assembly structure exposition in ENOVIA (Structure or Publication exposed). CGR format is a graphical representation of the geometry only that allows you to see a 3D visualization. You can automatically create and update CGRs by using File Introspection for 3D documents such as CATPart, CATProduct, CATShape or CATIA V4 documents.    

These CGR documents can be either attached as an alternate representation to a 3D geometrical representation (such as CATPart, CATShape, and CATIA V4 model) using Manage Representations in Representations Toolbar or to a PRC or any intermediate assembly nodes. The CGR document is available in the Objects to reconcile tree with the icon   for reconciliation in the Reconciliator Window dialog box just like CATShape or CATIA V4 model documents.
Note: For ENOVIA VPM V4, the Overwrite and New Version is inapplicable on alternate shape.


Standard Reconciliation rules for 3D representation document are applicable to CGR documents (such as New, Overwrite, Reload, New Revision and Keep as External). It is also possible to reconcile CGR document as a single document, as the document is associated as an alternate representation or as default 3D representation of an assembly node in CATIA.
