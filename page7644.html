Writing Axis Systems from a V5 Track to a V4 Model

This task shows how to export a V5 track definition in V4. Each V5 recorded position is exported as an Axis in one V4 model document. It is  then possible to rebuild a V4 track from these positions.

This functionality is very useful when you are designing products in CATIA V4 and reviewing your DMU in Fitting V5. You can easily retrieve information from your V5 review  to perform modifications in  CATIA V4.
The user needs to rebuild a route in V4 from the exported axis systems.
Open the EXPORT_V5_TRACK.CATProduct document.

  1. Double-click Track.2 in the specification tree.

  2. Select the arrow within Clash Detection from the DMU Check toolbar.

    You may need to undock the toolbar.  When you do, the toolbar looks like this:
     

    Clash Mode

  3. Click Clash detection (On) .

  4. Play your track simulation using the Player.

     
    There is a collision detected at recorded position 3. You need to modify your design.

  5. Select the track.2 either in the geometry area or in the specification tree.

     

  6. Select  Tools > Simulation > Track file export.

     
    The Track File Export dialog box automatically appears.

  7. Enter a meaningful name and select .model format from the Save as Type list.

  8. Open the EXPORT_V5_TRACK.model document you have just created.

     This is what you obtain:
    Double-click Master node to display the specification tree.
    In this V4 model document, we selected axis system 3 (highlighted).  In the dismounting review, it corresponds to the position 3 where a collision was detected.
    You are now ready to modify your design using this information.
    There are not any V4 Fitting entities created.
    • V4 tracks do not exactly share the same behaviors with V5 tracks (such as interpolation, etc.)
